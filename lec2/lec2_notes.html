Introduction to Artificial Intelligence

第 2 讲：逻辑与知识表示

用形式逻辑、数理逻辑、知识图谱与概率图，把现实世界转成可推理的结构。

学习目标

  • 区分形式逻辑、数理逻辑和数字逻辑在 AI 中承担的角色。
  • 理解知识图谱把实体和关系组织成有向图，并支持归纳与演绎推理。
  • 知道概率图把不确定性显式建模，是后续机器学习和决策问题的重要桥梁。

知识地图

模块内容
逻辑基础命题、谓词、规则、推理链，强调可解释性与严密性。
知识图谱实体、关系、三元组与路径；可以做补全、问答和关系推断。
概率推理把确定规则扩展为带概率的依赖结构，处理噪声和不完整信息。

核心概念讲解

要点 1

形式逻辑关注推理是否有效，数理逻辑把推理符号化、数学化。

要点 2

知识表示的关键不是“存储事实”，而是让事实之间的关系可查询、可组合、可推断。

要点 3

FOIL 属于归纳学习，PRA 借助路径排序做关系推理。

课堂讲解补充

逻辑与知识表示对应 AI 的“可解释推理”传统。与后续深度学习相比，本讲更强调符号、规则和关系结构：如果我们能把世界写成实体、关系和规则，就可以进行查询、补全和推断。它也提醒我们，数据驱动模型并不能替代良好的问题表示。

表达结构

复习时可以按“问题从哪里来、方法怎么做、公式或代码落在哪里、效果怎么评价、局限是什么”来整理。

与课程主线的连接

复习时顺手把它和前后几讲连起来：这一讲解决了什么问题，又给后面的模型或实验留下了哪些工具。

公式速查与解释

  • 三元组表示：(head entity, relation, tail entity)。
  • 贝叶斯思想：P(H|E) ∝ P(E|H)P(H)，把证据 E 转换为假设 H 的置信度。
阅读建议：不要孤立背公式；每个公式都要能说出变量含义、适用条件和在代码/实验中的位置。

从课堂到项目的标准流程

步骤操作要点
1. 定义任务先说清输入、输出、假设与评价指标，避免只罗列算法名。
2. 建立表示选择逻辑符号、状态空间、特征向量、张量或 MDP 状态。
3. 选择模型根据数据规模、结构先验和计算预算选择规则、搜索、线性模型、神经网络或 RL 方法。
4. 训练/推理明确损失函数、优化过程、推理步骤和失败模式。
5. 评估迭代用与任务匹配的指标验证效果，并记录可复现实验设置。

代码/伪代码示例

下面的代码只保留课堂最需要抓住的骨架；真正练习时再对照 notebooks、讲义和项目文件补全细节。

# 一个极小知识图谱查询示例
triples = [("Socrates", "is_a", "Human"), ("Human", "will", "Die")]

def infer(entity):
    types = {t for h, r, t in triples if h == entity and r == "is_a"}
    return [(entity, r, t) for h, r, t in triples if h in types]

print(infer("Socrates"))  # [('Socrates', 'will', 'Die')]

常见误区

  • 规则系统容易脆弱：知识不完整或规则冲突时会出现推理失败。
  • 知识图谱不是普通数据库表；路径、关系组合和语义约束才是重点。
考试与汇报提醒：回答问题时尽量使用“任务定义 → 方法原理 → 关键公式/代码 → 指标 → 局限”的结构。
