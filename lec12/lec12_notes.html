1. 目标检测概述 (Object Detection)

目标检测是计算机视觉中最基本、最具挑战性的问题之一。与图像分类（整张图 → 一个类别）和定位（单目标 → 一个框）不同，目标检测要求在图像中同时定位多个物体，并为每个物体生成一个带类别标签的边界框 (Bounding Box)。

先找 Bounding Box，再做分类 → **先粗后细**
Bounding Box 表示：通常用四个数字表示一个矩形框，如 \((x_1, y_1, x_2, y_2)\)（左上角和右下角坐标）或 \((x, y, w, h)\)（中心点坐标和宽高）。

"Object"（物体/目标）的定义是人为预设的，取决于训练集的类别设定。例如，在水果检测任务中，桃子、李子、梨是 object；而在一般场景检测中，它们可能只是背景。

1.1 Two-stage vs One-stage 检测范式

目标检测方法可分为两大范式：Two-stage（两阶段）先找候选框再分类，One-stage（单阶段）同时输出框和标签。

目标检测：Two-stage 方法 (R-CNN 系列)
维度Two-stage（两阶段）One-stage（单阶段）
核心思想先找候选框，再做分类 → 先粗后细同时输出 Box 和标签，一步到位
代表方法R-CNN, Fast R-CNN, Faster R-CNNYOLO, SSD
速度较慢，但精度高快，适合实时应用
检测范式先生成稀疏 RoI，再分类回归密集预测，直接回归
PipelineRegion Proposal → Classification + RegressionSingle-shot: 直接从图像预测框+类别

核心区别

Two-stage 方法将检测任务分解为"找框"和"认物"两个子任务；One-stage 方法将检测视为一个统一的回归问题，直接从图像像素映射到框坐标和类别概率。

1.2b Selective Search：外部候选框搜索详解

Selective Search (Uijlings et al., 2013) 是 R-CNN 和 Fast R-CNN 使用的传统候选框生成算法。它不是神经网络，而是基于图像结构的层次化区域合并方法，在 CPU 上运行。

核心思想

穷举搜索（滑动窗口）会产生海量候选框且效率极低。Selective Search 的思路是：先用图像分割得到大量超像素，再根据相似度层次化合并，从而在不同尺度上产生"可能包含物体"的区域。

Step 1: 超像素初始分割

使用 Felzenszwalb & Huttenlocher (2004) 的图分割算法，将原图分解为大量细小的超像素区域（superpixels）。该算法基于像素间的颜色差异构建图，通过最小生成树自适应地合并相似像素。

输出约 1000~2000 个初始区域 \(R = \{r_1, r_2, \ldots, r_n\}\)，每个区域内的像素颜色和纹理高度一致。

Step 2: 相似度度量

对每对相邻区域 \((r_i, r_j)\)，计算综合相似度 \(s(r_i, r_j)\)，由四个分量加权组成：

$$s(r_i, r_j) = a_1 \cdot s_{\text{color}}(r_i, r_j) + a_2 \cdot s_{\text{texture}}(r_i, r_j) + a_3 \cdot s_{\text{size}}(r_i, r_j) + a_4 \cdot s_{\text{fill}}(r_i, r_j)$$

其中 \(a_i \in \{0, 1\}\) 是开关（论文中四个都用，即 \(a_1=a_2=a_3=a_4=1\)）。

① 颜色相似度 \(s_{\text{color}}\)

对每个区域的颜色直方图（每个颜色通道 25 bins，共 75 维），用 L1 范数归一化后计算交集：

$$s_{\text{color}}(r_i, r_j) = \sum_{k=1}^{75} \min(c_i^k, c_j^k)$$

其中 \(c_i\) 是区域 \(r_i\) 的归一化颜色直方图。颜色相似的区域更应合并。

② 纹理相似度 \(s_{\text{texture}}\)

对每个颜色通道计算 高斯微分（Gaussian Derivatives, σ=1,2,4），提取 8 方向的纹理梯度，每个方向 10 bins，共 240 维直方图：

$$s_{\text{texture}}(r_i, r_j) = \sum_{k=1}^{240} \min(t_i^k, t_j^k)$$

纹理相似的区域（如草地+草地）更应合并，纹理不同的（如草地+天空）不应合并。

③ 大小相似度 \(s_{\text{size}}\)

鼓励小区域优先合并，防止大区域无限制膨胀：

$$s_{\text{size}}(r_i, r_j) = 1 - \frac{\text{size}(r_i) + \text{size}(r_j)}{\text{size(im)}}$$

其中 \(\text{size}(\cdot)\) 是区域内的像素数，\(\text{size(im)}\) 是整张图的像素数。两个小区域的 \(s_{\text{size}}\) 接近 1，鼓励合并。

④ 形状兼容度 \(s_{\text{fill}}\)

衡量合并后的区域是否"紧凑"（避免产生狭长的无意义框）。用包围框 (Bounding Box) \(BB_{ij}\) 内未被 \(r_i \cup r_j\) 覆盖的面积衡量：

$$s_{\text{fill}}(r_i, r_j) = 1 - \frac{\text{size}(BB_{ij}) - \text{size}(r_i) - \text{size}(r_j)}{\text{size(im)}}$$

合并后区域越填满其包围框，\(s_{\text{fill}}\) 越大，越鼓励合并。

Step 3: 层次化贪心合并

算法使用贪心策略反复合并最相似的区域对：

  1. 初始化：每个超像素作为独立区域，计算所有相邻对的相似度
  2. 合并最相似的一对 \((r_i, r_j) \to r_{\text{new}}\)
  3. 删除 \(r_i, r_j\)，加入 \(r_{\text{new}}\)
  4. 计算 \(r_{\text{new}}\) 与所有新邻居的相似度
  5. 重复 2~4，直到只剩一个区域

每次合并都会产生一个新的区域。所有中间出现过的区域都作为候选框输出——小区域对应小物体，大区域对应大物体或场景。

为什么是层次化合并？

每一轮合并产生更大的区域，自然形成从细到粗的区域层次（类似图像金字塔）。这保证了在不同尺度上都能找到包含物体的候选框。

Step 4: 多样化策略 (Diversification)

单一的分割和合并可能遗漏某些物体。Selective Search 通过多种颜色空间 + 随机扰动增加多样性：

策略具体做法目的
多种颜色空间分别在 RGB、HSV、Lab、rgI（归一化 RGB + 亮度）、Hue（仅色调）上独立运行整个算法不同颜色空间对光照、颜色的敏感度不同
多种分割参数Felzenszwalb 算法的参数 \((k, \sigma, \min)\) 取不同组合不同粒度的初始分割
随机扰动对图像做轻微的颜色抖动或裁剪后再运行增加随机性，减少遗漏

多策略合并示意

原图 → 颜色空间变换 (RGB / HSV / Lab / ...)
         ↓
  每种颜色空间独立运行:
    超像素分割 → 层次合并 → 区域集合
         ↓
  所有颜色空间的区域集合取并集
         ↓

输出后处理

  1. 去重：不同颜色空间/参数可能产生重叠度极高的框，用 IoU 阈值去除重复
  2. 面积过滤：剔除面积过小（<50 像素）或过大（接近全图）的区域
  3. 排序：按面积从小到大排列（小物体优先），或按"物体性"(objectness) 排序

最终输出约 2000 个候选框，覆盖图中不同位置、不同尺度的可能物体区域。

Selective Search 的局限性
  • CPU 运行：~2s/图，成为 Fast R-CNN 的速度瓶颈Fast R-CNN 仍然用 **Selective Search** 做候选框生成，这一步在 CPU 上进行，速度慢，且**无法和后面的网络一起训练**（不是可微分的）。
  • 不可学习：固定的图像算法，无法根据检测任务优化
  • 与 CNN 无参数共享：候选框生成和特征提取完全独立
  • 召回率有限：约 2000 个框仍可能遗漏小物体或遮挡物体

→ 这些局限性正是 Faster R-CNN 用 RPN 替代 Selective Search 的动机。

1.4 深入理解 RoI Pooling

RoI Pooling 是干什么的？

候选框映射到特征图后，形状大小各异（因为原图上的候选框有的大有的小）。全连接层要求输入特征尺寸固定，所以需要一种操作把不同大小的特征区域统一成相同尺寸 → RoI Pooling 就是干这个的。

1. **整图特征提取**：将整张输入图像送入 CNN（如 VGG16），得到一张全局特征图 (Feature Map)。 2. **候选框映射**：仍用 Selective Search 在原图提 2000 个候选框，但不再裁剪原图。改为将候选框坐标映射到特征图上，圈出对应区域。 3. **RoI Pooling（核心创新）**：映射回的特征区域大小不一，用 RoI Pooling 将它们统一池化为**固定尺寸**的特征向量（如 7×7）。这样才能送入后续全连接层。 4. **多任务输出**：固定尺寸特征经 FC 层后，分两个并行分支： - **Softmax 分类**（替代 SVM）：输出各类别概率（含背景类） - **Bounding Box Regression**：输出微调后的坐标偏移

关键问题：量化损失（Quantization / Misalignment）

\(h/H\) 或 \(w/W\) 常常不能整除，比如 7×7 的特征区域要池化成 2×2：7/2 = 3.5 → 不是整数！

RoI Pooling 的做法是直接取整（舍掉小数），导致两次取整：

1. **整图特征提取**：将整张输入图像送入 CNN（如 VGG16），得到一张全局特征图 (Feature Map)。 2. **候选框映射**：仍用 Selective Search 在原图提 2000 个候选框，但不再裁剪原图。改为将候选框坐标映射到特征图上，圈出对应区域。 3. **RoI Pooling（核心创新）**：映射回的特征区域大小不一，用 RoI Pooling 将它们统一池化为**固定尺寸**的特征向量（如 7×7）。这样才能送入后续全连接层。 4. **多任务输出**：固定尺寸特征经 FC 层后，分两个并行分支： - **Softmax 分类**（替代 SVM）：输出各类别概率（含背景类） - **Bounding Box Regression**：输出微调后的坐标偏移
  1. 第一次取整：原图坐标映射到特征图时，坐标四舍五入（如 \((10, 10)\) → 映射后 \((3.4, 3.4)\) → 取整为 \((3, 3)\)）
  2. 第二次取整：划分子窗口时，边长取整（3.5 → 3）
→ 两次取整导致像素级别的偏移，特征图上的区域和原图 RoI 不是严格对应的。这就是 不对齐 (Misalignment) 问题。
这在分类任务中影响不大（CNN 有一定平移不变性），但对像素级预测（如语义分割中的 Mask R-CNN）是致命问题。

Fast R-CNN 的瓶颈在哪？

Fast R-CNN 仍然用 Selective Search 做候选框生成，这一步在 CPU 上进行，速度慢（~2s），且无法和后面的网络一起训练（不是可微分的）。CNN 部分从 2000 次降到 1 次（~0.2s GPU），但 Selective Search 成了新瓶颈。

Faster R-CNN 的核心改进：RPN (Region Proposal Network)

  • 第二项：RPN 回归 Loss（只对前景 Anchor 计算，即 \(p_i^*=1\) 时才有效，Smooth L1）RPN 对每个 Anchor 输出两个东西，对应**两个 loss**：

  • \(N_{\text{cls}}\)：归一化项（Anchor 总数或 mini-batch 大小）
  • \(N_{\text{reg}}\)：归一化项（特征图位置数）1. **整图特征提取**：将整张输入图像送入 CNN（如 VGG16），得到一张全局特征图 (Feature Map)。 2. **候选框映射**：仍用 Selective Search 在原图提 2000 个候选框，但不再裁剪原图。改为将候选框坐标映射到特征图上，圈出对应区域。 3. **RoI Pooling（核心创新）**：映射回的特征区域大小不一，用 RoI Pooling 将它们统一池化为**固定尺寸**的特征向量（如 7×7）。这样才能送入后续全连接层。 4. **多任务输出**：固定尺寸特征经 FC 层后，分两个并行分支： - **Softmax 分类**（替代 SVM）：输出各类别概率（含背景类） - **Bounding Box Regression**：输出微调后的坐标偏移
  • \(\lambda\)：平衡权重

1.6 Anchor 机制与边框回归编码

什么是 Anchor（锚框）

Anchor 是 RPN 中预设的一系列不同尺寸和长宽比的参考框。在每个特征图位置，预先放置 k 个 Anchor（通常 k=9：3 种尺寸 × 3 种长宽比）。

``` 输入图像 ↓ 共享 CNN 特征提取 (如 VGG16 / ResNet) ↓ ├──→ **RPN 分支** ├──→ **Fast R-CNN 分支** │ (Region Proposal Network) │ (检测头) │ │ │ 1. 在特征图上滑动一个 3×3 窗口 │ │ 2. 每个窗口位置生成 k 个 Anchor │ │ 3. 判断 Anchor 是前景还是背景 │ │ 4. 回归 Anchor 的坐标偏移 │ │ 5. 输出 ≈ 300 个 Proposal │ │ │ └──→ 输出 Proposals ──────────→ RoI Pooling → 分类 + 回归 ```

RPN 不是"凭空预测框位置"，而是基于 Anchor 做微调（预测 Anchor 与真实框之间的偏移量 \(\Delta x, \Delta y, \Delta w, \Delta h\)），大大降低了学习难度。

RPN 做的事情不是"凭空预测框位置"，而是**基于 Anchor 做微调**（预测 Anchor 与真实框之间的偏移量 Δx, Δy, Δw, Δh），大大降低了学习难度。
3 种尺度 × 3 种长宽比 = 9 个 Anchor
尺度：\(\{128^2, 256^2, 512^2\}\)，长宽比：\(\{1:1, 1:2, 2:1\}\)

边框回归目标编码

网络不是直接预测 \((x, y, w, h)\) 的绝对值，而是预测 Anchor（或 Proposal）到真实框的偏移量。偏移量的数值范围小、变化稳定，比直接回归绝对坐标容易学。

网络不是直接预测 (x, y, w, h) 的绝对值，而是**预测 Anchor（或 Proposal）到真实框的偏移量**。

网络的回归目标不是真实框坐标本身，而是 4 个参数化偏移量 \(t^* = (t_x^*, t_y^*, t_w^*, t_h^*)\)：

1. **候选区域生成 (Region Proposal)**：用 Selective Search 在原图上提取约 **2000 个**可能包含物体的候选框。 2. **尺寸归一化 (Warp/Crop)**：将每个候选框裁剪出来，强制缩放到统一尺寸（如 227×227），以满足 CNN 固定输入要求。 3. **逐一提特征**：把 2000 多张归一化后的小图 **逐一送入 CNN**（AlexNet / VGG）提取特征。 > ⚠ **极度耗时**：一张图跑约 2000 次 CNN 前向传播！ 4. **SVM 分类**：对每个特征向量用 SVM 判断类别（猫/狗/背景）。 5. **Bounding Box Regression**：用线性回归对候选框位置 (x, y, w, h) 做微调，让框更贴合物体。
$$t_x^* = \frac{x^* - x_a}{w_a}, \quad t_y^* = \frac{y^* - y_a}{h_a}$$ $$t_w^* = \log\frac{w^*}{w_a}, \quad t_h^* = \log\frac{h^*}{h_a}$$

其中 \((x^*, y^*, w^*, h^*)\) 是真实框 (Ground Truth)，\((x_a, y_a, w_a, h_a)\) 是 Anchor（RPN 中）/ Proposal（检测头中）。

其中： - (x*, y*, w*, h*) — 真实框 (Ground Truth) - (x_a, y_a, w_a, h_a) — Anchor（RPN 中）/ Proposal（检测头中）

为什么要除 \(w_a / h_a\)？为什么要取 \(\log\)？

操作原因
除以 \(w_a, h_a\) 尺度归一化。大框和小框的偏移量不在一个量级：比如 Anchor 100×100 偏移 20，和 Anchor 1000×1000 偏移 20，绝对值相同但相对意义完全不同。除以 Anchor 的宽高后变成"相对偏移"，对所有尺度公平
取 \(\log(w^*/w_a)\) 框的宽高只能是正数，但网络预测值可正可负。取 \(\log\) 后把 \((0, +\infty)\) 映射到 \((-\infty, +\infty)\)，网络不需要加额外约束。且 \(\log\) 放缩使得"变大/变小"对称

网络实际预测的是什么？

网络回归分支输出 4 个值 \((t_x, t_y, t_w, t_h)\)，代表预测的偏移量。回归 Loss 就是 Smooth L1 损失：

1. **候选区域生成 (Region Proposal)**：用 Selective Search 在原图上提取约 **2000 个**可能包含物体的候选框。 2. **尺寸归一化 (Warp/Crop)**：将每个候选框裁剪出来，强制缩放到统一尺寸（如 227×227），以满足 CNN 固定输入要求。 3. **逐一提特征**：把 2000 多张归一化后的小图 **逐一送入 CNN**（AlexNet / VGG）提取特征。 > ⚠ **极度耗时**：一张图跑约 2000 次 CNN 前向传播！ 4. **SVM 分类**：对每个特征向量用 SVM 判断类别（猫/狗/背景）。 5. **Bounding Box Regression**：用线性回归对候选框位置 (x, y, w, h) 做微调，让框更贴合物体。
$$L_{\text{reg}} = \text{SmoothL1}(t - t^*) = \text{SmoothL1}(\text{预测偏移量} - \text{真实偏移量})$$

测试时：解码（把偏移量变回真实坐标）

测试时没有 Ground Truth，只能用预测的 \(t\) 反解出预测框坐标：

🎯 回归框的目标（Ground Truth）到底怎么构造？
$$x = x_a + t_x \cdot w_a, \quad y = y_a + t_y \cdot h_a$$ $$w = w_a \cdot e^{t_w}, \quad h = h_a \cdot e^{t_h}$$
例子：Anchor 是 100×100，GT 是 120×140
\(t_x^* = (120-100)/100 = 0.2, \quad t_y^* = (140-100)/100 = 0.4\)
\(t_w^* = \log(120/100) \approx 0.18, \quad t_h^* = \log(140/100) \approx 0.34\)
网络学会预测 \((0.2, 0.4, 0.18, 0.34)\) → 解码回 \((120, 140)\)。

1.8 RoI Pooling 池化 vs RoI Align

RoI Pooling 的量化问题

候选框映射到特征图后，需要进行两次取整（量化）：

  1. 原图坐标映射到特征图时，坐标四舍五入（如 \((10,10)\) → \((3.4,3.4)\) → \((3,3)\)）
  2. 划分子窗口时，边长取整（如 7/2 = 3.5 → 3）

→ 两次取整导致像素级偏移（Misalignment），特征图上的区域和原图 RoI 不是严格对应的。

→ **两次取整导致像素级别的偏移**，特征图上的区域和原图 RoI 不是严格对应的。这就是 **不对齐 (Misalignment)** 问题。

RoI Align — 彻底取消取整

RoI Align 来自 Mask R-CNN，核心改进：

RoI Align（对齐）— 来自 Mask R-CNN
  • 不取整：坐标保持浮点数
  • 均匀采样：每个子区域均匀采样 4 个点（2×2）
  • 双线性插值：每个采样点的值用周围 4 个像素加权平均得到
  • Pooling：对 4 个采样点做 Max/Average Pooling**做法**： 1. 不取整：坐标保持浮点数 2. 每个子区域均匀采样 4 个点（2×2） 3. 每个点的值用**周围相邻 4 个像素做双线性插值**得到 4. 对 4 个采样点做 Max Pooling / Average Pooling
    • **做法**： 1. 不取整：坐标保持浮点数 2. 每个子区域均匀采样 4 个点（2×2） 3. 每个点的值用**周围相邻 4 个像素做双线性插值**得到 4. 对 4 个采样点做 Max Pooling / Average Pooling

为什么 w 和 h 要开根号？
如果不加根号，大框和小框的相同像素偏移会产生相同的损失。例如 100×100 的框偏差 10 像素，和 10×10 的框偏差 10 像素，绝对误差一样但相对误差差别巨大。开根号后，小框的误差会被放大，使模型更关注小物体的定位精度。

2. 图像分割概述 (Image Segmentation)

图像分割的本质是像素级的分类：给图像中的每一个像素赋予一个类别标签。与分类（H×W → 1×1）和检测（H×W → 框+类别）不同，分割要求输出与输入相同的空间分辨率（H×W → H×W），不能丢失空间信息。

1. **候选区域生成 (Region Proposal)**：用 Selective Search 在原图上提取约 **2000 个**可能包含物体的候选框。 2. **尺寸归一化 (Warp/Crop)**：将每个候选框裁剪出来，强制缩放到统一尺寸（如 227×227），以满足 CNN 固定输入要求。 3. **逐一提特征**：把 2000 多张归一化后的小图 **逐一送入 CNN**（AlexNet / VGG）提取特征。 > ⚠ **极度耗时**：一张图跑约 2000 次 CNN 前向传播！ 4. **SVM 分类**：对每个特征向量用 SVM 判断类别（猫/狗/背景）。 5. **Bounding Box Regression**：用线性回归对候选框位置 (x, y, w, h) 做微调，让框更贴合物体。
核心挑战：分类网络通过池化层不断下采样，丢失了空间细节；分割网络需要上采样把缩小的特征图恢复到原图尺寸，同时保住边缘细节。

分类 vs 检测 vs 分割

分类:     [猫猫图片] → Conv → Pool → Conv → Pool → FC → [猫]          (1×1)
检测:     [猫猫图片] → Conv → Pool → ... → RPN → [框坐标 + 类别]      (去掉空间)
分割:     [猫猫图片] → Conv → ... → Conv → 上采样 → [像素级标签图]    (H×W)

2.1 什么是 Mask（掩码）

Mask（掩码/掩膜）就是一张二值图，和原图大小一样，每个像素的值是 0 或 1：

原图 (H×W×3)                  Mask (H×W, 二值)
┌──────────────────┐          ┌──────────────────┐
│  ████    ████    │          │  0000    1111    │
│  ████    ████    │   →     │  0000    1111    │
│          ████    │          │         1111    │
└──────────────────┘          └──────────────────┘
  彩色图片                       1=物体, 0=背景
  • 1（白色） = 这个像素属于目标物体
  • 0（黑色） = 这个像素是背景

就像用一张镂空纸板（模板）盖在图上，只露出你要的那个物体。实例分割里每个框对应一个 mask，语义分割里每个类别对应一整片 mask。

``` 原图 (H×W×3) Mask (H×W, 二值) ┌──────────────────┐ ┌──────────────────┐ │ ████ ████ │ │ 0000 1111 │ │ ████ ████ │ → │ 0000 1111 │ │ ████ │ │ 1111 │ └──────────────────┘ └──────────────────┘ 彩色图片 1=物体, 0=背景 ```

Mask R-CNN 在 Faster R-CNN 基础上做了两处关键改动：

Mask R-CNN 比 Faster R-CNN 多做了什么？

Faster R-CNN:  RPN + RoI Pooling + 分类分支 + 回归分支

改动 1：RoI Pooling 池化 → RoI Align
