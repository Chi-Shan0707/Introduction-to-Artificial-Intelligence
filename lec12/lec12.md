# 目标检测：Two-stage 方法 (R-CNN 系列)

## Two-stage 核心思想
先找 Bounding Box，再做分类 → **先粗后细**

与之相对的是 **One-stage**（如 YOLO、SSD）：同时输出 Box 和标签，一步到位。

---

## 1. R-CNN (Regions with CNN features)

最早将 CNN 引入目标检测的经典 two-stage 方法。

### 流程

1. **候选区域生成 (Region Proposal)**：用 Selective Search 在原图上提取约 **2000 个**可能包含物体的候选框。
2. **尺寸归一化 (Warp/Crop)**：将每个候选框裁剪出来，强制缩放到统一尺寸（如 227×227），以满足 CNN 固定输入要求。
3. **逐一提特征**：把 2000 多张归一化后的小图 **逐一送入 CNN**（AlexNet / VGG）提取特征。
   > ⚠ **极度耗时**：一张图跑约 2000 次 CNN 前向传播！
4. **SVM 分类**：对每个特征向量用 SVM 判断类别（猫/狗/背景）。
5. **Bounding Box Regression**：用线性回归对候选框位置 (x, y, w, h) 做微调，让框更贴合物体。

### 缺点
- **速度极慢**：2000 次 CNN 前向传播，无法实时
- **训练割裂**：CNN 提特征、SVM 分类、回归器微调是分开训练的，不能端到端
- **存储量大**：需要保存所有候选框的特征到磁盘

---

## 2. Fast R-CNN

### 核心改进：全图共享卷积 → 避免重复计算

R-CNN 对每个候选框都跑一遍 CNN，极度冗余。Fast R-CNN 让**整张图只过一次 CNN**，所有候选框共享特征图。

### 流程

1. **整图特征提取**：将整张输入图像送入 CNN（如 VGG16），得到一张全局特征图 (Feature Map)。
2. **候选框映射**：仍用 Selective Search 在原图提 2000 个候选框，但不再裁剪原图。改为将候选框坐标映射到特征图上，圈出对应区域。
3. **RoI Pooling（核心创新）**：映射回的特征区域大小不一，用 RoI Pooling 将它们统一池化为**固定尺寸**的特征向量（如 7×7）。这样才能送入后续全连接层。
4. **多任务输出**：固定尺寸特征经 FC 层后，分两个并行分支：
   - **Softmax 分类**（替代 SVM）：输出各类别概率（含背景类）
   - **Bounding Box Regression**：输出微调后的坐标偏移

### 相比 R-CNN 的优势
| | R-CNN | Fast R-CNN |
|---|---|---|
| 特征提取 | 2000 次/图 | 1 次/图 |
| 训练方式 | 分步独立 | 端到端（分类+回归联合训练）|
| 速度 | 极慢 | 快 ~10x |
| 存储 | 需保存特征到磁盘 | 不需要 |

---

## 3. 深入理解 RoI Pooling

### 它是干什么的？

候选框映射到特征图后，形状大小各异（因为原图上的候选框有的大有的小）。全连接层要求输入特征尺寸固定，所以需要一种操作把**不同大小的特征区域统一成相同尺寸** → RoI Pooling 就是干这个的。

### 怎么做？（分两步）

假设有一个 8×8 的特征图区域，要池化成 2×2 的输出：

1. **划分网格**：将 8×8 区域划分为 2×2 = 4 个子区域，每个子区域大小 = 8/2 = 4 → 每个子区域是 4×4。
2. **每个子区域做 Max Pooling**：在每个 4×4 格子里取最大值，得到 2×2 的输出。

一般情况：一个 h×w 的 RoI 区域，要池化成 H×W 的固定输出：
- 每个子窗口大小 ≈ h/H × w/W（通常不整除，下面会讲问题）

### ⚠ 关键问题：量化损失（Quantization / Misalignment）

h/H 或 w/W 常常**不能整除**，比如 7×7 的特征区域要池化成 2×2：
- 7/2 = 3.5 → 不是整数！

**RoI Pooling 的做法**：直接取整（舍掉小数）
- 第一次取整：把候选框坐标从原图映射到特征图时，坐标四舍五入（如 (10, 10) → 映射后可能是 (3.4, 3.4) → 取整为 (3, 3)）
- 第二次取整：划分子窗口时，边长取整（3.5 → 3）

→ **两次取整导致像素级别的偏移**，特征图上的区域和原图 RoI 不是严格对应的。这就是 **不对齐 (Misalignment)** 问题。

这在分类任务中影响不大（分类需要平移不变性），但对**像素级预测**（如语义分割中的 Mask R-CNN）是致命问题。

## 4. Faster R-CNN：真正的端到端

### Fast R-CNN 的瓶颈在哪？

Fast R-CNN 仍然用 **Selective Search** 做候选框生成，这一步在 CPU 上进行，速度慢，且**无法和后面的网络一起训练**（不是可微分的）。

### Faster R-CNN 的核心改进：RPN (Region Proposal Network)

用一个小网络 **RPN** 替代 Selective Search，直接在特征图上生成候选框，**让候选框生成也变成神经网络的一部分**，和检测网络共享卷积层。

### 整体架构（Two-stage 完全体）

```
输入图像
    ↓
共享 CNN 特征提取 (如 VGG16 / ResNet)
    ↓
    ├──→ **RPN 分支**                ├──→ **Fast R-CNN 分支**
    │    (Region Proposal Network)   │    (检测头)
    │                                │
    │ 1. 在特征图上滑动一个 3×3 窗口   │
    │ 2. 每个窗口位置生成 k 个 Anchor  │
    │ 3. 判断 Anchor 是前景还是背景    │
    │ 4. 回归 Anchor 的坐标偏移       │
    │ 5. 输出 ≈ 300 个 Proposal       │
    │                                │
    └──→ 输出 Proposals ──────────→ RoI Pooling → 分类 + 回归
```

### 什么是 Anchor（锚框）？

Anchor 是 RPN 中**预设的一系列不同尺寸和长宽比的参考框**。

在每个特征图位置，预先放置 k 个 Anchor（比如 k=9：3 种尺寸 × 3 种长宽比）。

RPN 做的事情不是"凭空预测框位置"，而是**基于 Anchor 做微调**（预测 Anchor 与真实框之间的偏移量 Δx, Δy, Δw, Δh），大大降低了学习难度。

### RPN 的训练（两个 Loss）

RPN 对每个 Anchor 输出两个东西，对应**两个 loss**：

1. **分类 Loss (前景/背景二分类)**：判断 Anchor 是前景（有物体）还是背景。用 Binary Cross Entropy。
2. **回归 Loss (坐标偏移)**：对前景 Anchor 进一步预测偏移，使其逼近真实框。用 Smooth L1 Loss。

### 总的 Loss 形式

```
L({p_i}, {t_i}) =  1/N_cls · Σ L_cls(p_i, p_i*)   +   λ · 1/N_reg · Σ p_i* · L_reg(t_i, t_i*)
                  └────── RPN 分类 Loss ──────┘     └─────── RPN 回归 Loss (只对前景计算) ──────┘
```

---

## 🎯 回归框的目标（Ground Truth）到底怎么构造？

网络不是直接预测 (x, y, w, h) 的绝对值，而是**预测 Anchor（或 Proposal）到真实框的偏移量**。

这样做的好处：偏移量的数值范围小、变化稳定，比直接回归绝对坐标容易学。

### 目标值编码公式

网络的回归目标不是真实框坐标本身，而是**4 个参数化偏移量** t* = (t_x*, t_y*, t_w*, t_h*)：

```
t_x* = (x* - x_a) / w_a          ← 中心点 x 的偏移（除以宽做归一化）
t_y* = (y* - y_a) / h_a          ← 中心点 y 的偏移（除以高做归一化）
t_w* = log(w* / w_a)              ← 宽的对数缩放
t_h* = log(h* / h_a)              ← 高的对数缩放
```

其中：
- (x*, y*, w*, h*) — 真实框 (Ground Truth)
- (x_a, y_a, w_a, h_a) — Anchor（RPN 中）/ Proposal（检测头中）

### 为什么要除 w_a / h_a？为什么要取 log？

| 操作 | 原因 |
|---|---|
| 除以 w_a, h_a | **尺度归一化**。大框和小框的偏移量不在一个量级，除以 Anchor 的宽高后变成"相对偏移"，对所有尺度公平 |
| log(w*/w_a) | 框的宽高只能是 **正数**，但网络预测值可正可负。取 log 后把 (0, +∞) 映射到 (-∞, +∞)，网络不需要加额外约束。且 log 放缩使得"变大/变小"对称 |

### 网络实际预测的是什么？

网络回归分支输出 4 个值 (t_x, t_y, t_w, t_h)，代表**预测的偏移量**。

回归 Loss 就是 Smooth L1 损失：
```
L_reg = SmoothL1(t - t*)
      = SmoothL1(预测偏移量 - 真实偏移量)
```

### 测试时：解码（把偏移量变回真实坐标）

测试时没有 Ground Truth，只能用预测的 t 反解出预测框坐标：

```
x = x_a + t_x · w_a
y = y_a + t_y · h_a
w = w_a · exp(t_w)
h = h_a · exp(t_h)
```

### RPN vs 检测头：回归目标本质上一样，但 Anchor 不同

| | RPN 回归 | 检测头回归 |
|---|---|---|
| 基准框 | Anchor（预设的 k 个框） | RPN 输出的 Proposal |
| 目标值 | Anchor → Ground Truth 的偏移 | Proposal → Ground Truth 的偏移 |
| 输出 | 粗糙的 Proposal 位置 | **最终的精确框** |
| 回归公式 | 完全一样 | 完全一样 |

**例子**：
- Anchor 是 100×100，GT 是 120×140
- t_x* = (120-100)/100 = 0.2, t_y* = (140-100)/100 = 0.4
- t_w* = log(120/100) ≈ 0.18, t_h* = log(140/100) ≈ 0.34
- 网络学会预测 (0.2, 0.4, 0.18, 0.34) → 解码回 (120, 140)

---

### ❓ 你问：Proposal 算的 Loss 和最后得到框的 Loss 是分开的吗？

**是分开的，但联合优化。** Faster R-CNN 的 Loss 有两大部分：

```
总 Loss = L_RPN + L_Fast_RCNN
```

| 组件 | Loss 来源 | 作用 |
|---|---|---|
| **L_RPN** | RPN 分支 | 训练 RPN 生成好的 Proposal（前景/背景分类 + 坐标回归）|
| **L_Fast_RCNN** | 检测头分支 | 训练检测网络对 Proposal 做精确分类 + 最终框回归 |

- **RPN Loss** 负责：学会"提好的候选框"
- **Fast R-CNN Loss** 负责：学会"给候选框做精确分类和微调"

它们**是两个独立的 Loss 函数**，计算方式不同（RPN 用二分类 + SmoothL1，检测头用多分类 Softmax + SmoothL1），但在训练时**相加作为总 Loss 一起反向传播**，共享的卷积层同时接收来自两个分支的梯度。

> 训练细节：通常用 **Alternating Training** 或者 **Approximate Joint Training** 来让两个分支协同工作。不完全是"先训 RPN 再训 Fast"，而是交替/联合优化。

---

### 推理时：Proposal → NMS → 最终框

测试时流程：
1. RPN 生成很多 Proposal（约 20000 个 Anchor → NMS 筛选到约 2000 或 300 个）
2. 这些 Proposal 经 RoI Pooling 送入检测头
3. 检测头输出分类概率 + 框偏移
4. **最终对检测头输出再做一次 NMS**，得到最终框

所以虽然 RPN 也会输出框（前景 Anchor 的位置），但这些框是粗糙的。**最终输出的检测框来自检测头（Fast R-CNN 分支），而不是 RPN**。

## 5. RoI Pooling vs RoI Align：对齐 vs 不对齐

### RoI Pooling（不对齐）

两次取整操作导致**量化偏差**：

```
原图坐标 (10, 10) → 映射到特征图 (3.4, 3.4) → 取整 (3, 3)  ← 第一次量化
划分网格：7/2 = 3.5 → 取整为 3                          ← 第二次量化
```

最终得到的特征区域和原始 RoI **有 1~2 个像素的偏移**。这在分类任务中影响不大（CNN 有一定平移不变性），但分割任务无法容忍。

### RoI Align（对齐）— 来自 Mask R-CNN

**彻底取消取整操作**，用**双线性插值**保持浮点数精度：

**做法**：
1. 不取整：坐标保持浮点数
2. 每个子区域均匀采样 4 个点（2×2）
3. 每个点的值用**周围相邻 4 个像素做双线性插值**得到
4. 对 4 个采样点做 Max Pooling / Average Pooling

```
子区域 (浮点边界)        采样 4 个点         每个点双线性插值         Pooling
┌──────────┐           ·       ·            查周围 4 个像素
│          │           ·       ·           加权平均算出该点值      →   输出
└──────────┘
```

### 对比

| | RoI Pooling | RoI Align |
|---|---|---|
| 坐标处理 | 取整量化 | 保持浮点数 |
| 采样方式 | 直接划分格子取最大值 | 子区域均匀采样 N 个点，插值 |
| 对齐性 | ❌ 不对齐（Misalignment） | ✅ 完全对齐 |
| 分类任务 | 够用 | 也可用（但没必要）|
| 分割任务 | ❌ 效果差 | ✅ 必要 |
| 速度 | 快 | 稍慢（插值开销）|

**一句话**：RoI Align 就是"不做量化的 RoI Pooling"，用浮点数 + 插值保住空间对应关系。

## 6. Region Proposal 在三者中的演变（结合lec12 PDF）

### 总览对比

| | R-CNN | Fast R-CNN | Faster R-CNN |
|---|---|---|---|
| **Proposal 方法** | Selective Search | Selective Search | **RPN (Region Proposal Network)** |
| **运行设备** | CPU | CPU | **GPU** |
| **是否可学习** | ❌ 固定算法 | ❌ 固定算法 | **✅ 可微分，端到端学习** |
| **是否共享特征** | ❌ 不共享 | ✅ 共享检测网络特征 | **✅ RPN 与检测网络共享特征** |
| **速度瓶颈** | 2000 次 CNN 前向 | **Selective Search (CPU)** | 几乎无瓶颈 |
| **每张图耗时** | ~47s (GPU) | ~0.32s (GPU) | **~0.2s (GPU)** |

### 6.1 R-CNN：Selective Search（外部算法，完全独立）

PDF 第 7-15 页：Selective Search 是**传统图像算法**（基于颜色、纹理、大小相似度合并超像素），在 **CPU** 上运行，与 CNN **没有任何参数共享**。

```
原图 → Selective Search (CPU) → ~2000 个候选框
         ↓
    逐个裁剪 → 逐个 CNN → SVM + 回归
```

**问题**：
- 2000 个框 → 2000 次 CNN，计算极度冗余
- 无法针对检测任务优化
- CPU 速度慢

### 6.2 Fast R-CNN：仍然是 Selective Search，但作用变了

PDF 第 16-18 页：仍然用 Selective Search 生成 2000 个候选框，但**不再裁剪原图，改为映射到特征图**：

```
原图候选框坐标 (x1, y1, x2, y2)
       ↓ 除以 stride (如 16)
特征图映射坐标 (x1/16, y1/16, x2/16, y2/16)   ← 第一次量化取整
       ↓
RoI Pooling 划分子网格                        ← 第二次量化取整
       ↓
固定尺寸特征 → 分类 + 回归
```

**Selective Search 成了新瓶颈** — PDF 第 29 页专门提问：

> "Fast R-CNN 过程中比较耗费时间的是哪个过程呢？"

答案：CNN 部分从 2000 次降到 1 次（~0.2s GPU），但 **Selective Search 在 CPU 上仍需 ~2s**。Region Proposal 这一步卡住了整体速度。

**关键矛盾**：
- 检测部分已可端到端训练
- 但候选框生成仍是**外部不可微的固定算法**
- Selective Search 无法与检测网络协同优化

### 6.3 Faster R-CNN：RPN 替换 Selective Search（革命性变化）

PDF 第 30 页原文：

> "用区域提议网络（Region Proposal Network, RPN）来从特征中预测候选框"

**RPN 的核心设计**：

```
共享 CNN 特征图 (如 512×20×15)
         ↓
   3×3 滑动卷积
         ↓
    ┌────┴────┐
    ↓          ↓
 分类分支     回归分支
 (2k 通道)    (4k 通道)
    ↓          ↓
 前景/背景概率  偏移量 (tx, ty, tw, th)
```

- **Anchor 机制**（PDF 第 32-35 页）：每个位置预设 k 个不同尺寸/长宽比的参考框（通常 k=9：3 scale × 3 ratio）
- RPN 是"Anchor 的二分类 + 回归微调"，不是"从零预测"
- 共享卷积特征 → **零额外计算成本**
- 全卷积 → **GPU 运行**

**为什么 Faster R-CNN 比 Fast R-CNN 快？**（PDF 第 37 页）：
1. Selective Search (CPU, ~2s) → RPN (GPU, ~0.01s)
2. RPN 与检测网络**共享卷积层**，特征图只需算一次

### 6.4 一张图看懂本质变化

```
R-CNN:
                       ┌─→ Warp → CNN → SVM + Reg (第 1 框)
原图 → Selective Search ──→ Warp → CNN → SVM + Reg (第 2 框)    ← 2000 次!
  (CPU, 固定算法)        └─→ ...                               ← 不共享

Fast R-CNN:
                        ┌─ RoI Pooling → FC → Softmax + Reg
原图 → CNN(一次) ──→ 特征图 ─┼─ RoI Pooling → FC → Softmax + Reg   ← 共享特征
   ↑              (GPU)     └─ ...                                ← 但 Proposal 还在 CPU
   └── Selective Search (CPU 瓶颈!)

Faster R-CNN:
                        ┌─ RPN ──→ Proposals
原图 → CNN(一次) ──→ 特征图 ─┼─ RoI Pooling → FC → Softmax + Reg   ← 全部 GPU!
                        └─ ...
                 ↑
           RPN 与检测头共享特征  ←  关键
```

### 6.5 Selective Search vs RPN：本质对比

| 维度 | Selective Search | RPN |
|---|---|---|
| **本质** | 图像分割+合并的规则算法 | 卷积神经网络 |
| **输入** | 原始图像像素 | 特征图（高层语义） |
| **输出** | 约 2000 个候选框坐标 | 约 20000 Anchor 前景概率+NMS→~300 |
| **可学习** | ❌ 固定不变 | ✅ 随检测任务训练 |
| **运行位置** | CPU | GPU |
| **耗时** | ~2s | ~0.01s（可忽略）|
| **是否端到端** | ❌ | ✅ |
| **对检测的贡献** | 提供"可能"有物体的位置 | 学会"什么样的框是好框" |

### 6.6 一句话总结

> **R-CNN 用 Selective Search 在外面提框；Fast R-CNN 共享了特征图但没共享参数，Selective Search 成了 CPU 瓶颈；Faster R-CNN 用 RPN 把提框变成一个可学习的网络，彻底端到端。**

## 7. R-CNN 系列进化总结

```
R-CNN          →  Selective Search + 2000次CNN + SVM + 回归
                    ↓ 去掉重复计算
Fast R-CNN     →  Selective Search + 1次CNN + RoI Pooling + 联合训练
                    ↓ 把 Proposal 也变成网络
Faster R-CNN   →  RPN（Anchor机制）+ 1次CNN + RoI Pooling + 端到端
                    ↓ 解决量化不对齐
Mask R-CNN     →  RPN + RoI Align + 多任务（分类+框+掩码）
```

| 模型 | 候选框生成 | 是否端到端 | 速度 | 特点 |
|---|---|---|---|---|
| R-CNN | Selective Search (CPU) | ❌ | 极慢 | 开山之作 |
| Fast R-CNN | Selective Search (CPU) | ✅ 检测部分 | 快 | RoI Pooling |
| Faster R-CNN | RPN (GPU, 可学习) | ✅ 全部 | 更快 | Anchor 机制 |
| Mask R-CNN | RPN (GPU) | ✅ 全部 | 更快 | RoI Align + 分割 |

# 图像分割

## 0. 到底什么是 Mask？

**Mask（掩码/掩膜）** 就是一张**二值图**，和原图大小一样，每个像素的值是 0 或 1：

```
原图 (H×W×3)                  Mask (H×W, 二值)
┌──────────────────┐          ┌──────────────────┐
│  ████    ████    │          │  0000    1111    │
│  ████    ████    │   →     │  0000    1111    │
│          ████    │          │         1111    │
└──────────────────┘          └──────────────────┘
  彩色图片                       1=物体, 0=背景
```

- **1（白色）** = 这个像素属于目标物体
- **0（黑色）** = 这个像素是背景

就像用一张镂空纸板（模板）盖在图上，只露出你要的那个物体。实例分割里每个框对应一个 mask，语义分割里每个类别对应一整片 mask。

---

## 1. 三种分割任务的定义

### 核心问题：这个物体"可数"吗？

```
路、天空、建筑群       vs        人、车、猫
  不可数 (stuff)                 可数 (things)
连成一片，不分个体              每个个体都要区分
```

但这个界限是**人为定义的**，不是绝对的：

> 石墩子 → 看你 dataset 怎么定。你不标它，它就是背景；你把它标成"石墩子"类，告诉标人员每个石墩子都框出来，它就变成了可数的 things。
> 窗户 → 你喜欢，就可以每个窗户都标出来让模型区分。
> 路 → 一般认为是不可数的 stuff，但在卫星图上"邯郸路""松湖路"可能每条路都要分开。

### 三种分割类型

| 类型 | 定义 | 可数/不可数 | 输出 |
|---|---|---|---|
| **语义分割 (Semantic Segmentation)** | 给**每个像素**打类别标签，同类物体**不区分个体** | 只管类别，不管个数 | 每个像素一个类别 ID |
| **实例分割 (Instance Segmentation)** | 检测每个物体个体，给每个物体生成一个 mask | 只处理可数的 things | 框 + 每个物体的 binary mask |
| **全景分割 (Panoptic Segmentation)** | 语义+实例合并：背景做语义，前景做实例 | stuff + things 全包 | 每个像素有标签+实例ID |

```
                        ┌───────┬───────┐
                        │  Cat  │ Cat   │   语义分割：只看类别
                        │       │       │   两只猫颜色一样
      ┌───────────┐     │       │       │
      │           │     ├───────┤       │   实例分割：Cat1, Cat2 分开
      │  Person   │     │  Car  │       │   每只猫有自己的 mask
      │           │     │       │       │
      └───────────┘     └───────┴───────┘   全景分割：全部都有
```

### 分割的难点：输入 H×W，输出也得是 H×W

分类（Image Classification）：**H×W → 1×1**，一路下采样到 1 个值就行。

检测（Object Detection）：**H×W → 框坐标+类别**，也不需要保持空间尺寸。

分割（Segmentation）：**H×W → H×W**，输入多大输出就得多大，不能把空间信息丢掉。

```
分类:     [猫猫图片] → Conv → Pool → Conv → Pool → FC → [猫]          (1×1)
检测:     [猫猫图片] → Conv → Pool → ... → RPN → [框坐标 + 类别]      (去掉空间)
分割:     [猫猫图片] → Conv → ... → Conv → 上采样 → [像素级标签图]    (H×W)
```

这就是为什么不能直接用分类网络做分割 — 需要**上采样**把缩小的特征图恢复到原图尺寸，还要保住边缘细节。

---

## 2. FCN (Fully Convolutional Network) — 语义分割的开山之作

## 2. FCN (Fully Convolutional Network) — 语义分割的开山之作

### 核心思想：把分类网络改成像素级预测

传统 CNN 最后是**全连接层 → 输出类别概率**，丢掉了空间信息。
FCN 把全连接层全部换成**卷积层**，输出不再是 1×1 的类别，而是 **H×W 的空间图**。

```
分类网络：  Conv → Conv → FC → FC → 类别 (1×1)
FCN：       Conv → Conv → Conv → Conv → 分割图 (H×W)
```

### 具体做法

1. **替换 FC 为 Conv**：VGG16 最后的 FC 层换成 1×1 卷积
2. **上采样（反卷积/转置卷积）**：特征图经过多次池化缩小了 32 倍，用转置卷积上采样回原图尺寸
3. **跳跃连接 (Skip Connection)**：融合浅层高分辨率特征和深层语义特征

### 为什么需要跳跃连接？

```
上采样 32 倍（FCN-32s）：特征太小，上采样后细节丢失严重 → 分割粗糙
                ↓
上采样 16 倍 + 融合 pool4 特征（FCN-16s）：加入中层细节 → 边缘改善
                ↓
上采样 8 倍 + 融合 pool3 特征（FCN-8s）：加入浅层细节 → 最好
```

FCN-32s 只是把最后一层直接放大 32 倍，结果很模糊。
FCN-8s 融合了浅层的精细特征，边缘更清晰。

## 3. U-Net — 医学图像分割的标杆

### 整体结构：对称的 U 形

```
编码器 (下采样)                          解码器 (上采样)
     ↓                                    ↑
输入 → Conv → Pool → Conv → Pool → ... → UpConv → Conv → UpConv → Conv → 输出
             |                    |                    ↑            ↑
             └──→ 跳跃连接 ────────┴────────────────────┘            |
             |                                         └──→ 跳跃连接 ─┘
```

### 详细结构图

```
输入图像
    │
┌───┴───┐
│ Conv  │  3×3, ReLU      ← 每个 stage 两个卷积
│ Conv  │
└───┬───┘
    │                          ┌──────────────┐
    │  2×2 Max Pooling         │  Up-Conv 2×2 │
    ▼                          │  + Conv 3×3  │
┌───┴───┐    跳跃连接 ────────→│  + Conv 3×3  │
│ Conv  │  (crop + concat)    └──────┬───────┘
│ Conv  │                           │
└───┬───┘                           │
    │                                │
    │  2×2 Max Pooling              │
    ▼                                │
┌───┴───┐    跳跃连接 ────────→      │
│ Conv  │                           │
│ Conv  │                           │
└───┬───┘                           │
    │                                │
   ...  (重复 4 次)                 │
    │                                │
    ▼                                ▼
  最底层 (bottleneck) ──────→      上采样 + 跳跃连接 → ... → 1×1 Conv → 输出分割图
```

### 核心设计：跳跃连接

U-Net 最关键的创新是 **编码器的每一层都通过跳跃连接直接拼接到对应解码器层**。

```
编码器第 i 层特征 (高分辨率, 浅层语义)    解码器第 i 层 (上采样后)
            │                                      │
            └───────────── 拼接 (concat) ──────────┘
                                       │
                                    Conv 3×3 融合
                                       │
                                       ▼
```

**为什么需要跳跃连接？**

| 问题 | 解决方式 |
|---|---|
| 下采样丢失空间细节（边缘、位置） | 跳跃连接把高分辨率的浅层特征直接送到解码器 |
| 上采样无法恢复细节 | 浅层特征提供像素级位置信息，深层特征提供语义 |
| 医学图像边缘重要（器官边界） | 跳跃连接保留边界细节 |

### U-Net vs FCN 的跳跃连接

| | FCN | U-Net |
|---|---|---|
| 连接方式 | **逐元素相加 (sum)** | **通道拼接 (concat)** |
| 连接数量 | 仅在最后几层 | **每一层都有** |
| 特征图大小 | 需一致才能相加 | 拼接后通道翻倍，信息更丰富 |
| 效果 | 改善边缘 | **极大提升定位精度** |

### U-Net 为什么在医学图像上表现好？

1. **数据量小**：医学图像数据集通常很小，U-Net 的对称结构+跳跃连接使得参数效率高
2. **边缘精细**：跳跃连接保住了浅层的高分辨率信息，分割边缘准确
3. **位置精确**：医学图像中器官位置相对固定，利用浅层定位信息效果好

## 4. Mask R-CNN — 实例分割

### Mask R-CNN 比 Faster R-CNN 多做了什么？

```
Faster R-CNN:  RPN + RoI Pooling + 分类分支 + 回归分支
                                             ↓
Mask R-CNN:    RPN + RoI Align   + 分类分支 + 回归分支 + Mask 分支
                      ↑                           ↑
                  改动 1                          改动 2
```

### 改动 1：RoI Pooling → RoI Align

PDF lec12 第 25-28 页详解了 RoI Align。实例分割要求像素级精度，RoI Pooling 的两次取整导致的**不对齐**问题对分割是致命的。

**RoI Align 彻底取消取整**，用**双线性插值**：

```
子区域 (浮点边界)
┌──────────┐    每个子区域均匀采样 4 个点
│  ·    ·  │   每个点用周围 4 个像素双线性插值算出
│  ·    ·  │   最后对这 4 个点做 Max Pooling
└──────────┘

→ 不再有量化损失，特征图和原图严格对齐
```

### 改动 2：新增 Mask 分支

在分类和回归之外，**额外增加一个并行的 Mask 分支**：

```
RoI Align 输出的固定尺寸特征 (如 14×14)
         │
    ┌────┴────┐
    │         │
    │         │
 ┌──┴──┐  ┌──┴──┐  ┌──────────┐
 │ FC  │  │ FC  │  │  Conv    │  ← 全卷积网络 (FCN)
 │ cls │  │ reg │  │  14×14   │
 │     │  │     │  │    ↓     │
 │     │  │     │  │  Conv    │
 │     │  │     │  │    ↓     │
 └──┬──┘  └──┬──┘  │  Conv    │
    │        │     │    ↓     │
 类别概率   框坐标  │  28×28   │
                   │  mask    │
                   └──────────┘
```

Mask 分支的特点：

1. **对每个类别独立预测 mask**：输出维度是 **K × 28 × 28**（K 是类别数），第 k 个通道是第 k 类的 mask
2. **分类和 mask 解耦**：训练时，Loss 只计算**该 RoI 真实类别对应的那个 mask 通道**的 loss，其他通道不计算
3. **用 FCN 做 mask 预测**：保持空间结构，没有全连接层

### Mask R-CNN 的 Loss

```
L = L_cls + L_box + L_mask
    ↑        ↑       ↑
  分类 Loss  回归 Loss  二值交叉熵 (每个像素 sigmoid)
  (Softmax)  (SmoothL1)  (只在真实类别对应的 mask 通道上算)
```

### 推理时的流程

```
输入图像 → CNN → RPN → RoI Align → 检测头
                                      ↓
                              ┌─── 分类: 得到类别 c
                              ├─── 回归: 得到框坐标
                              └─── Mask: 取出第 c 个 mask 通道 → 缩放回原图大小 → 阈值化(>0.5)
```

### Mask R-CNN 的完整架构图 (PDF lec12 第 63 页)

```
                            ┌──────────────────────┐
                            │   Mask R-CNN         │
                            │                      │
  ResNet-101 + FPN ────→ RPN ──→ RoI Align ──→  ┌──┼──┐
       ↑                          ↑             │  │  │
       │                          │          分类 回归 Mask
       └──────────────────────────┘          (FCN)
    共享卷积特征                    不对齐→对齐   新增分支
```

### PDF 第 63 页原文要点

> **主要贡献**：
> 1. 强化的特征提取网络：ResNet-101 + FPN 用作特征提取
> 2. 采用 RoI Align，解决 misalignment
> 3. 处理分割任务的分支使用全卷积网络
> 4. 检测分支和分割分支互相促进

### R-CNN 系列完整对比

| 模型 | 候选框 | RoI 处理 | 分类 | 回归 | 分割 | 可端到端 |
|---|---|---|---|---|---|---|
| R-CNN | Selective Search | Warp/Crop | SVM | 线性回归 | ❌ | ❌ |
| Fast R-CNN | Selective Search | **RoI Pooling** | Softmax | SmoothL1 | ❌ | ✅ 检测部分 |
| Faster R-CNN | **RPN** | RoI Pooling | Softmax | SmoothL1 | ❌ | ✅ 全部 |
| **Mask R-CNN** | RPN | **RoI Align** | Softmax | SmoothL1 | **FCN mask** | ✅ 全部 |
