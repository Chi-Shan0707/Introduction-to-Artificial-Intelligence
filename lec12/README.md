# 第 12 讲：计算机视觉与自然语言处理：检测、分割、生成

> 课程：Introduction to Artificial Intelligence · 复旦大学  
> 日期：2026-05-21  
> 资料：人工智能-12-CV&NLP.pdf、本讲 Markdown/实践材料

## 本讲定位

从图像分类扩展到目标检测、图像分割、视频分割、图像生成与 NLP 基础，连接 CNN 与多模态任务。

## 学习目标

- 理解分类、定位、检测、语义分割、实例分割之间的输出差异。
- 掌握 R-CNN 系列、YOLO/SSD 一阶段检测和 IoU/mAP 等评价指标。
- 知道图像生成与 NLP 基础如何走向多模态大模型。

## 知识地图

| 模块 | 内容 |
| --- | --- |
| 目标检测 | 在图中找出多个对象的位置和类别，输出 bounding boxes + labels + scores。 |
| 图像分割 | 语义分割给每个像素类别，实例分割进一步区分同类不同对象。 |
| 视频分割 | 在时间维度保持对象一致性，处理遮挡、消失再现和镜头变化。 |
| 生成与 NLP | 扩散模型、文本表示和语言模型共同支撑图文生成与视觉语言模型。 |

## 核心概念

- Two-stage 检测先产生候选区域再分类/回归，精度强但流程较重；One-stage 直接密集预测，速度更快。
- R-CNN 引入 CNN 特征用于检测；Fast/Faster R-CNN 用共享特征和 RPN 提升效率。
- YOLO 把检测转成一次前向预测，适合实时场景。
- 分割任务比检测更细粒度，模型必须同时理解语义和空间边界。

## 课堂讲解补充

第 12 讲是课程中计算机视觉应用最密集的一讲：检测解决“在哪里有什么”，分割解决“每个像素属于什么/哪个实例”，视频分割进一步加入时间一致性，图像生成和 NLP 则把视觉推向多模态。CNN 在这里从分类骨干扩展为视觉系统的特征提取器。

### 复习组织方式

- **问题背景**：这节课为什么要引入这个概念。
- **方法主线**：算法或模型按什么步骤工作。
- **公式/代码**：至少抓住一个能落地计算的表达。
- **局限性**：说明它在哪些场景下会失效或需要改进。

## 公式与记忆点

- IoU = area(box_pred ∩ box_gt) / area(box_pred ∪ box_gt)。
- mAP：在多个类别和 IoU 阈值下对 AP 求平均。
- 语义分割常用 mIoU：对各类别 IoU 取平均。

## 易错点

- 检测框看起来接近但 IoU 低，可能是小目标或框偏移造成。
- 把语义分割和实例分割混淆：前者不区分同类个体，后者区分。
- 只看可视化效果不看 mAP/mIoU，难以专业比较模型。
