Introduction to Artificial Intelligence

第 11 讲：Transformer 与大模型基础

系统学习自注意力、多头注意力、位置编码和 Transformer 架构，并连接 BERT/GPT/MOSS 等大模型。

学习目标

  • 掌握 Q/K/V、自注意力和多头注意力的计算流程。
  • 理解 Transformer 为什么比 RNN 更易并行、能捕捉长距离依赖。
  • 区分 encoder-only、decoder-only 与 encoder-decoder 架构及其代表模型。

知识地图

模块内容
自注意力同一序列内部每个 token 与其他 token 计算相关性。
多头机制不同 head 学习不同关系子空间，提高表达能力。
模型家族BERT 偏理解，GPT 偏生成，T5/原始 Transformer 支持 Seq2Seq。

核心概念讲解

要点 1

Q 是查询，K 是可匹配的键，V 是被加权汇聚的信息。

要点 2

位置编码弥补注意力本身不含顺序信息的问题。

要点 3

LLM 能力来自架构、数据规模、训练目标、对齐方法和推理策略共同作用。

课堂讲解补充

Transformer 的核心突破是用自注意力替代循环结构，使长距离依赖建模和并行训练同时成为可能。Q/K/V、多头注意力、位置编码、残差连接与 LayerNorm 共同组成大模型的基本模块。学习本讲要把公式和张量形状对应起来。

表达结构

复习时可以按“问题从哪里来、方法怎么做、公式或代码落在哪里、效果怎么评价、局限是什么”来整理。

与课程主线的连接

复习时顺手把它和前后几讲连起来：这一讲解决了什么问题，又给后面的模型或实验留下了哪些工具。

公式速查与解释

  • Attention(Q,K,V)=softmax(QKᵀ/√d_k)V。
  • 残差结构：x_{l+1}=LayerNorm(x_l+Sublayer(x_l))。
阅读建议：不要孤立背公式；每个公式都要能说出变量含义、适用条件和在代码/实验中的位置。

从课堂到项目的标准流程

步骤操作要点
1. 定义任务先说清输入、输出、假设与评价指标，避免只罗列算法名。
2. 建立表示选择逻辑符号、状态空间、特征向量、张量或 MDP 状态。
3. 选择模型根据数据规模、结构先验和计算预算选择规则、搜索、线性模型、神经网络或 RL 方法。
4. 训练/推理明确损失函数、优化过程、推理步骤和失败模式。
5. 评估迭代用与任务匹配的指标验证效果，并记录可复现实验设置。

代码/伪代码示例

下面的代码只保留课堂最需要抓住的骨架；真正练习时再对照 notebooks、讲义和项目文件补全细节。

import torch
from torch import nn

attn = nn.MultiheadAttention(embed_dim=512, num_heads=8, batch_first=True)
x = torch.randn(4, 128, 512)
y, weights = attn(x, x, x, need_weights=True)
print(y.shape, weights.shape)

常见误区

  • 把 attention 理解成固定规则；它是由数据训练出的动态加权。
  • 忽视 mask：生成式模型必须防止当前位置看到未来 token。
