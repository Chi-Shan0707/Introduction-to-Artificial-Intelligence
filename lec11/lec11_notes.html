03

重点难点：考试和汇报最容易卡住的地方

  • Q 是查询，K 是可匹配的键，V 是被加权汇聚的信息。
  • 位置编码弥补注意力本身不含顺序信息的问题。
  • LLM 能力来自架构、数据规模、训练目标、对齐方法和推理策略共同作用。
  • LSTM 不是三个随便的 sigmoid：遗忘门决定保留旧记忆，输入门决定写入新信息，输出门决定暴露多少状态。
  • Transformer 必须有位置编码，否则自注意力本身不含顺序。
  • 生成式 decoder 需要 causal mask，不能偷看未来 token。

公式推导：从目标到结论

下面不是孤立公式，而是推导链：先写目标，再说明约束，最后解释算法为什么长成现在这样。

RNN：h_t=tanh(W_xx_t+W_hh_{t-1}+b)。
LSTM：f_t=σ(W_f[x_t,h_{t-1}]+b_f), i_t=σ(W_i[x_t,h_{t-1}]+b_i), C_t=f_t⊙C_{t-1}+i_t⊙Ĉ_t。
Attention(Q,K,V)=softmax(QKᵀ/√d_k)V。
  1. 普通 RNN 用单一隐藏状态压缩历史，长序列中早期信息容易在多次乘法中衰减。
  2. LSTM 把状态拆成 cell state 和 hidden state，用遗忘门、输入门、输出门控制信息流。
  3. Q/K/V 把“我要找什么”“我能被什么匹配”“我提供什么内容”分开。
  4. 自注意力一次性计算 token 两两相关性，比 RNN 更易并行。

→

→

C_t=f_t⊙C_{t-1}+i_t⊙Ĉ_t, h_t=o_t⊙tanh(C_t)

# Recurrent Neural Network (RNN) 核心思想
## 1. 基本概念
Recurrent (循环) 的核心思想在于**网络结构没有变化且参数共享**。模型循环往复地处理序列数据。
- 上一个时间步（$t-1$）输出的隐状态 (Hidden State) $H_{t-1}$
根据这两个输入，模型计算得出当前步的隐状态 $H_t$。
## 2. 核心公式与参数共享
最朴素的 RNN 公式展现了 recurrent 的计算方式：
$$H_t = \tanh(W_{xh} X_t + W_{hh} H_{t-1})$$
- $W_{xh}$：输入特征 $X_t$ 对应的权重矩阵。
- $W_{hh}$：上一步隐状态 $H_{t-1}$ 对应的权重矩阵。
- **参数共享**：在所有的时间步 $t$ 中，$W_{xh}$ 和 $W_{hh}$ 以及网络结构都是**固定不变**的（不需要做任何变动）。
- **展开视角**：如果把 RNN 按照时间步展开来看，它相当于具有无限长层的网络，但它的特色在于**每一层的参数和前一层都是相同的**。
## 3. 激活函数 (Activation Function)
RNN 内部常用的是 **$\tanh$ (Hyperbolic Tangent)** 激活函数：
- 公式：$\tanh(x) = \frac{e^x - e^{-x}}{e^x + e^{-x}}$
- 形状与 Sigmoid 类似，但分布区间不同。Sigmoid 的输出区间是 $(0, 1)$，而 $\tanh$ 的输出区间是 $(-1, 1)$，中心点在 $0$。
## 4. 输出计算
有了当前步的隐状态 $H_t$ 之后，通常有两种处理方式：
1. **直接输出**：让预测值 $Y_t = H_t$。
2. **计算输出**：引入新的权重矩阵 $W_{hy}$ 计算输出，$Y_t = f_{why}(H_t)$。
## 5. 工程实现体会 (PyTorch)
由于 recurrent 的思想，模型仅仅定义了上述单个状态转化的操作（即这一层的结构）。只要按时间步顺序提供当前输入和上一步输出，就能无穷尽地处理序列输入。在 PyTorch 中，只需定义好 RNN 单层结构即可处理任意长度序列。

GRU (Gated Recurrent Unit) 是 RNN (Recurrent Neural Network) 的一种升级增强版（变体）。
RNN 是一大类网络结构的统称：只要网络存在着“把上一步的隐状态作为当前步输入”的循环（Recurrent）结构，都可以叫 RNN。我们通常所说的“最朴素的 RNN（Vanilla RNN）”就是你笔记中只有一层
GRU 是 RNN 的一种具体实现：它是为了克服基础 RNN 的致命缺陷而被发明出来的。
2. 为什么需要从基础 RNN 升级到 GRU？（解决“失忆”问题）
基础 RNN 的痛点：对于长序列数据（比如非常长的一段话），基础 RNN 存在序列长依赖问题（长期记忆衰退）。在反向传播更新参数时，会产生梯度消失（Vanishing Gradient）。这意味着它虽然理论上有记忆，但实际上只能记住最近几步的信息，前面的信息早就被冲刷掉了。
GRU 的拯救方案：GRU 创造性地在 RNN 的神经元内部引入了**“门控机制（Gating Mechanism）”**。
相比于基础 RNN 遇到什么信息都一股脑全盘接收，GRU 内部设立了两个“门”（其实就是两个神经网络分支产生的 0 到 1 之间的权重矩阵）：
更新门 (Update Gate)：决定上一个状态的记忆有多少要保留到这一步，以及包含多少当前的新信息。（解决长期记忆的关键）
重置门 (Reset Gate)：决定如何把过去的记忆与当前的输入结合。如果重置门接近 0，网络就会完全“忘记”过去的状态，只看当前的输入。
你可以把 基础 RNN 想象成一个记性不太好的人，听后面的话就会忘掉前面的话；
而 GRU 就是给他配了一个智能记事本（门控），能够自动判断“哪句是废话该忘掉（重置门）”、“哪句是关键信息该一直存着（更新门）”。
(注：另一个著名的变体是 LSTM，GRU 相比 LSTM 少了一个门，计算更简单、速度更快，但在大多数情况下的性能和 LSTM 不相上下。)

顾名思义，它的操作就像是把两块乐高积木拼在一起。对于张量（Tensor/矩阵）来说，就是在某一个指定的维度上，把数据接上**。
