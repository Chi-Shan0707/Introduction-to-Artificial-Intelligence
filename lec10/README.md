# 第 10 讲：序列建模：RNN、LSTM 与注意力

> 课程：Introduction to Artificial Intelligence · 复旦大学  
> 日期：2026-05-14 前后  
> 资料：无 PDF；基于 lec.md/notes.md

## 本讲定位

从循环神经网络处理序列开始，理解长期依赖问题，并过渡到注意力机制。

## 学习目标

- 理解序列数据与静态样本不同：当前输出依赖历史上下文。
- 掌握 RNN 的隐藏状态递推，以及梯度消失导致的长期依赖困难。
- 知道 LSTM/GRU 用门控机制缓解信息遗忘，并为 Transformer 铺垫。

## 知识地图

| 模块 | 内容 |
| --- | --- |
| RNN | h_t=f(Wx_t+Uh_{t-1})，共享参数逐步处理序列。 |
| LSTM/GRU | 门控机制决定写入、遗忘和输出，缓解长序列训练问题。 |
| Attention | 允许模型直接比较任意位置之间的相关性，而非只靠最后隐藏状态。 |

## 核心概念

- RNN 适合文本、语音、时间序列等顺序结构，但难以并行。
- LSTM 的遗忘门、输入门、输出门相当于可学习的信息路由。
- 注意力把“从哪里取信息”变成可微分的权重分配。

## 课堂讲解补充

序列建模回答“上下文如何影响当前预测”。RNN 用隐藏状态压缩历史，LSTM/GRU 用门控机制缓解长期依赖，而注意力机制进一步允许模型直接读取任意位置的信息。理解这条线索，才能自然过渡到 Transformer。

### 复习组织方式

- **问题背景**：这节课为什么要引入这个概念。
- **方法主线**：算法或模型按什么步骤工作。
- **公式/代码**：至少抓住一个能落地计算的表达。
- **局限性**：说明它在哪些场景下会失效或需要改进。

## 公式与记忆点

- RNN: h_t = tanh(W_x x_t + W_h h_{t-1} + b)。
- Attention(Q,K,V)=softmax(QKᵀ/√d_k)V。

## 易错点

- 把序列长度维和 batch 维弄混，尤其是 `batch_first` 参数。
- 认为 LSTM 已完全解决长期依赖；实际长文本仍更依赖注意力/Transformer。
