知识地图：本讲有哪些模块

RNN

h_t=f(Wx_t+Uh_{t-1})，共享参数逐步处理序列。

LSTM/GRU

门控机制决定写入、遗忘和输出，缓解长序列训练问题。

Attention

允许模型直接比较任意位置之间的相关性，而非只靠最后隐藏状态。

03

重点难点：考试和汇报最容易卡住的地方

  • RNN 适合文本、语音、时间序列等顺序结构，但难以并行。
  • LSTM 的遗忘门、输入门、输出门相当于可学习的信息路由。
  • 注意力把“从哪里取信息”变成可微分的权重分配。
  • 卷积输出尺寸必须逐层计算；只凭直觉很容易在 flatten 前维度错位。
  • 池化会丢空间细节，分割任务中过度下采样会损害边界。
  • 经典架构比较要说清：AlexNet 证明深 CNN 有效，VGG 强调小卷积堆叠，ResNet 解决深层退化。
答题时不要只写名词。每个名词至少配一个公式、一个流程节点、一个失败模式。

流程节点要追踪的问题
序列数据准备输入变长或等长文本序列，经由 Word Embedding 层映射成三维张量 X ∈ R^{B×T×d_model}。失败于未对填充 token 做 mask 处理。
RNN 循环步进在循环中一步一步输入 x_t，计算 h_t = tanh(W_x x_t + W_h h_{t-1} + b)。失败于长距离依赖导致的梯度消失。
LSTM 门控传递利用遗忘门 f_t、输入门 i_t、输出门 o_t，在 cell state 记忆管道上累加流转。缓解长序列衰减。失败于多门控乘法增加计算延迟。
Q/K/V 投影投射将输入序列线性映射为 Query、Key 和 Value，代表“找什么”、“凭什么被选”和“汇聚什么内容”。失败于参数初始化造成秩崩溃。
注意力得分矩阵计算 Q @ Kᵀ / √d_k。计算每一个 token 对所有其他 token 的关联概率分布。失败于长文本中 O(T²) 显存溢出。
注意力输出加权将权重 softmax 之后，加权汇聚 Value 矩阵得到最终表征，输出 Z ∈ R^{B×T×d_model}。失败于缺少因果遮掩导致未来信息泄露。

公式推导：从目标到结论

下面不是孤立公式，而是推导链：先写目标，再说明约束，最后解释算法为什么长成现在这样。

2D 卷积：Y[i,j]=Σ_uΣ_v X[i+u,j+v]K[u,v]。
same padding 在 K 为奇数、S=1 时常取 P=(K-1)/2。
ResNet 残差：y=F(x)+x，缓解深层网络退化。
  1. 局部性假设让每个神经元只看邻域，减少连接数。
  2. 局部不变性让共享卷积核在不同位置检测同一模式。
  3. 池化层没有可学习参数，却能降低分辨率并增强小幅平移鲁棒性。
  4. 残差连接给梯度提供近路，使更深网络更容易优化。

代码锚点：概念落到实现

\n# -*- coding: utf-8 -*-
# 第 10 讲: 序列建模与注意力 - PyTorch 实现 RNN Cell 与单头 Scaled Dot-Product Attention 的张量维度演变
import torch
import torch.nn as nn
import torch.nn.functional as F

# ================= 1. RNN Cell 单步推进 =================
batch_size, seq_len, input_size, hidden_size = 2, 4, 8, 16
rnn_cell = nn.RNNCell(input_size=input_size, hidden_size=hidden_size)

# 输入序列 X (2 x 4 x 8)，隐藏状态初始化为 0 (2 x 16)
X = torch.randn(batch_size, seq_len, input_size)
h_t = torch.zeros(batch_size, hidden_size)

print("--- RNN Cell 隐藏状态演化 ---")
for t in range(seq_len):
    # 逐时刻推递，h_t = tanh(W_x * x_t + W_h * h_{t-1} + b)
    h_t = rnn_cell(X[:, t, :], h_t)
    print(f"时刻 t={t} | 隐藏状态张量形状: {list(h_t.shape)}")

# ================= 2. 缩放点积注意力 (Scaled Dot-Product Attention) =================
# 假定输入序列通过线性变换得到了 Q, K, V (2 x 4 x 16)
# Attention(Q, K, V) = softmax(Q @ K.T / sqrt(d_k)) @ V
Q = torch.randn(batch_size, seq_len, hidden_size)
K = torch.randn(batch_size, seq_len, hidden_size)
V = torch.randn(batch_size, seq_len, hidden_size)

d_k = K.size(-1)
# 1. 相似度计算: Batch Matmul (B x N x H) @ (B x H x N) -> B x N x N
scores = torch.matmul(Q, K.transpose(-2, -1)) / (d_k ** 0.5)
# 2. 归一化为概率分布
attn_weights = F.softmax(scores, dim=-1)
# 3. 汇聚信息: (B x N x N) @ (B x N x H) -> B x N x H
output = torch.matmul(attn_weights, V)

print("\n--- Attention 机制张量变化 ---")
print(f"注意力得分矩阵 scores 形状      : {list(scores.shape)}")
print(f"归一化注意力权重 weights 形状  : {list(attn_weights.shape)}")
print(f"注意力输出 output 形状         : {list(output.shape)}")
\n

# Part 1: 引言（PDF p23–p30）
## 1.1 从全连接网络到卷积网络
### 问题：深度全连接网络处理图像的困境
输入层 → 隐藏层 → 隐藏层 → ... → 输出层
当输入是一张 **1000 × 1000 的灰度图像**时：
- 若第一个隐藏层维度也为 10⁶，且采用**全连接**
- 则参数数目 = 10⁶ × 10⁶ = **10¹²**
> 假设一个参数用 32-bit 浮点数（4 Byte）存储，仅输入层到第一个隐藏层的权重就需要 **4 TB** 内存。这对当时的硬件是完全不可接受的。
对于彩色图像（3 channels），问题更加严重（PDF p27）。
### 核心问题
> **如何把大量参数降维成少量参数，同时不影响任务结果？**
对于图像处理，我们是不是可以利用图像的某些**固有模式**？
## 1.2 图像的两个核心先验（PDF p29–p30）
### 先验一：局部性 (Locality)
**观察**：图像中目标通常只出现在局部区域，相邻像素关联性强，远距离像素关联弱。
1. 定义一种**提取局部特征**的方法，网络的前面几层只探索输入图像中的局部区域；
2. 用这种方法**遍历整张图片**，网络最后几层聚合这些局部特征，在整个图像级别进行预测。
### 先验二：局部不变性 (Local Invariance)
**观察**：目标可以出现在图像的不同位置、不同尺度。基于固定位置、大小的"单元格"(Cell-based) 方法有局限性。
**解决思路**：设计一个网络，使得目标执行**尺度缩放、平移**等操作时不影响其语义信息。
## 1.3 CNN 的三层架构范式（PDF p30）
┌─────────────────────────────────────────────────────┐
│  输入图像                                            │
│    ↓                                                │
│  【卷积层】每次卷积仅与部分像素连接，提取局部图像特征   │
│    ↓                                                │
│  【池化层】实现特征降维，压缩数据和参数量，减小过拟合   │
│    ↓                                                │
│  【卷积层】                                          │
│    ↓                                                │
│  【池化层】                                          │
│    ↓                                                │
│  【全连接层】从局部到整体，综合多个部分的识别线索      │
│    ↓                                                │
│  【Softmax】输出概率分布                              │
└─────────────────────────────────────────────────────┘
> **关键洞察**："卷积层 + 池化层"的组合可以覆盖不同目标尺度。

### 2.1.1 一维卷积示例（PDF p33–p40）
Input Vector:  [1, 2, 3, 4, 5, 6]
Kernel:        [2, 2, 2]   (长度 3)
| 窗口位置 | 窗口取值 | 逐元素乘法 | 求和 | 输出 |
|---------|---------|-----------|------|------|
| 1 | [1, 2, 3] | 1×2, 2×2, 3×2 | 2+4+6 | **12** |
| 2 | [2, 3, 4] | 2×2, 3×2, 4×2 | 4+6+8 | **18** |

**例题 2.2**（含零元素的卷积核，PDF p41–p44）：
Input Vector:  [1, 2, 3, 4, 5, 6]
Kernel:        [1, 0, 1]   (长度 3)
| 窗口位置 | 窗口取值 | 逐元素乘法 | 求和 | 输出 |
|---------|---------|-----------|------|------|
| 1 | [1, 2, 3] | 1×1, 2×0, 3×1 | 1+0+3 | **4** |

**例题 2.3**（无 Padding 的 2D 卷积）：
**输出尺寸**：`(4 - 3)/1 + 1 = 2`，即 2×2。
= 0 - 1 + 0 + 0 + 0 + 1 + 2 + 0 + 1 = 3
= 1 - 1 + 0 + 0 + 1 + 1 + 0 + 0 + 2 = 5

## 2.2 Padding：保持尺寸不变（PDF p48–p49）
上面的例子中，输入 4×4，卷积核 3×3，输出变成 2×2，**尺寸缩小了**。很多时候我们希望保持空间分辨率。
**解决方案：在输入周围填充 0（Zero Padding）**
若对 4×4 输入 padding 1 圈 0，则新尺寸为 6×6：
Output size = (6 - 3)/1 + 1 = 4×4
> 这种保持尺寸的 padding 策略在构建深层网络时至关重要，否则特征图会迅速缩小到 1×1。
## 2.3 控制输出尺寸的三驾马车（PDF p50–p51）
### 通用输出尺寸公式（PDF p50）
\text{Output Size} = \frac{\text{Input Size} + 2 \times \text{Padding} - \text{Kernel Size}}{\text{Stride}} + 1
> **注意**：必须确保分子能被 Stride 整除，否则会出现非整数输出尺寸，PyTorch 等框架通常会报错或取整。

序列建模深度补充：RNN/LSTM 到注意力的动机

序列数据的核心是当前位置依赖历史上下文。RNN 用共享参数递归更新隐藏状态，适合可变长度序列，但长距离依赖会受到梯度消失/爆炸影响。LSTM/GRU 用门控机制控制信息写入、遗忘和输出，缓解但没有彻底消除串行计算瓶颈。

h_t = f(W_xx_t + W_hh_{t-1}+b)
结构解决什么仍有什么问题
RNN可变长度序列建模长程依赖弱，难并行
LSTM用门控保留长期记忆计算更重，仍按时间步串行
GRU简化门控结构表达与稳定性依任务而定
Attention让当前位置直接读取相关位置需要处理位置和计算复杂度

通向 Transformer：RNN 的顺序偏置强但难并行；注意力弱化顺序递归，允许任意 token 直接交互。位置关系不再由递归天然提供，必须靠位置编码或相对位置机制补回。
