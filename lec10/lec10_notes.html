Introduction to Artificial Intelligence

第 10 讲：序列建模：RNN、LSTM 与注意力

从循环神经网络处理序列开始，理解长期依赖问题，并过渡到注意力机制。

2026-05-14 前后资料：无 PDF；基于 lec.md/notes.mdRNNLSTMGRUAttentionSequence Modeling

学习目标

  • 理解序列数据与静态样本不同：当前输出依赖历史上下文。
  • 掌握 RNN 的隐藏状态递推，以及梯度消失导致的长期依赖困难。
  • 知道 LSTM/GRU 用门控机制缓解信息遗忘，并为 Transformer 铺垫。

知识地图

模块内容
RNNh_t=f(Wx_t+Uh_{t-1})，共享参数逐步处理序列。
LSTM/GRU门控机制决定写入、遗忘和输出，缓解长序列训练问题。
Attention允许模型直接比较任意位置之间的相关性，而非只靠最后隐藏状态。

核心概念讲解

要点 1

RNN 适合文本、语音、时间序列等顺序结构，但难以并行。

要点 2

LSTM 的遗忘门、输入门、输出门相当于可学习的信息路由。

要点 3

注意力把“从哪里取信息”变成可微分的权重分配。

课堂讲解补充

序列建模回答“上下文如何影响当前预测”。RNN 用隐藏状态压缩历史，LSTM/GRU 用门控机制缓解长期依赖，而注意力机制进一步允许模型直接读取任意位置的信息。理解这条线索，才能自然过渡到 Transformer。

表达结构

复习时可以按“问题从哪里来、方法怎么做、公式或代码落在哪里、效果怎么评价、局限是什么”来整理。

公式速查与解释

  • RNN: h_t = tanh(W_x x_t + W_h h_{t-1} + b)。
  • Attention(Q,K,V)=softmax(QKᵀ/√d_k)V。
阅读建议：不要孤立背公式；每个公式都要能说出变量含义、适用条件和在代码/实验中的位置。

从课堂到项目的标准流程

步骤操作要点

代码/伪代码示例

下面的代码只保留课堂最需要抓住的骨架；真正练习时再对照 notebooks、讲义和项目文件补全细节。

import torch
from torch import nn

rnn = nn.LSTM(input_size=128, hidden_size=256, num_layers=2, batch_first=True)
x = torch.randn(32, 50, 128)  # batch, seq_len, feature
out, (h, c) = rnn(x)
print(out.shape, h.shape)  # [32,50,256], [2,32,256]

常见误区

  • 把序列长度维和 batch 维弄混，尤其是 `batch_first` 参数。
  • 认为 LSTM 已完全解决长期依赖；实际长文本仍更依赖注意力/Transformer。
