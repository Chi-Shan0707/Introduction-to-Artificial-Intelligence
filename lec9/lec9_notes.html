03

重点难点：考试和汇报最容易卡住的地方

  • PyTorch 的优势是动态图：前向计算即构图，调试体验接近普通 Python。
  • 训练模式 `model.train()` 与评估模式 `model.eval()` 会影响 Dropout/BatchNorm。
  • CV 代码要始终检查 image tensor 的尺度、归一化方式和类别映射。
  • 数据和模型必须在同一 device。
  • 输出类别数必须和数据集标签空间一致。
  • 图像增强要只作用于训练集，验证/测试保持稳定预处理。
  • 训练准确率高但验证差，优先排查过拟合、数据泄漏和归一化。

公式推导：从目标到结论

下面不是孤立公式，而是推导链：先写目标，再说明约束，最后解释算法为什么长成现在这样。

交叉熵：L=-Σ_c y_c log softmax(z)_c。
标准化：x_norm=(x-mean)/std。
训练循环不变量：zero_grad → forward → loss → backward → step。
  1. Dataset 负责单个样本，DataLoader 负责 batch、shuffle 和并行读取。
  2. Autograd 在 forward 时建立动态图，backward 时沿图累积梯度。
  3. model.train() 与 model.eval() 会影响 Dropout/BatchNorm 行为。
  4. 评估时用 torch.no_grad()，减少显存并避免构图。

PyTorch 调试清单：从能跑到可信

Shape first

每次换数据集或模型，先打印一个 batch 的 images.shape、labels.shape、logits.shape。图像分类通常要求输入为 [N,C,H,W]，输出为 [N,num_classes]。如果 CrossEntropyLoss 报错，优先检查类别数和 label dtype。

Loss sanity check

随机初始化时，十分类交叉熵通常接近 log(10)。如果 loss 一开始就是 NaN，检查学习率、归一化、除零和数据中是否有非法值；如果 loss 不降，检查 optimizer 是否拿到了 model.parameters()。

Train / eval switch

model.train() 打开 Dropout 并更新 BatchNorm 统计量；model.eval() 固定这些行为。验证集必须配合 torch.no_grad()，否则显存和计算图都会被浪费。

Reproducibility

记录随机种子、batch size、学习率、weight decay、数据增强、epoch 数和最佳验证指标。课程实验不只是跑出结果，还要能解释为什么这个结果可信。
