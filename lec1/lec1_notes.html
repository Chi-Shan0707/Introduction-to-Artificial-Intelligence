流程节点要追踪的问题
现实任务视觉目标感知与交互规划。输入为图像/视频数据，输出为像素分类/动作决策。失败于光照突变、未见分布。
输入/输出定义输入 x ∈ R^{H×W×3} 图像，输出 y ∈ R^{H×W} 像素掩码。参数为网络权重 θ。失败于标注噪声、语义模糊。
数据与表示将物理图像映射成像素特征及多维向量表示 φ(x)。失败于特征表达不充分、维度灾难。
模型或规则选择感知映射函数 fθ(x) = P(y|x)。利用数据分布驱动端到端参数优化。失败于结构归纳偏置不匹配。
训练/推理基于损失函数 L(y, fθ(x)) 使用 Adam 优化。推理时前向传播计算预测值。失败于局部最优或梯度消失。
评价与迭代在测试集上计算 mAP、mIoU 和 Accuracy。根据误差分析（如漏检、错检）调整模型结构和数据增强。

公式推导：从目标到结论

下面不是孤立公式，而是推导链：先写目标，再说明约束，最后解释算法为什么长成现在这样。

学习三要素：Experience E, Task T, Performance P；若经验 E 使任务 T 上的性能 P 提升，则称系统发生学习。
AI 系统闭环：数据 D → 表示 φ(x) → 模型 fθ → 预测 ŷ → 指标 M(ŷ,y) → 误差分析。
  1. 先把“智能”落到可观察任务：分类、检索、生成、规划或控制。
  2. 再确定输入输出，例如图像分割输入视频/文本/音频，输出像素级 mask。
  3. 选择性能指标，指标决定优化方向；没有指标的 AI 讨论不可检验。
  4. 最后回到工程闭环：数据是否覆盖真实场景，模型是否能泛化，错误是否可分析。

代码锚点：概念落到实现

\n# -*- coding: utf-8 -*-
# 第 1 讲: 绪论 - 人工智能系统工程闭环的极简 NumPy 实现
import numpy as np

# 1. 任务定义 (Task): 线性分类任务 (2D 点分类，判别是类别 0 还是 1)
# 输入: x = [x1, x2] ∈ R^2
# 输出: y ∈ {0, 1}
X = np.array([[1.0, 2.0], [2.0, 1.0], [2.0, 3.0], [3.0, 1.0]]) # 数据 D
Y = np.array([0, 0, 1, 1])

# 2. 模型与参数 (Model & Representation): 线性分类器 + Sigmoid 激活
# f_theta(x) = sigmoid(w^T * x + b)
w = np.array([0.5, -0.5])
b = 0.0

def sigmoid(z):
    return 1 / (1 + np.exp(-z))

def forward(x):
    return sigmoid(np.dot(x, w) + b)

# 3. 优化与学习 (Optimization): 单步梯度下降更新
# Loss = - [y * log(p) + (1-y) * log(1-p)] (交叉熵损失)
# 更新规则: w <- w - alpha * (p - y) * x
alpha = 0.1 # 学习率
print("--- 初始状态 ---")
for x, y in zip(X, Y):
    p = forward(x)
    loss = - (y * np.log(p) + (1-y) * np.log(1-p))
    print(f"输入: {x}, 目标: {y}, 预测概率: {p:.4f}, 损失: {loss:.4f}")

# 执行单步梯度更新
grad_w = np.zeros_like(w)
grad_b = 0.0
for x, y in zip(X, Y):
    p = forward(x)
    grad_w += (p - y) * x
    grad_b += (p - y)
w -= alpha * grad_w / len(X)
b -= alpha * grad_b / len(X)

print("\n--- 梯度下降一步后参数 ---")
print(f"更新后的 w: {w}, b: {b}")

# 4. 评价指标 (Evaluation): 准确率 (Accuracy)
predictions = np.array([1 if forward(x) >= 0.5 else 0 for x in X])
accuracy = np.mean(predictions == Y)
print(f"训练集准确率: {accuracy * 100:.1f}%")

# 第1讲：人工智能发展与学习范式演进
## 本讲提要
- 计算范式演进：计算 -> 逻辑计算 -> 机器计算
## 1. 计算范式演进
从“计算”到“逻辑计算”再到“机器计算”，核心变化是：人类从手工设计算法，逐步走向让机器自动学习表示和规则。
里程碑事件：明斯基证明了单层神经网络不能解决异或（XOR）问题，这直接推动了后续多层网络与表示学习的发展。
## 2. 机器学习与深度学习对比
1. 传统机器学习常依赖人工特征工程：任务一变，特征通常要重做；性能瓶颈也常来自特征设计不足。
2. 深度学习强调表征学习：让模型从数据中自动学习多层特征，减少对人工特征的依赖。
## 3. 人工智能三起两落的历程
### 第一起：黄金时代（1956-1970）
- 1956年达特茅斯会议标志着人工智能作为独立学科的诞生
- 乐观主义盛行，学者们认为20年内机器将超越人类
### 第一落：第一次寒冬（1970-1980）
### 第二起：专家系统时代（1980-1987）
- 神经网络研究复兴，Hopfield网络、BP算法相继提出
### 第二落：第二次寒冬（1987-2006）
### 第三起：深度学习时代（2006-至今）
- 2006年深度学习突破，Hinton提出深度信念网络
- ImageNet竞赛突破，AlphaGo战胜李世石等里程碑
## 4. Hinton 的观点（摘录）
- Digital computation requires a lot of energy but makes it very easy for agents that have the same model of the world to share what they have learned by sharing weights or gradients.
Example: GPT-4 knows thousands of times more than any one person, using only about 2% as many weights.
- Biological computation requires much less energy but is much worse at sharing knowledge between agents.
Final statement: If energy is cheap, digital computation is just better.
对应理解：数字计算的优势在于“知识可复制、可传播、可叠加”；生物计算的优势在于“低能耗”，但跨个体知识传递效率低。这一判断对大模型时代的工程路线具有很强解释力。

绪论深度补充：AI 是任务—数据—模型—评价闭环

绪论不能只背“人工智能是什么”。更专业的理解是：AI 系统把现实问题转成可计算任务，再通过数据、表征、模型、优化和评价形成闭环。课程中展示的视觉分割、视频目标分割、具身导航和生成模型案例说明，AI 往往不是一个孤立网络，而是数据集、模型结构、训练目标、推理协议和评价指标共同构成的系统。

AI problem = Task definition + Data/Experience + Representation + Optimization + Evaluation
层次要问的问题为什么重要
任务定义输入、输出、约束、成功标准是什么？任务定义错，模型越强越可能优化错目标。
数据经验数据覆盖哪些场景？是否有偏差？模型只能从经验分布中学习，分布外泛化需要额外假设。
模型与优化模型结构是否匹配任务先验？损失函数优化什么？结构决定可表达内容，损失决定训练方向。
评价协议指标是否反映真实风险？高 benchmark 分数不等于真实场景可靠。

有价值的思考：如果一个模型在分类 benchmark 上准确率高，但对少数类、遮挡、小目标或分布外场景失败，它并不能被简单称为“智能”。绪论之后每一讲都要回到这个问题：该方法到底解决了 AI 闭环中的哪一个环节，又在哪些环节留下了风险？
