# 模型评估：混淆矩阵与核心指标

## 1. 混淆矩阵（Confusion Matrix）

混淆矩阵是分类模型评估的基础工具，用于统计模型预测结果与真实标签的匹配情况。

以二分类为例（正类 Positive / 负类 Negative）：

| | 预测为正类 (Predicted Positive) | 预测为负类 (Predicted Negative) |
|---|---|---|
| **真实为正类 (Actual Positive)** | **TP**（True Positive）<br>真正例：预测正确 | **FN**（False Negative）<br>假负例：漏报 |
| **真实为负类 (Actual Negative)** | **FP**（False Positive）<br>假正例：误报 | **TN**（True Negative）<br>真负例：预测正确 |

### 四个基本量的含义

| 缩写 | 全称 | 含义 | 直观理解 |
|------|------|------|----------|
| **TP** | True Positive | 真实为正，预测为正 | 抓对了坏人 |
| **FP** | False Positive | 真实为负，预测为正 | 冤枉了好人（误报） |
| **TN** | True Negative | 真实为负，预测为负 | 放过了好人 |
| **FN** | False Negative | 真实为正，预测为负 | 放跑了坏人（漏报） |

## 2. 核心评估指标

基于 TP、FP、TN、FN 四个基本量，可以计算以下常用指标：

### 2.1 准确率（Accuracy）

$$
\text{Accuracy} = \frac{TP + TN}{TP + FP + TN + FN}
$$

- **含义**：所有样本中，预测正确的比例。
- **适用场景**：类别分布均衡时。
- **局限**：类别不平衡时可能产生误导（如 99% 负样本，全猜负也有 99% 准确率）。

### 2.2 错误率（Error Rate）

$$
\text{Error Rate} = \frac{FP + FN}{TP + FP + TN + FN} = 1 - \text{Accuracy}
$$

- **含义**：所有样本中，预测错误的比例。
- **与准确率关系**：两者互补。

### 2.3 精确率 / 查准率（Precision）

$$
\text{Precision} = \frac{TP}{TP + FP}
$$

- **含义**：在所有预测为正的样本中，真正是正的比例。
- **关注点**：预测为正的"含金量"。
- **适用场景**：误报代价高时（如垃圾邮件检测：不想把正常邮件误判为垃圾邮件）。

### 2.4 召回率 / 查全率 / 灵敏度（Recall / Sensitivity）

$$
\text{Recall} = \frac{TP}{TP + FN}
$$

- **含义**：在所有真实正样本中，被正确找出的比例。
- **关注点**：正样本的"覆盖率"。
- **适用场景**：漏报代价高时（如疾病筛查：不想漏掉真正的患者）。

### 2.5 特异度（Specificity）

$$
\text{Specificity} = \frac{TN}{TN + FP}
$$

- **含义**：在所有真实负样本中，被正确识别为负的比例。
- **关注点**：负样本的识别能力。
- **与召回率关系**：召回率关注正类，特异度关注负类。

### 2.6 F1 Score

$$
F_1 = 2 \cdot \frac{\text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}}
$$

- **含义**：精确率和召回率的调和平均。
- **为什么用调和平均**：对极端值更敏感，两者都高时 F1 才高。
- **适用场景**：需要同时兼顾 Precision 和 Recall 时。

### 2.7 Fβ Score

$$
F_\beta = (1 + \beta^2) \cdot \frac{\text{Precision} \cdot \text{Recall}}{\beta^2 \cdot \text{Precision} + \text{Recall}}
$$

- **含义**：F1 的推广，允许调节 Precision 和 Recall 的权重。
- **$\beta > 1$**：更重视 Recall（如疾病筛查）。
- **$\beta < 1$**：更重视 Precision（如垃圾邮件检测）。

## 3. 指标汇总对比表

| 指标 | 公式 | 关注点 | 适用场景 |
|------|------|--------|----------|
| **Accuracy** | $\frac{TP+TN}{TP+FP+TN+FN}$ | 整体正确率 | 类别均衡 |
| **Error Rate** | $\frac{FP+FN}{TP+FP+TN+FN}$ | 整体错误率 | 类别均衡 |
| **Precision** | $\frac{TP}{TP+FP}$ | 预测为正的可靠性 | 误报代价高 |
| **Recall** | $\frac{TP}{TP+FN}$ | 正样本的覆盖率 | 漏报代价高 |
| **Specificity** | $\frac{TN}{TN+FP}$ | 负样本的识别率 | 需同时关注负类 |
| **F1 Score** | $2\cdot\frac{P\cdot R}{P+R}$ | Precision 与 Recall 的平衡 | 类别不平衡 |

## 4. 多分类场景下的扩展

多分类问题中，混淆矩阵扩展为 $K \times K$ 矩阵（$K$ 为类别数）：

| | 预测为类 1 | 预测为类 2 | ... | 预测为类 K |
|---|---|---|---|---|
| **真实为类 1** | $C_{11}$ | $C_{12}$ | ... | $C_{1K}$ |
| **真实为类 2** | $C_{21}$ | $C_{22}$ | ... | $C_{2K}$ |
| **...** | ... | ... | ... | ... |
| **真实为类 K** | $C_{K1}$ | $C_{K2}$ | ... | $C_{KK}$ |

- 对角线 $C_{ii}$ 表示正确分类的样本数。
- 非对角线 $C_{ij}$ 表示真实为类 $i$ 但被预测为类 $j$ 的样本数。

多分类指标计算常用两种策略：

| 策略 | 计算方式 | 特点 |
|------|----------|------|
| **Macro-average** | 对每个类单独算指标，再取算术平均 | 各类权重相等，适合类别均衡 |
| **Micro-average** | 先汇总所有类的 TP/FP/FN，再算指标 | 受大类主导，适合类别不平衡 |
| **Weighted-average** | 按各类别样本数加权平均 | 兼顾类别分布 |

---

## 5. 实践建议

1. **不要只看 Accuracy**：类别不平衡时，Accuracy 可能产生严重误导。
2. **根据业务选指标**：
   - 医疗筛查：优先看 Recall（不能漏诊）。
   - 垃圾邮件：优先看 Precision（不能误删重要邮件）。
   - 综合评估：看 F1 Score。
3. **结合 ROC 曲线与 AUC**：单一阈值下的指标不够全面，ROC-AUC 能反映模型在所有阈值下的综合表现。
4. **看混淆矩阵本身**：不仅看汇总指标，还要看具体哪两类容易混淆，有助于针对性改进模型。

## 6. PR曲线与平均精度（AP）

这四张幻灯片系统地展示了在机器学习（特别是目标检测和信息检索领域）中，**如何将一条原始的、波动的 Precision-Recall (PR) 曲线，通过严谨的数学平滑（插值）手段，最终量化为一个单一的核心评估指标：平均精度（Average Precision, AP）**。

这是诸如 PASCAL VOC、MS COCO 等权威计算机视觉竞赛中评估模型性能的标准演进过程。为了严谨解析，我们逐张图进行数学与物理意义的拆解：

### 第一张图：原始 PR 曲线的“锯齿”物理意义
幻灯片提出了一个核心问题：“这个 Precision-Recall 曲线的形状具体代表了什么？”

在真实的评估中，模型会为每一个预测样本输出一个置信度得分（Confidence Score）。我们将所有预测结果按置信度**从高到低排序**，然后逐一降低判定阈值，将更多的样本纳入“正类”预测中。
* **Precision 下降（曲线向下掉）**：当随着阈值降低，新纳入的预测是一个**假正例（False Positive, FP）**时，召回率（Recall）分母不变，分子不变，而精确率（Precision）的分母增加，导致 Precision 局部下降。
* **Precision 攀升（曲线往回折或平缓）**：当继续降低阈值，紧接着纳入了一批**真正例（True Positive, TP）**时，Recall 增加，同时这批高质量的 TP 会把整体的 Precision 重新拉高。

**结论**：这种锯齿状的非单调波动，代表了模型在局部排序上的“颠簸”。它说明置信度排序并不完美，有些 FP 被赋予了比 TP 更高的得分。

### 第二张图：引入单调性保障的“插值精度” (Interpolated Precision)
由于锯齿状曲线很难直接用于比较两个模型的优劣（曲线可能互相交叉），学术界引入了**插值精度（Interpolated Precision, $P_{interp}$）**，即图中的绿色包络线。

其核心数学定义为：
$$P_{interp}(r) = \max_{\tilde{r} \ge r} p(\tilde{r})$$

**理论意义**：这个公式表达了一种“单调非递增”的强约束。它的逻辑是：如果我在一个较高的 Recall（$\tilde{r}$）下能达到某个最高精度 $p(\tilde{r})$，那么在任何更低的 Recall（$r$）下，我的系统**理应**能达到至少同样高的精度（大不了我只输出那部分高置信度的 TP）。
通过取右侧（即大于等于当前召回率 $r$ 的区间内）的**最大精确率**，绿色曲线将原本波动的橙色曲线转化为了一个单调递减的阶梯函数（Step Function）。这消除了因局部排序误差带来的评估噪音。

### 第三张图：11点插值平均精度 (11-point AP)
在获得了单调的插值 PR 曲线后，我们需要计算曲线下的面积来作为单一标量指标 AP。幻灯片第三张展示的是**早期的近似积分法**（被 PASCAL VOC 2007 挑战赛采用）。

计算方法采用黎曼和（Riemann sum）的一种离散采样形式，在 Recall 轴上等距选取 11 个点：$\{0, 0.1, 0.2, \dots, 1.0\}$。
$$AP_{11} = \frac{1}{11} \sum_{r \in \{0, 0.1, \dots, 1.0\}} P_{interp}(r)$$
**局限性**：这种方法有两个明显缺陷。首先，它非常粗糙，11 个采样点可能会漏掉曲线在采样点之间的剧烈衰减；其次，对于那些 Recall 很低、性能很差的模型，即使最大 Recall 只有 0.05，在 $r=0$ 处的高精度也会被不合理地放大。

### 第四张图：全区间精确积分 (Exact Area Under Curve)
为了解决 11 点采样的误差，现代标准（如 PASCAL VOC 2010 及之后，以及 MS COCO 采用的 101 点/全积分思想）改用**精确计算插值曲线下方的面积（AUC, Area Under Curve）**。

数学表达为：
$$AP = \sum_{n} (r_{n+1} - r_n) P_{interp}(r_{n+1})$$

由于 $P_{interp}$ 是一个阶梯函数，这个积分实际上就是计算所有矩形区块的面积之和。
* $r_n$ 和 $r_{n+1}$ 代表 Recall 发生变化的两个相邻临界点。
* $(r_{n+1} - r_n)$ 是矩形的底（Recall 的增量）。
* $P_{interp}(r_{n+1})$ 是该区间对应的最大插值精度（矩形的高）。

**总结**：这四张图构成了一个完整的逻辑链条：**暴露原始排序的噪音（图1） $\rightarrow$ 施加数学约束消除局部噪音（图2） $\rightarrow$ 历史上的离散近似评估方案（图3） $\rightarrow$ 现代严谨的连续积分评估方案（图4）**。
