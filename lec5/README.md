## 本讲定位

用最大间隔理解 SVM，并系统梳理模型选择、验证集、交叉验证与分类指标。

## 学习目标

- 理解 SVM 的目标是寻找最大间隔超平面。
- 知道软间隔、核函数如何处理噪声与非线性。
- 能根据类别不平衡程度选择 Accuracy、F1、ROC-AUC 或 PR-AUC。

## 知识地图

| 模块 | 内容 |
| --- | --- |
| SVM | 凸优化、支持向量、最大间隔、软间隔。 |
| 模型选择 | 验证集、K 折交叉验证、超参数搜索。 |
| 评估指标 | 混淆矩阵、精确率、召回率、F1、ROC/PR 曲线。 |

## 核心概念

- 支持向量是距离决策边界最近、真正决定间隔的样本。
- 核技巧允许在高维特征空间做线性分割，而不显式计算高维映射。
- 类别极不平衡时 Accuracy 可能误导，应重点看召回、F1 和 PR-AUC。

## 课堂讲解补充

SVM 与模型评估帮助我们从“能训练模型”进入“能判断模型是否可靠”。最大间隔思想强调泛化边界，交叉验证强调调参规范，混淆矩阵和 AUC/PR-AUC 则要求我们根据真实任务风险选择指标。

## 公式与记忆点

- 硬间隔目标：min 1/2||w||², s.t. yᵢ(wᵀxᵢ+b) ≥ 1。
- 函数间隔/分类间隔：`z_i = y_i f(x_i)`；`z_i` 越大，说明样本不仅分对，而且离决策边界越远。
- Hinge 损失：`L_hinge(z)=max(0, 1-z)`。它不是“分错才惩罚”，而是“间隔不足也惩罚”：`0 < z < 1` 虽然分类正确，但仍在 margin 内。
- Precision=TP/(TP+FP)，Recall=TP/(TP+FN)。
- F1=2PR/(P+R)。

## 典型试题：由分类间隔计算 Hinge 损失

二分类标签通常取 `y∈{−1,+1}`，模型输出为实数打分 `f(x)`。定义分类间隔：

```text
z = y f(x)
```

则 Hinge 损失只看 `z`：

```text
L(z) = max(0, 1 - z)
```

| 分类间隔 `z` | 分类是否正确 | 几何含义 | Hinge 损失 `max(0,1-z)` |
|---:|---|---|---:|
| `2` | 正确 | 在间隔外，置信度充足 | `0` |
| `1` | 正确 | 正好压在 margin 边界上 | `0` |
| `0.5` | 正确 | 分对了，但离边界太近，间隔不足 | `0.5` |
| `0` | 边界上 | `f(x)=0`，模型没有明确倾向 | `1` |
| `-0.5` | 错误 | 分错，且离边界不远 | `1.5` |
| `-1` | 错误 | 分错，并且错误置信度更强 | `2` |

考试时先判断 `z`，不要先纠结 `y` 和 `f(x)` 各自的符号：

- `z ≥ 1`：分对且 margin 足够，损失为 `0`。
- `0 < z < 1`：分对但 margin 不够，损失为 `1-z`。
- `z = 0`：在决策边界上，损失为 `1`。
- `z < 0`：分错，损失大于 `1`，且错得越自信损失越大。

SVM 的软间隔目标可以写成经验 Hinge 损失与正则项的折中：

```text
min_{w,b}  1/2 ||w||^2 + C Σ_i max(0, 1 - y_i(w^T x_i + b))
```

其中 `C` 控制“少犯训练错误/少违反间隔”和“保持大间隔、提高泛化”之间的权衡。

## 易错点

- 只报告最高 Accuracy 但不说明数据分布，会掩盖少数类失败。
- 在测试集上反复试超参数，本质上泄漏了测试信息。
