1. Confusion Matrix 1. 混淆矩阵 — The Foundation of All Metrics

Every classification metric is derived from the confusion matrix, a table that cross-references predicted labels against true labels. For binary classification:

from sklearn.metrics import average_precision_score ap = average_precision_score(y_true, y_scores) # PRAUC 近似值 ``` > 注意：`average_precision_score` 计算的是 **Average Precision (AP)**，是 PRAUC 的一个保守近似（矩形插值），与逐点梯形插值的 PRAUC 略有差异。
Predicted Positive Predicted Negative
Actual Positive TP (True Positive) FN (False Negative / Type II Error)
Actual Negative FP (False Positive / Type I Error) TN (True Negative)
Mnemonic: The first letter tells you if the prediction was correct (T) or wrong (F). The second letter tells you what the model predicted (P or N). So FP = model predicted Positive but was wrong → actually Negative.

2. Accuracy

$$Accuracy = \frac{TP + TN}{TP + TN + FP + FN}$$
  • Meaning: The fraction of all samples that were correctly classified.
    • - **Precision**：预测为正的样本中，真正为正的比例 → **"查准率"** - **Recall**：实际为正的样本中，被正确找出来的比例 → **"查全率"** - 二者通常是 **trade-off** 关系：提高阈值 → Precision↑ Recall↓；降低阈值 → Precision↓ Recall↑
  • When to use: Balanced datasets where positive and negative classes are roughly equal.
  • Pitfall: Highly misleading on imbalanced data. If 99% of samples are negative, a model that always predicts negative achieves 99% accuracy but is useless for detecting positives.

3. Precision & Recall

3.1 Precision (查准率)

$$Precision = \frac{TP}{TP + FP}$$

Of all samples the model predicted as positive, what fraction are actually positive? High precision means few false alarms.

3.2 Recall (查全率 / Sensitivity / True Positive Rate)

$$Recall = \frac{TP}{TP + FN}$$

Of all samples that are actually positive, what fraction did the model correctly identify? High recall means few missed positives.

Precision–Recall Trade-off: Raising the classification threshold increases Precision but decreases Recall (fewer positives predicted, but more confident). Lowering the threshold has the opposite effect. They are inherently in tension.

4. F1-Score F1分数 & Fβ

4.1 F1-Score F1分数

$$F1 = 2 \cdot \frac{Precision \cdot Recall}{Precision + Recall} = \frac{2TP}{2TP + FP + FN}$$

The harmonic mean of Precision and Recall. Unlike the arithmetic mean, the harmonic mean penalizes extreme imbalances — a model with Precision=1.0 and Recall=0.0 gets F1=0, not 0.5. F1 is more reliable than Accuracy on imbalanced datasets.

- **Precision**：预测为正的样本中，真正为正的比例 → **"查准率"** - **Recall**：实际为正的样本中，被正确找出来的比例 → **"查全率"** - 二者通常是 **trade-off** 关系：提高阈值 → Precision↑ Recall↓；降低阈值 → Precision↓ Recall↑

4.2 Generalization: Fβ

$$F_\beta = (1+\beta^2) \cdot \frac{Precision \cdot Recall}{\beta^2 \cdot Precision + Recall}$$
  • \( \beta > 1 \): weights Recall more heavily — use when missing positives is costly (e.g., disease screening, security threats).
  • \( \beta < 1 \): weights Precision more heavily — use when false alarms are costly (e.g., spam filtering, recommendation).
  • \( \beta = 1 \): standard F1 (equal weight).

5. ROC Curve ROC曲线 & AUC-ROC

5.1 The ROC Curve ROC曲线

ROC (Receiver Operating Characteristic) plots TPR vs. FPR as the classification threshold sweeps from 0 to 1:

$$FPR = \frac{FP}{FP + TN}, \qquad TPR = Recall = \frac{TP}{TP + FN}$$
  • The diagonal (y = x) represents a random classifier.
    • 1. 混淆矩阵（Confusion Matrix）—— 一切的基础
  • The ideal point is (0, 1) — zero false positives, perfect recall.
    • 3. Precision（精确率）& Recall（召回率）
  • A curve above the diagonal indicates a useful classifier.

5.2 AUC-ROC (Area Under the ROC Curve ROC曲线)

  • Range: [0, 1]. AUC = 1 is a perfect classifier; AUC = 0.5 is random guessing.
    • - PR 曲线下面积。 - **相比 AUC-ROC 的优势**：在 **正负样本极度不均衡** 时（如正样本只占 0.1%），PRAUC 更有区分度。 - ROC 在不均衡时可能看起来很好（因为 TN 很多，FPR 看起来低），但 PR 曲线能暴露 Precision 的问题。 - **适用场景**：信息检索、推荐系统、欺诈检测等不均衡场景。
  • Probabilistic interpretation: The probability that the model ranks a randomly chosen positive sample higher than a randomly chosen negative sample.
    • | | 预测 Positive | 预测 Negative | |----------------|--------------|--------------| | 实际 Positive | TP（真阳性） | FN（假阴性） | | 实际 Negative | FP（假阳性） | TN（真阴性） |
  • Advantage: Invariant to class ratio — a useful property when comparing across datasets with different balance.

6. PR Curve PR曲线 & AUC-PR (PRAUC)

6.1 The PR Curve PR曲线

The Precision–Recall curve plots Recall (x-axis) vs. Precision (y-axis) as the threshold varies. Unlike ROC, it does not use TN — making it far more sensitive to class imbalance.

6.2 When to Prefer AUC-PR over AUC-ROC

Highly Imbalanced Datasets: When negatives vastly outnumber positives (e.g., fraud detection: 0.1% positives), AUC-ROC can be deceptively high because the huge number of TN keeps FPR small. AUC-PR exposes the actual difficulty of finding positives, because it only considers Precision and Recall.

Typical use cases: information retrieval, recommendation systems, fraud detection, anomaly detection, medical diagnosis with rare conditions.

6.3 Computing AUC-PR in Code

# sklearn provides Average Precision (AP), a conservative approximation of AUC-PR
from sklearn.metrics import average_precision_score, precision_recall_curve

ap = average_precision_score(y_true, y_scores)  # AP ≈ PRAUC (rectangular interpolation)
precisions, recalls, thresholds = precision_recall_curve(y_true, y_scores)

Note: average_precision_score computes Average Precision (AP) using rectangular interpolation, which is a conservative approximation of the true trapezoidal PRAUC.

7. Log Loss (Cross-Entropy Loss)

$$LogLoss = -\frac{1}{N}\sum_{i=1}^{N}\left[y_i \log(p_i) + (1-y_i)\log(1-p_i)\right]$$
  • \( y_i \): true label (0 or 1); \( p_i \): predicted probability of being positive.
  • Range: [0, +\infty). Lower is better.
    • 3. Precision（精确率）& Recall（召回率）
  • Key property: Severely penalizes confident but wrong predictions (e.g., predicting 0.99 when the true label is 0 yields a very large loss).
    • from sklearn.metrics import average_precision_score ap = average_precision_score(y_true, y_scores) # PRAUC 近似值 ``` > 注意：`average_precision_score` 计算的是 **Average Precision (AP)**，是 PRAUC 的一个保守近似（矩形插值），与逐点梯形插值的 PRAUC 略有差异。
  • Use when: The downstream task needs well-calibrated probabilities, not just hard class labels.

8. CLIP Score CLIP分数 (Image–Text Alignment Score)

8.1 Background

CLIP (Contrastive Language-Image Pre-training) is a vision–language model by OpenAI that maps images and text into a shared semantic embedding space. The CLIP Score leverages this shared space to measure how well an image and a text description align — without needing any human reference annotations.

CLIP（Contrastive Language-Image Pre-training）是 OpenAI 提出的视觉-语言预训练模型，能将图像和文本映射到同一语义空间。**CLIP Score 利用这一特性来衡量图文之间的语义一致性**。

8.2 How CLIP Score CLIP分数 Is Computed

  1. Pass the image through CLIP's Image Encoder (ViT-L/14) to obtain embedding \( I \in \mathbb{R}^{512} \).**步骤**： 1. 用 CLIP 的 Image Encoder 提取图像特征 $I \in \mathbb{R}^{512}$（ViT-L/14） 2. 用 CLIP 的 Text Encoder 提取文本特征 $T \in \mathbb{R}^{512}$（Transformer） 3. 计算 $I$ 和 $T$ 的余弦相似度，即为 CLIP Score
    2. **步骤**： 1. 用 CLIP 的 Image Encoder 提取图像特征 $I \in \mathbb{R}^{512}$（ViT-L/14） 2. 用 CLIP 的 Text Encoder 提取文本特征 $T \in \mathbb{R}^{512}$（Transformer） 3. 计算 $I$ 和 $T$ 的余弦相似度，即为 CLIP Score
  2. Pass the text through CLIP's Text Encoder (Transformer) to obtain embedding \( T \in \mathbb{R}^{512} \).**步骤**： 1. 用 CLIP 的 Image Encoder 提取图像特征 $I \in \mathbb{R}^{512}$（ViT-L/14） 2. 用 CLIP 的 Text Encoder 提取文本特征 $T \in \mathbb{R}^{512}$（Transformer） 3. 计算 $I$ 和 $T$ 的余弦相似度，即为 CLIP Score
    3. **步骤**： 1. 用 CLIP 的 Image Encoder 提取图像特征 $I \in \mathbb{R}^{512}$（ViT-L/14） 2. 用 CLIP 的 Text Encoder 提取文本特征 $T \in \mathbb{R}^{512}$（Transformer） 3. 计算 $I$ 和 $T$ 的余弦相似度，即为 CLIP Score
  3. Compute the cosine similarity between the two embeddings:
$$CLIP\text{-}Score = \cos(I, T) = \frac{I \cdot T}{\|I\| \; \|T\|}$$

8.3 Value Range

  • Raw cosine similarity: [−1, 1].
  • After sigmoid normalization: [0, 1]. Higher = better alignment.

8.4 Code Example 例子

import torch
from transformers import CLIPProcessor, CLIPModel

model = CLIPModel.from_pretrained("openai/clip-vit-large-patch14")
processor = CLIPProcessor.from_pretrained("openai/clip-vit-large-patch14")

inputs = processor(
    text=["a dog sitting on the grass"],
    images=dog_image,
    return_tensors="pt",
    padding=True
)
outputs = model(**inputs)

# CLIP Score = image-text cosine similarity (via logits)
clip_score = outputs.logits_per_image.softmax(dim=1)[0, 0].item()
print(f"CLIP Score: {clip_score:.4f}")

Core Distinction

CLIP Score is reference-free — it needs only a text prompt, no ground-truth images. This makes it uniquely valuable for evaluating generative models (text-to-image, image captioning) where collecting references is expensive or impossible.

8.6 Strengths & Limitations 局限性

  • Strengths: No human annotation required; captures high-level semantic alignment; language-agnostic to some degree.
  • Limitations: CLIP has known biases (prefers English, prefers photorealistic styles); cannot replace human subjective evaluation; insensitive to fine details (text rendering accuracy, number of fingers, spatial relations).
  • Variant: CLIP Score (Ref) uses a reference image for improved correlation with human judgments.
    • **步骤**： 1. 用 CLIP 的 Image Encoder 提取图像特征 $I \in \mathbb{R}^{512}$（ViT-L/14） 2. 用 CLIP 的 Text Encoder 提取文本特征 $T \in \mathbb{R}^{512}$（Transformer） 3. 计算 $I$ 和 $T$ 的余弦相似度，即为 CLIP Score

8.7 Applications 应用

  • Text-to-Image generation: Evaluate whether the generated image matches the input prompt.
    • - **文生图**（Text-to-Image）：评估生成图像与输入 prompt 的匹配度 - **图像描述**（Image Captioning）：评估生成的 caption 与图像的一致性 - **图像编辑**：评估编辑后图像与目标描述的对齐程度
  • Image Captioning: Evaluate whether the generated caption describes the image correctly.
    • - **文生图**（Text-to-Image）：评估生成图像与输入 prompt 的匹配度 - **图像描述**（Image Captioning）：评估生成的 caption 与图像的一致性 - **图像编辑**：评估编辑后图像与目标描述的对齐程度
  • Image Editing: Evaluate whether the edited image aligns with the target description.

10. Padding (填充)

10.1 Why Padding 填充 (Padding)?

When a convolution kernel slides over an image, edge pixels are covered fewer times than center pixels — boundary information is under-represented. Padding adds extra pixels (typically zeros) around the boundary so that:

  • Edge pixels receive full coverage, enjoying the same attention as center pixels.
  • Output size can be controlled — particularly, kept equal to input size.
    • **推导**：令 OutputSize = InputSize，代入公式：

10.2 The Three Padding 填充 (Padding) Cases

Padding Effect Output Size
P = 0 (Valid) No fill; edges lost; output shrinks \( Input - K + 1 \)
P = (K−1)/2 (Same) Output size = Input size \( Input \) (unchanged)
P > (K−1)/2 Over-padding; output grows \( > Input \)
Why \( 2P \) in the formula? Padding is added to both sides (top & bottom, left & right), so the effective dimension increase is \( 2P \).

11. Output Size Formula (必考)

$$W_{out} = \left\lfloor \frac{W_{in} + 2P - K}{S} \right\rfloor + 1$$
Parameter Meaning Typical Values
\( W_{in} \)Input spatial size (width or height)32, 224, etc.
\( K \)Kernel size3, 5, 7
\( P \)Padding (pixels added per side)0, 1, 2, 3
\( S \)Stride (step size)1, 2

11.1 Same Padding 填充 (Padding) Derivation

To keep output = input, set \( S = 1 \) and solve \( W_{out} = W_{in} \):

**推导**：令 OutputSize = InputSize，代入公式：
$$W_{in} = \frac{W_{in} + 2P - K}{1} + 1 \implies 2P = K - 1 \implies P = \frac{K-1}{2}$$

Common values: K=3 → P=1; K=5 → P=2; K=7 → P=3.

from sklearn.metrics import average_precision_score ap = average_precision_score(y_true, y_scores) # PRAUC 近似值 ``` > 注意：`average_precision_score` 计算的是 **Average Precision (AP)**，是 PRAUC 的一个保守近似（矩形插值），与逐点梯形插值的 PRAUC 略有差异。
Must-remember rule: Odd kernel + Stride=1 + P=(K−1)/2 → output size unchanged.

12. Receptive Field (感受野)

12.1 Definition 定义

The Receptive Field (RF) of a neuron is the region in the original input image that influences that neuron's activation. Deeper neurons have larger receptive fields, capturing more global context.

$$RF_1 = K_1 = 3 \qquad \text{(first layer sees } 3 \times 3 \text{ patch)}$$

12.2 Multi-Layer Accumulation Formula 公式

$$RF_{new} = RF_{old} + (K - 1) \times \prod_{i=1}^{l-1} S_i$$

where \( \prod_{i=1}^{l-1} S_i \) is the product of all strides of layers before the current layer.

- **Precision**：预测为正的样本中，真正为正的比例 → **"查准率"** - **Recall**：实际为正的样本中，被正确找出来的比例 → **"查全率"** - 二者通常是 **trade-off** 关系：提高阈值 → Precision↑ Recall↓；降低阈值 → Precision↓ Recall↑

Why Receptive Field 感受野 Matters

Shallow layers (small RF) capture local textures and edges. Deep layers (large RF) capture global semantics and shapes. Stacking multiple small kernels (e.g., three 3×3) achieves the same RF as one large kernel (7×7) but with fewer parameters and more non-linear activations.

13. Downsampling (下采样)

13.1 Why Downsample?

  1. Reduce computation: Smaller feature maps = fewer operations in subsequent layers.
    2. **定义**：FLoating-point OPerations，即模型进行一次前向推理所需的**浮点运算总次数**，决定了模型的**计算开销/推理速度**。
  2. Increase receptive field: Without adding depth, spatially smaller maps let later layers "see" a larger input region.
  3. Add translation invariance: Small positional shifts are absorbed by the coarser resolution.

13.2 Three Downsampling Methods

Method Description Learnable?
Stride > 1 Conv Stride=2 convolution skips positions Yes (weights learned)
Pooling Max or Average over local windows No (fixed operation)
Global Average Pooling (GAP) Average each channel to 1×1, then flatten No

14. Pooling 池化

14.1 Max Pooling 池化 — Most Common

Takes the maximum value in each local window, retaining the strongest activation:

Input 4×4:               Max Pooling 2×2, Stride=2:

1  3  2  1               3  4
4  6  3  2       →       8  6
2  8  5  1               Output 2×2
3  4  1  0

14.2 Average Pooling 池化

Takes the mean of each local window, retaining smoother, global information:

Input 4×4:               Avg Pooling 2×2, Stride=2:

1  3  2  1               3.50  2.00
4  6  3  2       →       4.25  1.75
2  8  5  1               Output 2×2
3  4  1  0

14.3 Max Pooling 池化 vs. Average Pooling

Aspect Max Pooling Average Pooling
What it retainsStrongest activation (dominant feature)Smoothed overall response
Typical useHidden layers (classification)Network end (Global Average Pooling)
Sensitivity to noiseMore robust (picks max)Can be diluted by many zeros

14.4 Pooling 池化 vs. Stride-2 Convolution

Comparison Pooling Stride-2 Conv
Learnable parametersNoneYes

Computation costLowerHigher
Modern trendStill widely usedResNet etc. replace some pooling with stride conv
Key rule: Pooling layers only change spatial dimensions (H×W), never the channel count (C). Changing C is the job of convolution layers (determined by the number of filters).

15. Parameters Calculation 参数量计算

15.1 Conv Layer Parameters

$$Params_{conv} = C_{out} \times (C_{in} \times K_h \times K_w + 1_{bias})$$

Intuition: Each output filter needs \( C_{in} \times K_h \times K_w \) weights to aggregate information across all input channels, plus 1 bias. There are \( C_{out} \) such filters.

Critical insight: Conv parameters are independent of spatial resolution (H×W) — only \( C_{in} \), \( C_{out} \), and \( K \) matter.

15.2 Fully Connected (FC) Layer Parameters

$$Params_{FC} = C_{in} \times C_{out} + C_{out}$$

FC layers typically have far more parameters than conv layers — this is precisely why modern architectures (GoogLeNet, ResNet) replace them with Global Average Pooling.

15.3 Worked Example 例子s

Single Conv Layer: Cin=3, Cout=64, K=3×3 (with bias)

$$Params = 64 \times (3 \times 3 \times 3 + 1) = 64 \times 28 = 1{,}792$$

Single Conv Layer: Cin=64, Cout=128, K=3×3 (with bias)

$$Params = 128 \times (64 \times 3 \times 3 + 1) = 128 \times 577 = 73{,}856$$

FC Layer: 512 → 10

$$Params = 512 \times 10 + 10 = 5{,}130$$

16. FLOPs Calculation 计算量 (FLOPs)

16.1 Definition 定义

FLOPs (FLoating-point OPerations, lowercase s) = total number of floating-point operations for one forward pass. It determines inference speed / computational cost.

**定义**：FLoating-point OPerations，即模型进行一次前向推理所需的**浮点运算总次数**，决定了模型的**计算开销/推理速度**。
FLOPs vs. FLOPS: FLOPs (lowercase s) = operation count (e.g., 1 GFLOPs = 10⁹ ops). FLOPS (uppercase S) = FLoating-point Operations Per Second — a hardware throughput metric (GPU compute power).

16.2 Conv Layer FLOPs Formula 公式

$$FLOPs_{conv} = 2 \times C_{out} \times C_{in} \times K_h \times K_w \times H_{out} \times W_{out}$$
  • Factor 2: 1 multiplication + 1 addition per element (one MAC = 2 FLOPs).
    • 其中： - 系数 **2** = 1 次乘法 + 1 次加法（一次 MAC = 2 FLOPs） - $H_{out} \times W_{out}$：输出特征图的空间尺寸
  • \( H_{out} \times W_{out} \): spatial size of the output feature map.$$OutputSize = \lfloor \frac{InputSize + 2 \times Padding - KernelSize}{Stride} \rfloor + 1$$
Key difference from Params: FLOPs depend on spatial resolution \( H \times W \). A model with few parameters can still be slow if it operates on large feature maps.

16.3 Worked Example 例子

Conv: Cin=64, Cout=128, K=3×3, input 28×28, P=1, S=1 → output 28×28

| Padding | 效果 | 输出尺寸 | |---------|------|---------| | **Padding=0**（Valid） | 不填充，边缘丢失，尺寸缩小 | $Input - Kernel + 1$ | | **Padding=(K-1)/2**（Same） | 填充使输出=输入 | $Input$（尺寸不变） | | **Padding > (K-1)/2** | 过度填充，尺寸反而变大 | $> Input$ |
$$FLOPs = 2 \times 128 \times 64 \times 3 \times 3 \times 28 \times 28 = 109{,}890{,}048 \approx 0.11 \text{ GFLOPs}$$

Layer Type FLOPs Formula Notes

FC (Linear) \( 2 \times C_{in} \times C_{out} \) Many params but usually only at network end
Max Pooling \( K^2 \times H_{out} \times W_{out} \) No learnable params, minimal ops
ReLU \( C \times H \times W \) Negligible
BatchNorm \( 4 \times C \times H \times W \) Usually not dominant

17. Params vs. FLOPs — Key Distinction 关键区别s

Aspect Params (参数量) FLOPs (计算量)
What it measures Storage / model size Inference speed / compute cost
Unit M (millions) GFLOPs / MFLOPs
Depends on H×W? No (only Cin, Cout, K) Yes (scales with spatial size)
Bottleneck Memory / GPU VRAM Inference latency

Params Few ≠ FLOPs Few

Depthwise Separable Convolution (MobileNet) dramatically reduces FLOPs, but the reduction in Params is proportionally smaller. A 1×1 convolution has tiny Params (kernel size = 1), but when channel counts are large, FLOPs can still be substantial.

**Params 少 ≠ FLOPs 少**： - 深度可分离卷积（Depthwise Separable Conv，MobileNet）大幅减少 FLOPs，但 Params 减少比例不同 - 1×1 卷积：Params 不大（核小），但通道数大时 FLOPs 仍可观
Exam strategy: Given a network architecture, you should be able to compute Params and FLOPs for each layer and explain the difference between the two.

18. Summary 8. 本讲总结

  1. Confusion matrix is the foundation: all metrics (Accuracy, Precision, Recall, F1) derive from TP, FP, TN, FN.
    2. - 横轴：**FPR**（False Positive Rate）$= \frac{FP}{FP + TN}$ - 纵轴：**TPR**（True Positive Rate）$= \frac{Recall} = \frac{TP}{TP + FN}$ - 通过不断改变分类阈值，绘制 (FPR, TPR) 点，连成曲线。
  2. Precision vs. Recall is a fundamental trade-off; Fβ lets you control which side matters more.
    3. - **Precision**：预测为正的样本中，真正为正的比例 → **"查准率"** - **Recall**：实际为正的样本中，被正确找出来的比例 → **"查全率"** - 二者通常是 **trade-off** 关系：提高阈值 → Precision↑ Recall↓；降低阈值 → Precision↓ Recall↑
  3. AUC-ROC measures ranking quality, invariant to class ratio. AUC-PR (PRAUC) is better for extreme class imbalance.
    4. - PR 曲线下面积。 - **相比 AUC-ROC 的优势**：在 **正负样本极度不均衡** 时（如正样本只占 0.1%），PRAUC 更有区分度。 - ROC 在不均衡时可能看起来很好（因为 TN 很多，FPR 看起来低），但 PR 曲线能暴露 Precision 的问题。 - **适用场景**：信息检索、推荐系统、欺诈检测等不均衡场景。
  4. Log Loss penalizes confident wrong predictions; use when calibrated probabilities are needed.
  5. CLIP Score is reference-free — evaluates image–text alignment using cosine similarity in CLIP's embedding space, uniquely suited for generative model evaluation.
    6. **步骤**： 1. 用 CLIP 的 Image Encoder 提取图像特征 $I \in \mathbb{R}^{512}$（ViT-L/14） 2. 用 CLIP 的 Text Encoder 提取文本特征 $T \in \mathbb{R}^{512}$（Transformer） 3. 计算 $I$ 和 $T$ 的余弦相似度，即为 CLIP Score
  6. Padding controls output size: P = (K−1)/2 with S=1 gives "Same" padding.
    7. **示例 1**：输入 32×32，Kernel=3，Padding=1，Stride=1（Same Padding） $$Output = \lfloor \frac{32 + 2 \times 1 - 3}{1} \rfloor + 1 = \lfloor 31 \rfloor + 1 = 32$$ → 尺寸不变
  7. Output size: \( W_{out} = \lfloor (W_{in} + 2P - K) / S \rfloor + 1 \).$$OutputSize = \lfloor \frac{InputSize + 2 \times Padding - KernelSize}{Stride} \rfloor + 1$$
    8. **示例 1**：输入 32×32，Kernel=3，Padding=1，Stride=1（Same Padding） $$Output = \lfloor \frac{32 + 2 \times 1 - 3}{1} \rfloor + 1 = \lfloor 31 \rfloor + 1 = 32$$ → 尺寸不变
  8. Receptive field grows with depth: \( RF_{new} = RF_{old} + (K-1) \times \prod S_i \). Stacking small kernels is more parameter-efficient than one large kernel.
    9. **示例 1**：输入 32×32，Kernel=3，Padding=1，Stride=1（Same Padding） $$Output = \lfloor \frac{32 + 2 \times 1 - 3}{1} \rfloor + 1 = \lfloor 31 \rfloor + 1 = 32$$ → 尺寸不变
  9. Downsampling reduces spatial size via stride conv, pooling, or GAP — each with different trade-offs.
    10. | 方式 | 说明 | 是否可学习 | |------|------|-----------| | **Stride > 1 的卷积** | 卷积步长为 2，直接跳着采样 | 是（卷积参数可学习） | | **池化（Pooling）** | 在局部区域内取最大值或平均值 | 否（固定操作） | | **全局平均池化（GAP）** | 每个通道取所有像素的平均值，输出 1×1 | 否 |
  10. Params = storage (independent of spatial size); FLOPs = compute cost (scales with spatial size). FC layers dominate Params; conv layers dominate FLOPs.
    11. **Params 少 ≠ FLOPs 少**： - 深度可分离卷积（Depthwise Separable Conv，MobileNet）大幅减少 FLOPs，但 Params 减少比例不同 - 1×1 卷积：Params 不大（核小），但通道数大时 FLOPs 仍可观
