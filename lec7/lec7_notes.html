1. Neural Network Foundations 1. 神经网络基础

1.1 Starting Point: Linear Classifiers

Linear classifiers (SVM, Softmax) use a score function:

线性分类器（如 SVM、Softmax）的得分函数：
$$s = Wx, \quad x \in \mathbb{R}^d \to s \in \mathbb{R}^K$$

Problem: Linear classifiers can only draw hyperplanes. They cannot handle non-linearly separable data (e.g., red and blue points arranged in concentric rings).

2. Activation Functions 2. 激活函数

Activation functions introduce non-linearity, allowing networks to learn complex manifolds.

Function Formula Derivative Pros / Cons
Sigmoid \( \sigma(x) = \frac{1}{1+e^{-x}} \) \( \sigma(x)(1-\sigma(x)) \) Output (0,1). Vanishing gradients for large \( |x| \).
Tanh \( \tanh(x) = \frac{e^x - e^{-x}}{e^x + e^{-x}} \) \( 1 - \tanh^2(x) \) Zero-centered. Better than sigmoid, but still saturates.
ReLU \( \max(0, x) \) \( 1 \) if \( x > 0 \), else \( 0 \) Fast, no saturation in positive region. “Dead ReLU” issue.
Leaky ReLU \( \max(\alpha x, x),\; \alpha \ll 1 \) \( 1 \) if \( x > 0 \), else \( \alpha \) Fixes dead ReLU by allowing small negative slope.
ELU Similar to Leaky ReLU but smoother Smooth near zero Output mean closer to 0. Helps convergence.
Maxout \( \max(w_1^T x + b_1,\; w_2^T x + b_2) \) Piecewise linear Learns the activation function itself. Doubles parameters.
Practice: ReLU works best on the vast majority of tasks. Use ReLU as the default choice.

4. Biological Inspiration

Neural networks draw inspiration from biological neurons:

  • Dendrites: receive signals → corresponds to input \( x \)
    • $$\text{Stochastic Depth: } y = \begin{cases} \mathcal{F}(x, W_l) + x & \text{以概率 } 1 - p_l \\ x & \text{以概率 } p_l \end{cases}$$
  • Cell Body: integrates signals → corresponds to summation \( w^T x + b \)
    • - **树突（Dendrite）**：接收信号 → 对应输入 $x$ - **胞体（Cell body）**：整合信号 → 对应求和 $w^T x + b$ - **轴突（Axon）**：传递信号 → 对应输出 - **突触（Synapse）**：连接权重 → 对应权重 $w$ - Sigmoid 激活函数本身就是对生物神经元"全或无"发放规律的建模
  • Axon: transmits signals → corresponds to output
  • Synapse: connection strength → corresponds to weights \( w \)
    • - **树突（Dendrite）**：接收信号 → 对应输入 $x$ - **胞体（Cell body）**：整合信号 → 对应求和 $w^T x + b$ - **轴突（Axon）**：传递信号 → 对应输出 - **突触（Synapse）**：连接权重 → 对应权重 $w$ - Sigmoid 激活函数本身就是对生物神经元"全或无"发放规律的建模
  • The Sigmoid activation function itself models the “all-or-none” firing law of biological neurons

Key difference: Biological neurons have complex, random connections, while artificial neurons are organized in regular layered structures for computational efficiency. However, research has shown that randomly connected networks can also work.

6.2 Chain Rule

The core mathematical tool of backpropagation:

$$\frac{\partial f}{\partial x} = \underbrace{\frac{\partial f}{\partial q}}_{\text{upstream gradient}} \times \underbrace{\frac{\partial q}{\partial x}}_{\text{local gradient}}$$

Each node only needs to know two things:

Stochastic Depth（Huang et al., 2016 "Deep Networks with Stochastic Depth"）的做法非常直接：
  1. Upstream gradient: gradient flowing from downstream
    2. 每个节点只需要知道两件事： 1. **上游梯度**（upstream gradient）：从后面传过来的梯度 2. **局部梯度**（local gradient）：该节点自身运算的导数
  2. Local gradient: the derivative of that node’s own operation
    3. 每个节点只需要知道两件事： 1. **上游梯度**（upstream gradient）：从后面传过来的梯度 2. **局部梯度**（local gradient）：该节点自身运算的导数

Their product = downstream gradient, passed to the previous node.

两者相乘 = **下游梯度**（downstream gradient），传给前面的节点。

7. Computation Graphs & Automatic Differentiation

7.1 What Is a Computation Graph?

A computation graph is a directed acyclic graph (DAG) whose nodes represent variables or operations, and whose edges represent data dependencies. Every mathematical expression can be decomposed into primitive operations (additions, multiplications, unary functions).

Why computation graphs matter: Modern frameworks (TensorFlow, PyTorch, JAX) represent every model as a computation graph, enabling automatic differentiation: the framework computes exact gradients for every parameter without the user writing derivative code by hand.

7.2 Forward Pass

During the forward pass, the graph is evaluated bottom-up: each node computes its value from its inputs and caches the result for the backward pass.

7.3 Backward Pass

The backward pass traverses the graph in reverse topological order. At each node, the local gradient is multiplied with the upstream gradient via the chain rule. All parameter gradients are computed in a single sweep with cost proportional to one forward pass.

$$\frac{\partial L}{\partial x_i} = \sum_{j \in \text{children}(i)} \frac{\partial L}{\partial y_j} \cdot \frac{\partial y_j}{\partial x_i}$$

7.4 Worked Example 例子: \( f(x, y, z) = (x + y)z \)

Introduce intermediate variable \( q = x + y \), so \( f = qz \).

**前向传播**： - $q = x + y = -2 + 5 = 3$ - $f = q \cdot z = 3 \times (-4) = -12$

Step 1 — Forward Pass

Let \( x = -2 \), \( y = 5 \), \( z = -4 \).

以 $f(x, y, z) = (x + y) \cdot z$ 为例：
$$q = x + y = -2 + 5 = 3, \quad f = q \cdot z = 3 \times (-4) = -12$$

Step 2 — Local Gradients

$$\frac{\partial q}{\partial x} = 1, \quad \frac{\partial q}{\partial y} = 1, \quad \frac{\partial f}{\partial q} = z = -4, \quad \frac{\partial f}{\partial z} = q = 3$$

Step 3 — Backward Pass (Chain Rule)

$$\frac{\partial f}{\partial z} = q = 3$$ $$\frac{\partial f}{\partial x} = \frac{\partial f}{\partial q} \cdot \frac{\partial q}{\partial x} = (-4)(1) = -4$$ $$\frac{\partial f}{\partial y} = \frac{\partial f}{\partial q} \cdot \frac{\partial q}{\partial y} = (-4)(1) = -4$$
Result: \( \nabla f = (-4,\; -4,\; 3) \) at \( (-2, 5, -4) \). The backward pass required only one traversal from output to inputs — the power of reverse-mode autodiff.

7.5 Why Computation Graphs Enable Efficient Gradients

  • Cost proportional to the forward pass: \( O(1) \times \) cost of one forward evaluation (“cheap gradient principle”).
  • Automatic: Framework handles all bookkeeping (define-by-run or define-and-run).
    • 1. **病态条件数（Poor Conditioning）**：损失函数在不同方向上变化速度不同，Hessian 矩阵的最大奇异值与最小奇异值的比值很大。导致在浅方向（shallow）进展缓慢，在陡方向（steep）波动剧烈。 2. **局部最小值和鞍点**：梯度为零时，SGD 会停滞。高维空间中鞍点比局部最小值更常见。 3. **梯度噪声**：mini-batch 的梯度只是真实梯度的估计值，包含采样噪声。
  • Composable: Any differentiable operation can be added; only its local gradient is needed.
  • Scalable: Works for networks with millions of parameters.

11.1 L1 vs L2 Weight Preference

13. Stochastic Depth (DropPath) — Detailed Analysis

13.1 Motivation: Redundancy in Deep Networks

After ResNet introduced residual connections, networks can be stacked to hundreds or even thousands of layers. But a key observation is:

| 论文 | 名称 | 应用场景 | |---|---|---| | Huang et al., 2016 | Stochastic Depth | ResNet | | Touvron et al., 2021 (DeiT) | DropPath | Vision Transformer (ViT) | | Liu et al., 2021 (Swin) | DropPath | Swin Transformer |
Not all layers in a deep network are equally important.
  • Shallow layers extract low-level features (edges, textures) — nearly all samples need them
    • 1. **病态条件数（Poor Conditioning）**：损失函数在不同方向上变化速度不同，Hessian 矩阵的最大奇异值与最小奇异值的比值很大。导致在浅方向（shallow）进展缓慢，在陡方向（steep）波动剧烈。 2. **局部最小值和鞍点**：梯度为零时，SGD 会停滞。高维空间中鞍点比局部最小值更常见。 3. **梯度噪声**：mini-batch 的梯度只是真实梯度的估计值，包含采样噪声。
  • Deep layers extract high-level semantics (object parts, complex patterns) — not every sample needs all deep layers
  • During training mid-stage, gradients through many layers become very weak for early layers

Essence: The deeper the network, the greater the redundancy, the higher the overfitting risk, and the harder the training.

13.2 Core Idea

Stochastic Depth (Huang et al., 2016) extends Dropout from individual neurons to entire residual blocks. During training, each residual block is either executed normally or bypassed (replaced by the identity shortcut) according to a Bernoulli random variable.

| 论文 | 名称 | 应用场景 | |---|---|---| | Huang et al., 2016 | Stochastic Depth | ResNet | | Touvron et al., 2021 (DeiT) | DropPath | Vision Transformer (ViT) | | Liu et al., 2021 (Swin) | DropPath | Swin Transformer |
$$\text{Normal: } y = \mathcal{F}(x, W_l) + x$$ $$\text{Stochastic Depth: } y = \begin{cases} \mathcal{F}(x, W_l) + x & \text{with probability } 1 - p_l \\ x & \text{with probability } p_l \end{cases}$$

13.3 Linear Decay Schedule

The survival probability uses a linear decay from first to last layer:

| 方法 | 使用位置 | 作用 | |---|---|---| | **Dropout** | Attention 权重后、FFN 中间层、Embedding 层、输出层前 | 最主要的正则化手段，默认 $p=0.1$ | | **Layer Normalization** | 每个子层之后 | 训练稳定性 + 隐式正则化 | | **Label Smoothing** | 训练标签 | 不让模型 100% 自信，提高泛化 | | **Weight Decay (L2)** | AdamW 中 | 对权重施加 L2 惩罚 | | **DropPath / Stochastic Depth** | 深层 Transformer（ViT、BERT） | 随机丢弃整条残差分支 |
$$p_l = 1 - \frac{l}{L}(1 - p_L), \quad p_L = 0.5$$

Intuition:

Shallow (l ≈ 1):  p_l ≈ 0   → almost never drop (shallow layers are critical)
Deep    (l ≈ L):  p_l ≈ 0.5 → high drop probability (deep layers are redundant)

Shallow layers are “infrastructure”; deep layers are “expert layers.” Infrastructure cannot be shut down randomly, but experts can take turns.

13.4 Test-Time Handling

At test time, no layers are dropped (otherwise output is non-deterministic). Use all residual blocks with full weights (similar to how Dropout is disabled at test time).

13.5 Four Effects of Stochastic Depth

  1. Regularization: Each training step uses a random sub-network, preventing the entire deep network from co-overfitting.
  2. Implicit Ensemble: Different mini-batches use different sub-networks. The final full network is equivalent to an implicit ensemble of all sub-networks — similar to Bagging without training multiple models.
    3. Dinh et al. (2017) 的反例论文 "Sharp Minima Can Generalize For Deep Networks" 证明：通过简单的参数重参数化（如 $W \to W/2$），可以找到一个"尖锐"的最优解和一个"平坦"的最优解，它们对应完全相同的网络函数、完全相同的测试误差。尖锐与否取决于**参数化的选择**，而不是泛化能力的本质。
  3. Reduced Training Time: Deep layers have 50% probability of being skipped, meaning forward/backward computation is roughly halved. Training time is reduced by ~25-50%.
  4. Alleviated Vanishing Gradients: Because shallow layers are often directly connected to the output through identity shortcuts (skipped intermediate layers), gradients can flow more directly back to shallow layers.

14. Pooling 池化: Max, Average, Global Average, Fractional

14.1 What Is Pooling 池化?

Pooling is a downsampling operation that reduces the spatial size of feature maps (height × width) while preserving important information.

Analogy: shrinking a high-resolution image into a thumbnail — you can’t see every pixel, but you can still recognize what’s in the photo.

**奥卡姆剃刀**（Occam's Razor）：Among multiple competing hypotheses, the simplest is the best.

Why pooling? Three reasons:

| 方法 | 核心机制 | |---|---| | Dropout | 训练时随机丢弃神经元 | | Batch Normalization | 利用 mini-batch 统计量的随机噪声 | | Stochastic Depth | 随机丢弃整条残差分支 | | Fractional Pooling | 随机化池化操作 |
  1. Reduce computation: halving feature map size drastically reduces parameters and FLOPs in subsequent layers
  2. Increase receptive field: after pooling, one output pixel corresponds to a larger input region
    3. | 方法 | 核心机制 | |---|---| | Dropout | 训练时随机丢弃神经元 | | Batch Normalization | 利用 mini-batch 统计量的随机噪声 | | Stochastic Depth | 随机丢弃整条残差分支 | | Fractional Pooling | 随机化池化操作 |
  3. Provide translation invariance: even if an object shifts a few pixels, pooled features change minimally
    4. if random.random() < p: return x # 直接返回输入，block 被完全跳过 else: return block(x) + x

14.2 Max Pooling 池化

$$y_{i,j} = \max_{(m,n) \in \text{window}(i,j)} x_{m,n}$$

Retains the most salient feature (brightest point, strongest edge) in each window.

14.3 Average Pooling 池化

$$y_{i,j} = \frac{1}{|\text{window}|} \sum_{(m,n) \in \text{window}(i,j)} x_{m,n}$$

Retains overall statistical information of the window.

14.4 Global Average Pooling 池化 (GAP)

$$y_c = \frac{1}{H \times W} \sum_{i=1}^{H} \sum_{j=1}^{W} x_{i,j,c}$$

In modern networks (e.g., ResNet), GAP replaces the final FC layer: \( H \times W \times C \to 1 \times 1 \times C \), drastically reducing parameters.

在现代网络（如 ResNet）中，GAP 常用来替代最后的全连接层：$H \times W \times C \to 1 \times 1 \times C$，大幅减少参数。

Traditional:  Conv(stride=1) → ReLU → Pool(stride=2)    ← two steps

Advantage: the downsampling manner can be learned (convolution kernel parameters are trainable), rather than fixed max or mean. However, pooling is still useful when feature maps need to shrink dramatically (e.g., \( 224 \times 224 \to 7 \times 7 \)).

14.7 Fractional Max Pooling 池化

Fractional Max Pooling (FMP) generalizes standard max pooling by using non-integer pooling ratios. Instead of fixed non-overlapping windows, FMP generates pooling regions using a random spatial grid with slightly non-uniform boundaries.

  • Non-deterministic pooling: the random grid changes each iteration, so the network cannot overfit to a fixed spatial structure
  • Smoother size reduction: non-integer ratios allow more gradual spatial downsampling
  • Implicit ensemble effect: over many iterations, the network sees many different pooling configurations

14.8 Random Pooling 池化

Same window as Max Pooling, but instead of taking the max, sample proportionally to values:

1. **非线性激活函数**是神经网络拥有强大表示能力的关键——没有它，多层网络退化为线性分类器 2. **反向传播**的本质是沿计算图递归应用链式法则，每个节点只需知道上游梯度和局部梯度 3. **正则化**不仅是加惩罚项，还包括 Dropout、BN、Stochastic Depth、Fractional Pooling 等结构层面的技巧 4. **优化器**从 SGD → Momentum → RMSProp → Adam 的演进，核心趋势是让学习率越来越"自适应" 5. **池化**是卷积网络中的核心下采样操作，Max Pooling 取最显著特征，Average Pooling 保留整体信息，现代趋势是用 stride 卷积替代显式池化 6. BN 的正则化效果是一个**副产品**（mini-batch 随机噪声），Transformer 不用 BN 而用 LayerNorm 7. **Stochastic Depth** 随机丢弃整条残差分支，既是正则化（隐式集成）也是加速手段（减少 25-50% 训练时间），浅层不丢、深层多丢 8. **Fractional Pooling** 在池化操作中引入空间随机性，通过随机切分位置打破确定性，使模型学到对空间位置不敏感的特征 9. **"Batch"有两层含义**：梯度下降中控制"一次更新看多少数据"（mini-batch GD），归一化中控制"按什么维度统计"（BN vs LN vs GN）。BN/LN/GN/IN/RMSNorm 构成一个统一谱系，区别只在于统计量跨哪些维度计算 10. **Hessian 矩阵**描述损失面的二阶曲率，条件数 $\kappa = \lambda_{\max}/\lambda_{\min}$ 决定优化难度，现代优化器（RMSProp、Adam）本质上是在隐式近似 Newton 法 $H^{-1} \nabla F$ 11. **Robbins-Monro 条件** $\sum \alpha_t = \infty, \sum \alpha_t^2 < \infty$ 保证 SGD 精确收敛；实践中用常数学习率是因为深度学习的非凸性使噪声有利（逃逸鞍点、倾向平坦最优解），常数学习率 SGD 收敛到最优解附近半径为 $O(\alpha \sigma^2 / 2\mu)$ 的邻域 12. **Dvoretzky 定理**是 Robbins-Monro 的一般化：核心递推 $d_{t+1} \leq (1+\tilde{a}_t)d_t + \tilde{b}_t$ 中，$\sum\tilde{a}_t < \infty$ 控制噪声累积，$\sum c_t = \infty$ 保证走到目标，通过无穷乘积引理和鞅收敛定理完成严格证明 
def random_pool(region):
    weights = softmax(region)  # convert values to probabilities
    return random_sample(region, weights)  # high values more likely, but not guaranteed

15. Optimization: Gradient Computation & GD Variants

15.1 Numerical vs Analytic Gradient

Numerical Gradient:

$$\frac{\partial L}{\partial w_i} \approx \frac{L(w_i + h) - L(w_i - h)}{2h}$$
  • Pro: easy to implement
    • 8.2 随机近似算法（Stochastic Approximation）的收敛性证明
  • Con: approximate, slow (must iterate over all dimensions)
    • 1. **病态条件数（Poor Conditioning）**：损失函数在不同方向上变化速度不同，Hessian 矩阵的最大奇异值与最小奇异值的比值很大。导致在浅方向（shallow）进展缓慢，在陡方向（steep）波动剧烈。 2. **局部最小值和鞍点**：梯度为零时，SGD 会停滞。高维空间中鞍点比局部最小值更常见。 3. **梯度噪声**：mini-batch 的梯度只是真实梯度的估计值，包含采样噪声。

Analytic Gradient: via chain rule.

**解析梯度**（Analytic Gradient）：
  • Pro: exact, fast
    • Adam = **Momentum + RMSProp**，目前最常用的深度学习优化器。
  • Con: error-prone
    • - **2-layer Neural Net** = 1 个隐藏层（输入层不算，输出层算一层） - **3-layer Neural Net** = 2 个隐藏层 - 通用术语：**全连接层（Fully-connected layer）**或**多层感知机（MLP）**
Practice: Always use analytic gradients, but verify correctness with numerical gradients (Gradient Checking).

16. SGD Problems & Momentum

16.1 SGD Problems

  1. Poor Conditioning: The loss function changes at different rates in different directions. The Hessian’s max/min singular value ratio is large. Slow progress in shallow directions, oscillation in steep directions.
    2. 1. **病态条件数（Poor Conditioning）**：损失函数在不同方向上变化速度不同，Hessian 矩阵的最大奇异值与最小奇异值的比值很大。导致在浅方向（shallow）进展缓慢，在陡方向（steep）波动剧烈。 2. **局部最小值和鞍点**：梯度为零时，SGD 会停滞。高维空间中鞍点比局部最小值更常见。 3. **梯度噪声**：mini-batch 的梯度只是真实梯度的估计值，包含采样噪声。
  2. Local Minima & Saddle Points: When the gradient is zero, SGD stalls. In high-dimensional spaces, saddle points are more common than local minima.
  3. Gradient Noise: Mini-batch gradients are only estimates of the true gradient, containing sampling noise.
    4. **策略三：Mini-batch Gradient Descent（小批梯度下降）——实践中最常用**

16.2 SGD + Momentum

Core idea: simulate physical “inertia” by building a velocity variable to smooth gradient updates.

$$v_t = \rho \cdot v_{t-1} + \nabla L$$ $$W_{t+1} = W_t - \alpha \cdot v_t$$
  • \( \rho \) controls velocity smoothing, typically 0.9 or 0.99
  • Effect: faster convergence, reduced oscillation, helps escape local minima and saddle points
# Python implementation
v = 0
rho = 0.9
while True:
    dw = compute_gradient(W)
    v = rho * v + dw
    W -= learning_rate * v

17. RMSProp & Adam (Adaptive Moment Estimation)

17.1 RMSProp

Adaptive learning rate based on the moving average of squared gradients. Dampens steep directions and accelerates shallow ones.

$$v_t = \rho \cdot v_{t-1} + (1-\rho)(\nabla L)^2$$ $$W_{t+1} = W_t - \frac{\alpha}{\sqrt{v_t} + \epsilon} \nabla L$$
  • Large gradient direction (steep): \( v_t \) large, effective learning rate suppressed
    • while True: dw = compute_gradient(W) m = beta1 * m + (1 - beta1) * dw # 更新一阶矩 v = beta2 * v + (1 - beta2) * dw**2 # 更新二阶矩 m_hat = m / (1 - beta1**t) # 偏差校正 v_hat = v / (1 - beta2**t) # 偏差校正 W -= learning_rate * m_hat / (sqrt(v_hat) + eps) ```
  • Small gradient direction (flat): \( v_t \) small, effective learning rate boosted
    • while True: dw = compute_gradient(W) m = beta1 * m + (1 - beta1) * dw # 更新一阶矩 v = beta2 * v + (1 - beta2) * dw**2 # 更新二阶矩 m_hat = m / (1 - beta1**t) # 偏差校正 v_hat = v / (1 - beta2**t) # 偏差校正 W -= learning_rate * m_hat / (sqrt(v_hat) + eps) ```
  • Intuition: brake on steep slopes, accelerate on flat terrain
    • 1. **病态条件数（Poor Conditioning）**：损失函数在不同方向上变化速度不同，Hessian 矩阵的最大奇异值与最小奇异值的比值很大。导致在浅方向（shallow）进展缓慢，在陡方向（steep）波动剧烈。 2. **局部最小值和鞍点**：梯度为零时，SGD 会停滞。高维空间中鞍点比局部最小值更常见。 3. **梯度噪声**：mini-batch 的梯度只是真实梯度的估计值，包含采样噪声。

17.2 Adam

Adam = Momentum + RMSProp. The most commonly used deep learning optimizer. Uses both the first moment (mean = Momentum) and second moment (variance = RMSProp) of gradients.

Adam = **Momentum + RMSProp**，目前最常用的深度学习优化器。 
# Adam core steps
m = 0   # first moment (gradient mean)
v = 0   # second moment (gradient variance)
beta1, beta2, eps = 0.9, 0.999, 1e-8

while True:
    dw = compute_gradient(W)
    m = beta1 * m + (1 - beta1) * dw          # update first moment
    v = beta2 * v + (1 - beta2) * dw**2       # update second moment
    m_hat = m / (1 - beta1**t)                # bias correction
    v_hat = v / (1 - beta2**t)                # bias correction
    W -= learning_rate * m_hat / (sqrt(v_hat) + eps)
Note: AdamW is an improved version of Adam that decouples weight decay from the gradient update.

18. Learning Rate Scheduling

All optimizers have learning rate as a hyperparameter. There is no single “best” answer. Common scheduling strategies:

Strategy Description
Step decay Multiply LR by 0.1 at fixed epochs (e.g., 30, 60, 90)
Cosine annealing \( \eta_t = \frac{\eta_0}{2}(1 + \cos(\frac{\pi t}{T})) \)
Linear warmup First ~5000 steps: linearly increase LR from 0 to initial value
Inverse sqrt \( \eta_t = \eta_0 / \sqrt{t} \)
Rule of thumb: When batch size increases by \( N \times \), initial learning rate should also scale by \( N \).

19. Batch Normalization 批归一化 as Regularization

19.1 BN’s Original Design Goal

Batch Normalization was designed to accelerate training by normalizing each layer’s input distribution, alleviating Internal Covariate Shift and making optimization more stable.

Batch Normalization 的设计初衷是**加速训练**，通过归一化每层的输入分布来缓解**内部协变量偏移（Internal Covariate Shift）**，让优化更稳定。
$$\hat{x}_i = \frac{x_i - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}}, \quad y_i = \gamma \hat{x}_i + \beta$$

19.2 Why BN Also Acts as Regularization

Reason 1: Mini-batch introduces random noise

| 方法 | 核心机制 | |---|---| | Dropout | 训练时随机丢弃神经元 | | Batch Normalization | 利用 mini-batch 统计量的随机噪声 | | Stochastic Depth | 随机丢弃整条残差分支 | | Fractional Pooling | 随机化池化操作 |

\( \mu_B \) and \( \sigma_B \) are statistics of the current mini-batch. Each training step, the mini-batch containing a given sample is different, so normalization results vary each time. This is equivalent to adding random perturbation to each sample — the model cannot memorize any single sample’s exact value.

Reason 2: Analogy to Dropout

| 方法 | 核心机制 | |---|---| | Dropout | 训练时随机丢弃神经元 | | Batch Normalization | 利用 mini-batch 统计量的随机噪声 | | Stochastic Depth | 随机丢弃整条残差分支 | | Fractional Pooling | 随机化池化操作 |
Dropout Batch Normalization
Mechanism Randomly drop neurons Randomly perturb feature values
Training Network “shape” differs each time Feature “scale” differs each time
Testing Disable randomness Use running mean/var instead of mini-batch stats

Both inject randomness during training, remove randomness during testing.

Reason 3: Gradient noise helps find flat optima

The stochasticity of mini-batch statistics injects noise into gradients, forcing the model to search in a larger solution space region, tending to find flatter optima (better generalization).

19.3 BN Limitations 局限性 & LayerNorm

BN normalizes along the batch dimension, suitable for CV with fixed-size images. But in NLP:

**特点**： - 不依赖 batch size（每个样本独立归一化） - 训练和测试行为一致（不需要 running statistics） - NLP / Transformer 的标准选择 - 正则化效果比 BN 弱（因为没有 mini-batch 随机性）
  • Sequence lengths vary
    • | 任务 | 典型 Batch Size | 典型学习率 | 优化器 | 特殊处理 | |---|---|---|---|---| | 图像分类（ResNet） | 32~256 | 0.1~0.4 | SGD+Momentum | Step decay 或 cosine | | 目标检测（Faster R-CNN） | 2~16 | 0.01~0.02 | SGD+Momentum | 小 batch，不能用 BN | | 机器翻译（Transformer） | ~4096 tokens | 0.0003~0.001 | Adam | Inverse sqrt warmup | | BERT 预训练 | 256~1024 sequences | 2e-4~5e-4 | Adam | Linear warmup 10000 步 | | GPT-3 预训练 | 3.2M tokens | 6e-5 | Adam | Cosine decay, 375M token warmup | | LLaMA 预训练 | 4M tokens | 3e-4 | AdamW | Cosine decay, 2000 step warmup | | Stable Diffusion | 2048~4096 | 1e-4 | AdamW | Warmup 10000 步 |
  • Batch-dimension statistics are unstable
    • ``` N: 样本数量（batch size） C: 通道数（channel / feature dimension） H: 高度（spatial height） W: 宽度（spatial width） ```

Therefore Transformers use Layer Normalization:

因此 Transformer 使用 **Layer Normalization** 替代 Batch Normalization：
$$\hat{x}_i = \frac{x_i - \mu_L}{\sqrt{\sigma_L^2 + \epsilon}} \cdot \gamma + \beta$$

LayerNorm normalizes along the feature dimension per sample, independent of mini-batch. Its randomization is much smaller than BN, so its regularization effect is also much weaker.

23. Hessian Matrix & Loss Landscape Geometry

23.1 From Gradient to Hessian

Gradients tell us the first-order information (slope) — which direction is steepest. The Hessian provides second-order information (curvature) — how “curved” each direction is.

For a scalar function \( f: \mathbb{R}^d \to \mathbb{R} \):

**PyTorch 中的对应关系**： - PyTorch 的每个运算符（operator）都自动实现了 `forward()` 和 `backward()` - `autograd` 自动构建计算图并执行反向传播 - `forward`：计算结果，并缓存中间变量 - `backward`：应用链式法则计算梯度
$$H_{ij} = \frac{\partial^2 f}{\partial w_i \partial w_j}, \quad H \in \mathbb{R}^{d \times d}$$

The Hessian is symmetric (when \( f \) is twice continuously differentiable).

**奥卡姆剃刀**（Occam's Razor）：Among multiple competing hypotheses, the simplest is the best.

23.2 Second-Order Taylor Expansion

$$f(W + \Delta W) \approx f(W) + \nabla f(W)^T \Delta W + \frac{1}{2} \Delta W^T H \Delta W$$

The extra \( \frac{1}{2} \Delta W^T H \Delta W \) term is the curvature information:

$$f(W + \Delta W) \approx f(W) + \nabla f(W)^T \Delta W + \frac{1}{2} \Delta W^T H \Delta W \quad \text{（加上 Hessian，是二次近似）}$$
  • \( \Delta W^T H \Delta W > 0 \): function is “concave up” (bowl-shaped)
  • \( \Delta W^T H \Delta W < 0 \): function is “convex up” (ridge-shaped)

23.3 Eigenvalues & Loss Landscape Geometry

Eigenvalue decomposition: \( H = V \Lambda V^T \), where \( \Lambda = \text{diag}(\lambda_1, \dots, \lambda_d) \).

对 Hessian 做特征值分解：$H = V \Lambda V^T$，其中 $\Lambda = \text{diag}(\lambda_1, \lambda_2, \dots, \lambda_d)$。
Eigenvalue Curvature Intuition
\( \lambda_i > 0 \) Concave direction (bowl bottom) Moving along \( v_i \), function increases
\( \lambda_i < 0 \) Convex direction (ridge) Moving along \( v_i \), function decreases
\( \lambda_i = 0 \) Flat direction Function doesn’t change (saddle point)
\( \lambda_i \gg 1 \) Steep direction Function changes rapidly
\( \lambda_i \ll 1 \) Shallow direction Function changes slowly

Determining critical point type from eigenvalues:

  • All \( \lambda_i > 0 \): positive definite → local minimum (bowl)
    • 每个节点只需要知道两件事： 1. **上游梯度**（upstream gradient）：从后面传过来的梯度 2. **局部梯度**（local gradient）：该节点自身运算的导数
  • All \( \lambda_i < 0 \): negative definite → local maximum (hilltop)
    • 每个节点只需要知道两件事： 1. **上游梯度**（upstream gradient）：从后面传过来的梯度 2. **局部梯度**（local gradient）：该节点自身运算的导数
  • Mixed signs: indefinite → saddle point

Saddle points are far more common than local minima in high-dimensional spaces. In \( d \) dimensions, the probability that all \( d \) eigenvalues are positive is exponentially small.

23.4 Condition Number & Optimization Difficulty

$$\kappa(H) = \frac{\lambda_{\max}}{\lambda_{\min}}$$

The condition number measures “how elliptical” the loss surface is:

  • \( \kappa \approx 1 \): nearly spherical → gradient descent converges quickly from any direction
  • \( \kappa \gg 1 \): elongated elliptical contours → ill-conditioned, slow convergence

When \( \kappa \) is large:

**奥卡姆剃刀**（Occam's Razor）：Among multiple competing hypotheses, the simplest is the best.
