# 搜索算法基础

## 图的遍历

### 欧拉图与欧拉回路

**欧拉回路**：遍历图中每条边恰好一次的回路。

**欧拉回路存在的充要条件**：
- **无向图**：图是连通的（忽略孤立点），且所有顶点的度都是偶数
- **有向图**：图是强连通的（忽略孤立点），且每个顶点的出度等于入度

**欧拉路径**：遍历图中每条边恰好一次的路径（不必回到起点）。

**欧拉路径存在的充要条件**：
- **无向图**：图是连通的，且恰好有0个或2个奇度顶点
- **有向图**：图是强连通的，且最多有两个顶点不满足出度=入度（其中一个出度比入度大1，另一个入度比出度大1，其余顶点出度=入度）

---

### 哈密顿路径与哈密顿回路

**哈密顿回路**：经过图中每个顶点恰好一次的回路。

**哈密顿回路存在的充要条件**：
- **必要条件**：若图存在哈密顿回路，则对于图的任意非空真子集 $S$，有 $k(G-S) \leq |S|$（其中 $k(G-S)$ 表示移除 $S$ 后连通分量个数）
- **充分条件（Ore定理）**：对于任意两个不相邻的顶点 $u, v$，有 $deg(u) + deg(v) \geq n$，则图存在哈密顿回路
- **充分条件（Dirac定理）**：若 $n \geq 3$ 且每个顶点的度都 $\geq n/2$，则图存在哈密顿回路

**哈密顿路径**：经过图中每个顶点恰好一次的路径（不必回到起点）。

**哈密顿路径存在的充要条件**：
- 与哈密顿回路类似，但约束条件相对宽松
- 必要条件与哈密顿回路相同

> **注意**：哈密顿回路问题是NPC问题，目前没有多项式时间算法。

---

### 中国邮递员问题（Chinese Postman Problem）

**问题描述**：邮递员需要遍历每条街道至少一次，返回起点，求最短路线。

**问题分类**：
- **欧拉图**：若图中所有顶点度数为偶数，则存在欧拉回路，答案为所有边权之和
- **非欧拉图**：需要添加重复边使图变为欧拉图，使总添加权值最小

**解法步骤**：
1. 找出图中所有奇度顶点
2. 在奇度顶点之间找最短路径
3. 构造奇度顶点间的完全图（边权为最短路径长度）
4. 在完全图中找最小权完美匹配
5. 将匹配对之间的最短路径加边，转化为欧拉图
6. 在欧拉图上求欧拉回路

---

### 旅行商问题（Travelling Salesman Problem, TSP）

**问题描述**：推销员从起点出发，遍历所有城市恰好一次，返回起点，求最短路线。

**问题性质**：
- **NPC问题**：没有已知的多项式时间算法
- **特殊情形**：若城市间距离满足三角不等式，可使用近似算法

**近似算法**：
- **最近邻算法**：从某点出发，每次选择最近的未访问点
- **最小生成树算法**：基于MST的近似解，路径长度 $\leq 2 \times$ 最优解（满足三角不等式）
- **Christofides算法**：基于MST + 最小权匹配，路径长度 $\leq 1.5 \times$ 最优解（需满足三角不等式）

**求解方法**：
- 动态规划：$O(n^2 \cdot 2^n)$，适用于 $n \leq 20$
- 分支限界法：剪枝加速搜索
- 启发式算法：遗传算法、模拟退火、蚁群算法等

## 无信息搜索

无信息搜索指的是**不依赖启发式评估函数**的搜索方法，典型代表包括 DFS、BFS、迭代加深 DFS 和回溯搜索。它们都不预知答案在哪里，只是按不同规则系统地展开状态空间。

### 四种算法的统一理解

- **DFS**：优先沿一条路径向深处走，走不通再回退
- **BFS**：按层扩展，先处理深度小的节点
- **迭代加深 DFS（DFS-ID / IDDFS）**：多次执行深度受限 DFS，每次把深度上限加 1
- **回溯搜索**：本质上是“带约束检查和剪枝的 DFS”，重点是解空间搜索而不是图遍历

### 对比总表

| 算法 | 搜索方式 | 是否保证找到解 | 是否保证最优解 | 时间复杂度 | 空间复杂度 | 主要特点 |
|------|----------|----------------|----------------|------------|------------|---------|
| **DFS** | 一条路走到黑，后退再试 | 不一定（有限图通常可保证完备） | 不保证 | 图：$O(V+E)$；树：$O(b^m)$ | 图：$O(V)$；树：$O(m)$ 级别栈空间 | 实现简单，内存小，容易陷入深层无效分支 |
| **BFS** | 按层扩展 | 保证（有限分支、可达时） | 保证最短路径（单位代价） | 图：$O(V+E)$；树：$O(b^d)$ | 图：$O(V)$；树：$O(b^d)$ | 最短路能力强，但占内存大 |
| **IDDFS** | 反复做“深度受限 DFS” | 保证（有限分支、可达时） | 保证最短路径（单位代价） | $O(b^d)$ | $O(bd)$ | 兼顾 BFS 的完备性和 DFS 的低空间 |
| **回溯搜索** | DFS + 约束检查 + 剪枝 | 不一定，取决于剪枝与建模 | 不保证，除非配合限界策略 | 最坏 $O(b^d)$，实际常因剪枝显著下降 | $O(d)$ | 面向约束满足/组合搜索，核心是“尝试-撤销-剪枝” |

> 说明：上表里的 $b$ 表示分支因子，$d$ 表示目标深度，$m$ 表示最大深度。

### 各自详细说明

#### DFS - 深度优先搜索

**定义**：深度优先搜索是一种**图或树的遍历策略**，优先探索当前分支到底，再回溯到其他分支。

**核心特征**：
- **目标**：遍历结构中的节点
- **实现**：递归或显式栈
- **优点**：空间开销低，写法直接
- **缺点**：可能深入无效路径，不能保证最短路

**时间复杂度**：
- **图**：$O(V + E)$
- **树**：$O(b^m)$，其中 $m$ 是最大深度

**空间复杂度**：
- 递归/栈深度约为 $O(m)$
- 图搜索时还需记录访问标记，通常为 $O(V)$

#### BFS - 广度优先搜索

**定义**：广度优先搜索按层推进，先访问距离起点近的节点，再访问更远的节点。

**核心特征**：
- **目标**：按层遍历，优先浅层节点
- **实现**：队列
- **优点**：如果边权相同，第一次到达目标时就是最短路径
- **缺点**：队列可能非常大，内存消耗高

**时间复杂度**：
- **图**：$O(V + E)$
- **树**：$O(b^d)$，其中 $d$ 是目标深度

**空间复杂度**：
- **图**：$O(V)$
- **树**：$O(b^d)$，因为要保存整层节点

#### 迭代加深 DFS - DFS-ID / IDDFS

**定义**：迭代加深 DFS 是对 DFS 的重复调用，每次限制最大搜索深度为 0、1、2、...、$d$。

**核心特征**：
- **目标**：兼顾 BFS 的“找最浅解”和 DFS 的低空间
- **实现**：多轮深度受限 DFS
- **优点**：内存小，且能找到最短深度解
- **缺点**：会重复搜索浅层节点，存在一定重复开销

**时间复杂度**：
- **树**：$O(b^d)$
- 虽然有重复搜索，但重复主要发生在浅层，总体仍与 BFS 同阶

**空间复杂度**：
- $O(bd)$，通常可近似看作 $O(d)$ 级别的栈空间

#### 回溯搜索 - 约束问题求解范式

**定义**：回溯搜索是一种**算法设计范式**，用于在**隐式解空间树**中通过“选择 - 检查 - 撤销”来找解。

**核心特征**：
- **目标**：求解满足约束的解，而不是单纯遍历图
- **约束检查**：每步判断当前部分解是否可行
- **剪枝**：一旦发现无效分支，立即停止深入
- **应用**：N 皇后、数独、排列组合、CSP、背包搜索等

**时间复杂度**：
- **最坏情况**：$O(b^d)$
- **实际情况**：由剪枝强度决定，常远小于最坏情况

**空间复杂度**：
- $O(d)$，只需维护当前路径上的状态和递归栈

### 一句话区分

- **DFS**：先深后广，适合遍历
- **BFS**：先浅后深，适合最短路
- **IDDFS**：像 BFS 一样找最浅解，像 DFS 一样省内存
- **回溯搜索**：带约束和剪枝的 DFS，用来“找可行解”

### 关键结论

1. **DFS 和回溯搜索不是同一层面的概念**：DFS 是遍历策略，回溯搜索是求解范式。
2. **BFS 和 IDDFS 的核心优势是“最短解”**：在单位代价场景下，它们都能保证找到最浅层解。
3. **空间上 DFS / 回溯 / IDDFS 明显优于 BFS**：BFS 常因保存整层节点而内存爆炸。
4. **如果你关心“能不能尽快找到一个解”**：DFS 或回溯更直接。
5. **如果你关心“最短路径/最浅解”**：BFS 或 IDDFS 更合适。



### 动态规划

**马尔可夫性**：当前状态一旦确定，未来状态的演化只依赖于当前状态，而与更早的历史无关。也就是说，后续决策所需的全部信息都已经被压缩到了“状态”里。

**在动态规划中的意义**：
- 动态规划就是利用马尔可夫性，把一个复杂问题分解为若干子问题
- 只要“当前状态”能够完整描述未来决策所需的信息，就可以写出状态转移方程
- 如果状态设计不完整，就会失去马尔可夫性，导致无法正确递推

**典型表达**：
- 状态：$dp[i]$ 或 $dp[i][j]$
- 转移：$dp[i]$ 只由更小规模的状态推出
- 例如最短路径、背包问题、序列问题都常依赖这种“当前状态决定未来”的性质



## 启发式搜索

**启发式搜索**（Heuristic Search）是在搜索过程中利用问题特有的领域知识来加速解的搜索，通过**启发函数** $h(n)$ 估计从当前节点到目标的"距离"或"代价"。核心思想是：**不再全盲地搜索，而是有方向地指向目标**。

### 评估函数与启发函数的基础

启发式搜索的核心由两个关键函数组成：**evaluation function（评估函数）** 和 **heuristic function（启发函数）**。理解它们的区别与联系至关重要。

#### Heuristic Function（启发函数） $h(n)$

**定义**：启发函数是一个**对问题特定的估计函数**，用于估计从节点 $n$ 到目标状态的**预期代价**或**距离**。

$$
h(n) \approx \text{从 } n \text{ 到目标的最小代价}
$$

**关键特性**：

1. **与问题相关**：$h(n)$ 必须利用问题领域的知识
   - 路径规划：欧几里得距离、曼哈顿距离
   - 棋盘游戏：已到达位置数、错位数

2. **非负性**：$h(n) \geq 0$ 对所有节点成立

3. **目标特性**：$h(n_{\text{goal}}) = 0$（在目标处，预期代价为 0）

4. **不计算真实代价**：$h(n)$ 是估计，不是精确值

**可采纳性（Admissibility）**：

启发函数称为**可采纳的**，当且仅当：

$$
h(n) \leq h^*(n) \quad \forall n
$$

其中 $h^*(n)$ 是从 $n$ 到目标的**真实最优代价**。

**含义**：启发函数永不高估（underestimate）

**重要性**：
- 若启发可采纳，则使用该启发的 A* 算法必然找到最优解
- 可采纳启发的松弛（relaxation）原理：忽略问题的某些约束条件

**常见启发函数例子**：

| 问题 | 启发函数 | 是否可采纳 |
|------|---------|----------|
| 8数字谜题 | 曼哈顿距离：$\sum_i \|x_i - x_i^*\|$ | 是 |
| 路径规划（网格） | 曼哈顿距离：$\|x-x_g\| + \|y-y_g\|$ | 是 |
| 路径规划（平面） | 欧几里得距离：$\sqrt{(x-x_g)^2 + (y-y_g)^2}$ | 是 |
| TSP | 最小生成树代价（MST）| 是 |
| 块世界 | 移动块数 | 是 |
| $h \equiv 0$ | 零函数 | 是（但无用） |

**启发函数的质量**（用 $h_1$ 更好于 $h_2$ 表示）：

$$
h_2(n) \geq h_1(n) \geq h^*(n) \land h_1(n) \geq 0
$$

则 $h_2$ **支配**（dominates）$h_1$，使用 $h_2$ 会展开更少的节点。

#### Evaluation Function（评估函数） $f(n)$

**定义**：评估函数是用于**决定节点扩展顺序**的综合评分函数，在搜索中用于优先级队列的排序。

$$
f(n) = g(n) + h(n)
$$

其中：
- **$g(n)$**：从起点到节点 $n$ 的**实际已知代价**（已花费）
- **$h(n)$**：从节点 $n$ 到目标的**启发估计代价**（预期代价）
- **$f(n)$**：通过节点 $n$ 到目标的**总预计代价**

**物理含义**：

假设你在地图上导航：
- $g(n)$ = 已经走过的距离
- $h(n)$ = 估计还要走的距离
- $f(n)$ = 总的预计行程

**不同算法的评估函数变体**：

| 算法 | 评估函数 | 特点 |
|------|---------|------|
| **BFS** | $f(n) = \text{depth}(n)$ | 只考虑深度，无启发 |
| **Dijkstra/UCS** | $f(n) = g(n)$ | 只考虑已知代价 |
| **Greedy Best-First** | $f(n) = h(n)$ | 只考虑启发估计 |
| **A*** | $f(n) = g(n) + h(n)$ | 平衡已知和预期 |
| **Weighted A*** | $f(n) = g(n) + w \cdot h(n)$，$w > 1$ | 可调的权衡 |

**评估函数的使用方式**：

```python
# 伪代码
frontier = PriorityQueue()
while not frontier.is_empty():
    # 按 f(n) 值取出最优节点（f(n) 最小）
    n = frontier.pop()  # f(n) 最小
    
    if is_goal(n):
        return path
    
    for successor in expand(n):
        f_value = g(successor) + h(successor)
        frontier.push((f_value, successor))
```

#### h(n) 和 f(n) 的关系

| 角度 | h(n) | f(n) |
|------|------|------|
| **定义** | 从 n 到目标的代价估计 | 完整路径代价估计 |
| **层次** | 启发学部分 | 完整评估准则 |
| **影响范围** | 决定搜索"方向"（Greedy 用它朝向目标） | 决定搜索"优先级"（A* 用它平衡） |
| **单独使用** | Greedy Best-First Search | A* Search |
| **质量条件** | 可采纳性：$h(n) \leq h^*(n)$ | 实现最优性需要 $h(n) \leq h^*(n)$ |

#### 启发函数的三大性质：可采纳性、一致性、相容性

启发函数的质量取决于三个关键性质。理解它们的定义、区别与相互关系，对设计和使用启发函数至关重要。

##### 1. 可采纳性（Admissibility）

**定义**：启发函数 $h$ 称为**可采纳的**，当且仅当：

$$
h(n) \leq h^*(n) \quad \forall n
$$

其中 $h^*(n)$ 是从 $n$ 到目标的**真实最优代价**。

**含义**：启发函数永不高估（underestimate）

**重要性**：
- **保证最优性**：若启发可采纳，则使用该启发的 A* 算法必然找到最优解
- 这是 A* 算法最基本的要求

**例子**：
- 欧几里得距离在路径规划中可采纳（直线距离≤任何真实路径）
- $h(n) = 0$ 总是可采纳的（但完全无用）
- 曼哈顿距离在网格路径规划中可采纳

**反例**：
- 若估计距离超过真实最短路径，则不可采纳
- 比如估计"北京到上海的直线距离为 2000 km"，但实际最短路径为 1200 km，就违反可采纳性

##### 2. 一致性 / 相容性（Consistency / Monotonicity）

**定义**：启发函数 $h$ 称为**一致的**（consistent）或**相容的**（compatible），若对所有边 $(n, n')$ 满足：

$$
h(n) \leq c(n, n') + h(n')
$$

其中 $c(n, n')$ 是从 $n$ 到 $n'$ 的边权。

**物理理解**：
- 这个条件就是**三角不等式**
- "从 $n$ 到目标的估计代价"不应超过"$n$ 到 $n'$ 的实际代价 + $n'$ 到目标的估计代价"

**直观例子**：

在导航中
- $h(n)$ = 当前位置到目标的直线距离
- $c(n, n')$ = 从 $n$ 走到邻近城市 $n'$ 的实际距离
- $h(n')$ = 从 $n'$ 到目标的直线距离

三角不等式保证：当前估计距离 ≤ 走一步的实际距离 + 下一个位置的估计距离。

**一致性的优势**：
- 若 $h$ 一致，则 A* **首次扩展节点时该节点已是最优**
- 无需回溯重新打开已扩展的节点
- 时间复杂度更优，通常为 $O(d)$ 而不是与节点集大小相关

**一致性在代码中的影响**：

```python
# 若 h 不一致，A* 可能需要重新打开节点
if neighbor not in closed_set or new_g[neighbor] < g[neighbor]:
    # 需要重新评估已关闭的节点
    g[neighbor] = new_g
    f[neighbor] = new_g + h[neighbor]
    frontier.push(neighbor)

# 若 h 一致，不需要重新打开
if neighbor not in closed_set:
    g[neighbor] = new_g
    frontier.push((f(neighbor), neighbor))
```

##### 3. 可采纳性、一致性、相容性的关系

**关键结论：一致性 $\Rightarrow$ 可采纳性**

**证明**：

假设 $h$ 一致，取任意节点 $n$，考虑从 $n$ 经过最优路径到目标的路径：

$$
n = n_0 \xrightarrow{c_1} n_1 \xrightarrow{c_2} n_2 \cdots \xrightarrow{c_k} n_k = \text{goal}
$$

由一致性反复应用：

$$
h(n_0) \leq c_1 + h(n_1) \leq c_1 + c_2 + h(n_2) \leq \cdots \leq c_1 + c_2 + \cdots + c_k + h(\text{goal})
$$

因为 $h(\text{goal}) = 0$（目标处无需代价），所以：

$$
h(n) = h(n_0) \leq c_1 + c_2 + \cdots + c_k = h^*(n)
$$

这正是可采纳性的定义。

**反向不成立：可采纳性 ⇏ 一致性**

**反例**：考虑网格路径规划

```
假设网格中：
  目标 G 在 (2, 2)
  当前 A 在 (0, 0)
  邻接 B 在 (1, 0)

h_A（到目标的欧距离）= sqrt(8) ≈ 2.83
h_B（到目标的欧距离）= sqrt(5) ≈ 2.24
c(A, B)（A 到 B 的距离）= 1

检查一致性：h_A ≤ c(A,B) + h_B 吗？
2.83 ≤ 1 + 2.24 = 3.24 ✓（一致）

但这只是特例。如果修改启发函数：
h'_A = 2.5（故意定得低一点，但仍≤真实代价 2.83）
h'_B = 2.24

检查一致性：h'_A ≤ c(A,B) + h'_B 吗？
2.5 ≤ 1 + 2.24 = 3.24 ✓（仍一致）

但若：
h''_A = 3.5（违反可采纳性）
```

实际上，一个"可采纳但不一致"的例子是：

设有路径 A → B → G，其中
- $c(A, B) = 1$
- $c(B, G) = 1$
- $h^*(A) = 2$（真实最优）
- $h^*(B) = 1$（真实最优）

定义非一致启发：
- $h(A) = 1.5$（可采纳）
- $h(B) = 1$（可采纳）

检查一致性：$h(A) \leq c(A,B) + h(B)$？
$1.5 \leq 1 + 1 = 2$ ✓（一致）

要构造非一致的可采纳启发，需要更复杂的图结构。实际上在**有向无环图（DAG）**或特殊拓扑结构下，可以构造出可采纳但非一致的启发。

##### 三大性质的对比总表

| 性质 | 可采纳性 | 一致性/相容性 |
|------|---------|--------------|
| **定义** | $h(n) \leq h^*(n)$ | $h(n) \leq c(n, n') + h(n')$ |
| **限制范围** | 估计值 vs 真实最优代价 | 估计值 vs 当前代价+下一步估计 |
| **对最优性的影响** | **必须有**（A* 的基础） | 有（加强一致性） |
| **对效率的影响** | 无（仅保证最优） | **有**（避免重新打开） |
| **逻辑关系** | ← | → 包含关系 |
| **核实方法** | 与真实 $h^*$ 比较（通常困难） | 检查所有边，相对容易 |
| **实践建议** | 设计时必须满足 | 尽量满足（提高效率） |

##### 相容性的含义澄清

**相容性（Compatibility）**就是**一致性（Consistency）**的同义词，在中文文献中有时也称为**单调性（Monotonicity）**。它们都指：

$$
h(n) \leq c(n, n') + h(n')
$$

三个名字源自不同背景：
- **一致性（Consistency）**：强调启发值的内部一致性
- **相容性（Compatibility）**：强调与图结构的相容
- **单调性（Monotonicity）**：强调 $f$ 值沿路径单调不减

在 A* 语境中，它们的含义完全相同。

##### 关键结论

1. **可采纳性是 A* 最优性的必要条件**——不可松动
2. **一致性/相容性是优化 A* 效率的充分条件**——设计时应追求
3. **若启发一致，则自动可采纳**——一致性更强
4. **可采纳但不一致的启发仍能找到最优解，但可能重复扩展节点**
5. **在实践中，优秀的启发通常既可采纳又一致**

#### 启发函数的一致性（Consistency / Monotonicity）详细讲解

**定义**：启发函数 $h$ 称为**一致的**（consistent），若对所有边 $(n, n')$ 满足：

$$
h(n) \leq c(n, n') + h(n')
$$

其中 $c(n, n')$ 是从 $n$ 到 $n'$ 的边权。

**物理理解**：启发估计满足三角不等式

**一致性的优势**：
- 若 $h$ 一致，则 A* 首次扩展节点时该节点的 $f$ 值就已最优
- 无需回溯重新打开已扩展的节点
- 时间复杂度更优

**一致性 vs 可采纳性**：
- 一致性 $\Rightarrow$ 可采纳性
- 可采纳性 ⇏ 一致性（反方向不一定成立）

#### 启发函数的设计方法

**1. 松弛问题（Problem Relaxation）**

移除原问题的某些约束，解松弛问题的最优代价即为启发值。

**例子：8 数字谜题**

原问题约束：每个数字一次只能移动到相邻空位

松弛：允许任意数字移动到任意位置（一步）

松弛问题的解 = 曼哈顿距离

**2. 模式数据库（Pattern Databases）**

预先计算子问题最优解的成本，存储为表格。

**例子：15 数字谜题**

预计算所有排列中部分数字（如前 10 个）到目标的距离，运行时查表。

**3. 线性组合**

$$
h(n) = w_1 h_1(n) + w_2 h_2(n) + \cdots + w_k h_k(n)
$$

若各 $h_i$ 都可采纳，加权组合的最大值也可采纳：

$$
h(n) = \max(h_1(n), h_2(n), \ldots, h_k(n))
$$

**4. 学习型启发（Learned Heuristics）**

用神经网络学习 $h(n)$，无需人工设计。

Trade-off：精度 vs 计算时间。

### A* 算法（A-Star Algorithm）

#### 核心定义与公式

**A* 算法是启发式搜索中最著名的算法**，它评估每个节点的"总代价"为：

$$
f(n) = g(n) + h(n)
$$

其中：

- **$g(n)$**：起点到节点 $n$ 的**实际已知代价**（路径长度、时间、燃料消耗等）
- **$h(n)$**：从节点 $n$ 到目标的**启发估计代价**（启发函数）
- **$f(n)$**：通过 $n$ 的路径的**预计总代价**

**核心逻辑**：
- 每次从 frontier 中取出 **$f(n)$ 最小**的节点展开
- 这样既利用了已知信息（$g(n)$），也利用了对未来的估计（$h(n)$）

#### A* 的伪代码

```python
def A_star(start, goal, graph):
    frontier = PriorityQueue()
    frontier.put((0, start))  # (f_value, node)
    came_from = {start: None}
    g = {start: 0}
    h = lambda n: heuristic(n, goal)  # 启发函数
    
    while not frontier.empty():
        _, current = frontier.get()
        
        if current == goal:
            return reconstruct_path(came_from, start, goal)
        
        for neighbor in graph[current]:
            # 计算经由 current 到 neighbor 的新代价
            new_g = g[current] + cost(current, neighbor)
            
            # 如果找到更优路径
            if neighbor not in g or new_g < g[neighbor]:
                g[neighbor] = new_g
                h_neighbor = h(neighbor)
                f_neighbor = new_g + h_neighbor
                frontier.put((f_neighbor, neighbor))
                came_from[neighbor] = current
    
    return None  # 无解
```

#### A* 的详细执行流程

**核心数据结构**：

- **Open List（开放列表）**：待展开的节点，通常用优先级队列实现，按 $f(n)$ 值排序
- **Closed Set（关闭集合）**：已展开的节点，保存已评估过的节点
- **g(n) 表**：起点到各节点的最小已知代价
- **h(n) 表**：启发函数估计值（问题相关）
- **f(n) 表**：$f(n) = g(n) + h(n)$，用于排序优先级
- **来源表（came_from）**：记录每个节点的前驱，用于重建路径

**详细步骤**：

```
步骤 1：初始化
────────────────
初始化 Open List：{(start, f(start) = 0 + h(start))}
初始化 Closed Set：{}
初始化 g(start) = 0
初始化 came_from(start) = null

步骤 2：主循环（while Open List not empty）
───────────────────────────────────────────
2.1  从 Open List 中取出 f(n) 最小的节点，记为 current
     将 current 加入 Closed Set
     
2.2  若 current == goal：找到路径，执行步骤 5（回溯）

2.3  否则，展开 current，遍历其所有邻接节点 neighbor：
     
     2.3.1 计算新代价：new_g = g(current) + cost(current, neighbor)
     
     2.3.2 判断 neighbor：
           ├─ 若 neighbor 在 Closed Set 中：跳过（已最优）
           ├─ 若 neighbor 不在任何列表中：
           │  ├─ 设置 g(neighbor) = new_g
           │  ├─ 计算 h(neighbor)（启发值）
           │  ├─ 计算 f(neighbor) = new_g + h(neighbor)
           │  ├─ 加入 Open List
           │  └─ 记录 came_from(neighbor) = current
           └─ 若 neighbor 已在 Open List 中且 new_g < g(neighbor)：
              ├─ 更新 g(neighbor) = new_g
              ├─ 更新 f(neighbor) = new_g + h(neighbor)
              └─ 记录 came_from(neighbor) = current

步骤 3：循环条件检查
──────────────────
若 Open List 为空且未找到目标：返回"无解"

步骤 4：终止条件
────────────
当 current == goal 时，执行步骤 5

步骤 5：路径回溯
────────────
从 goal 出发，利用 came_from 表逆向回溯到 start：
path = []
current = goal
while current != null:
    path.push_front(current)
    current = came_from[current]
return path
```

#### A* 执行流程的具体示例

**示例图**（简化版）：

```
     Start(A)
       / | \
      2  3  4
     /   |   \
    B    C    D
    |    |\   |
    1    2 1  2
    |    | |  |
    E    F G  Goal(H)
```

**假设**：
- 所有边权如图所示
- 启发函数 $h$ 为直线距离估计（下面给出）
- Goal = H

| 节点 | 坐标 | h(n) |
|------|------|------|
| A(Start) | (0,0) | 4.47 |
| B | (1,2) | 3.16 |

**执行过程**：

| 迭代 | Open List（按f排序） | Closed Set | 展开节点 | 新发现节点 | 更新操作 |
|------|---------------------|-----------|--------|----------|--------|
| 初始 | A(f=4.47) | {} | - | - | - |
| 1 | B(f=5.16), C(f=5.83), D(f=6.00) | A | A | B,C,D | g(B)=2, g(C)=3, g(D)=4 |

| 7 | H(f=6.00) | A,B,C,D,F,G | H | **找到目标！** | **终止** |

**路径回溯**：
```
came_from 表：
  A → null

回溯过程：
  current = H → came_from[H] = D
  current = D → came_from[D] = A
  current = A → came_from[A] = null
  
最终路径：A → D → H（总代价 = 4 + 2 = 6）
```

#### Open List 与 Closed Set 的管理

**关键规则**：

1. **节点在 Open List 中**：待评估，可能被更新
   - 若发现更优路径，更新 $g(n)$、$f(n)$ 和 $\text{came\_from}(n)$
   - 节点在优先级队列中的位置自动调整

2. **节点在 Closed Set 中**：已确定最优
   - 根据 A* 的最优性，若启发可采纳，首次展开该节点时已达到最优代价
   - **重要**：在一致性启发下，无需重新打开
   - 在非一致启发下，可能需要重新打开（回到 Open List）

3. **都不在**：未访问节点，首次发现时加入 Open List

**优先级队列排序**：

```python
# Open List 按 f(n) 升序排列
Open_List.sort(key=lambda node: f[node])

# 若 f(n) 相等，可用次级排序标准（如 h(n) 或执行顺序）
def compare(n1, n2):
    if f[n1] != f[n2]:
        return f[n1] < f[n2]
    else:
        return h[n1] < h[n2]  # 偏好更接近目标的节点
```

#### A* 算法的效率要点

**时间瓶颈**：

1. **优先级队列操作**：每次 get/put 为 $O(\log |Open\ List|)$
   - 总的队列操作时间：$O(|展开节点数| \times \log |展开节点数|)$

2. **邻接节点遍历**：对每个展开的节点，遍历所有邻接节点
   - 若有 $E$ 条边，复杂度为 $O(E)$

3. **启发函数计算**：$h(n)$ 的计算开销
   - 若 $h$ 计算复杂，整体时间会显著增加

**总时间复杂度**（启发质量相关）：

$$
T = O(b^d \log b^d) = O(d \cdot b^d \log b)
$$

其中 $b$ 是分支因子，$d$ 是目标深度。

**空间瓶颈**：

| 部分 | 空间大小 | 说明 |
|------|---------|------|
| Open List | $O(b^d)$ | 最坏保存所有未展开节点 |
| Closed Set | $O(b^d)$ | 最坏保存所有已展开节点 |
| g 表 / h 表 | $O(b^d)$ | 每个节点一条记录 |
| came_from 表 | $O(b^d)$ | 每个节点一条回溯指针 |
| **总计** | $O(b^d)$ | 主要内存消耗 |

#### 常见的 A* 变体与优化

**1. 重开策略（Reopening）**

对于非一致启发，可能需要重新打开已关闭节点：

```python
if neighbor in closed_set and new_g < g[neighbor]:
    # 发现更优路径，重新打开
    closed_set.remove(neighbor)
    open_list.add(neighbor)
    g[neighbor] = new_g
    came_from[neighbor] = current
```

**2. 跳过策略（Pruning）**

若节点的 $f$ 值已超过当前最优解的下界，可以剪枝：

```python
if f[neighbor] > best_possible_cost:
    continue  # 剪枝，不需探索
```

**3. 记忆 A*（Memory A*）**

限制内存使用，若 Open List 超出内存限制，删除最高 $f$ 值的节点。

**4. 加权 A*（Weighted A*）**

$$
f(n) = g(n) + w \cdot h(n), \quad w > 1
$$

通过权重 $w$ 控制加速程度与质量权衡。

#### A* 与手工追踪的关键观察

**常见错误**：

1. ❌ 节点重复打开：一旦加入 Closed Set，不应再展开（一致启发下）
2. ❌ 忘记更新 came_from：导致路径回溯错误
3. ❌ 启发函数设计不当：过高估计导致找不到最优解
4. ❌ 优先级队列混乱：f 值更新后未重新排序

**正确做法**：

✓ 维护 Open / Closed 两个独立集合
✓ 每次取出 f 最小的节点展开
✓ 发现更优路径时立即更新信息
✓ 目标找到时立即停止（不继续搜索）
✓ 启发函数必须可采纳：$h(n) \leq h^*(n)$

#### A* 的关键性质

**1. 完备性（Completeness）**

如果目标存在且可达，A* 总是能找到解。

| 条件 | 完备性 |
|------|--------|
| 有限分支因子 | **是** |
| 有限搜索空间 | **是** |
| 无限深度但存在解 | 需要 $h(n) \geq 0$ 且无穷不可达 |

**2. 最优性（Optimality）**

**当启发函数满足"可采纳性"时，A* 找到的解是最优的。**

**可采纳启发函数（Admissible Heuristic）**：

$$
h(n) \leq h^*(n) \quad \forall n
$$

其中 $h^*(n)$ 是从 $n$ 到目标的**真实最优代价**。

**含义**：启发函数永不高估到达目标的代价。

**证明核心思路**：
- 若 A* 找到的解为 $p$，则存在某个节点 $n$ 在目标前被展开
- 此时 $f(n) = g(n) + h(n) \leq g(n) + h^*(n)$（可采纳）
- 所以 $f(n)$ 不会超过最优路径代价，A* 必然找到最优解

**3. 时间复杂度**

$$
\text{Time} = O(b^d)
$$

其中 $b$ 是分支因子，$d$ 是目标深度。

**实际情况**：取决于启发函数的质量
- **优秀启发**：大幅剪枝，接近 $O(d)$
- **差启发**：接近无信息搜索，$O(b^d)$

**4. 空间复杂度**

$$
\text{Space} = O(b^d)
$$

因为需要保存所有展开过的节点到 frontier 和 closed list。

#### 启发函数的设计与性质

**常见启发函数**：

| 启发函数 | 表达式 | 适用场景 | 是否可采纳 |
|----------|--------|---------|----------|
| **曼哈顿距离** | $h(n) = \|x_n - x_g\| + \|y_n - y_g\|$ | 网格，4-方向移动 | 可采纳 |
| **欧几里得距离** | $h(n) = \sqrt{(x_n-x_g)^2 + (y_n-y_g)^2}$ | 网格，8-方向移动；平面图 | 可采纳 |
| **切比雪夫距离** | $h(n) = \max(\|x_n-x_g\|, \|y_n-y_g\|)$ | 网格，对角移动允许 | 可采纳 |
| **绝不低估** | $h(n) = 0$ | 退化为 Dijkstra/UCS | 可采纳但无用 |

**启发函数的性质**：

- **一致性/单调性（Consistency）**：$h(n) \leq cost(n, n') + h(n')$
  - 含义：启发估计满足三角不等式
  - 若启发一致，则 A* 无需重新打开已展开的节点
  
- **优势性（Dominance）**：如果 $h_1(n) \leq h_2(n) \leq h^*(n)$，则 $h_2$ 比 $h_1$ 更好
  - 更好的启发会展开更少的节点

#### A* vs 其他算法的对比

| 算法 | 评估函数 | 完备性 | 最优性 | 时间复杂度 | 空间复杂度 | 特点 |
|------|---------|--------|--------|------------|-----------|------|
| **BFS** | 深度 | 是 | 是（单位代价） | $O(b^d)$ | $O(b^d)$ | 无信息，简单但耗内存 |
| **Dijkstra/UCS** | $g(n)$ | 是 | 是 | $O(\log V + E \log E)$ | $O(V)$ | 基于代价，不用启发信息 |
| **Greedy Best-First** | $h(n)$ | 否 | 否 | 通常快，但无保证 | $O(b^d)$ | 快速但可能找到次优解 |
| **A*** | $g(n) + h(n)$ | 是 | 是（可采纳启发） | 取决于 $h$；通常 $O(b^{d\epsilon})$，$\epsilon$ 很小 | $O(b^d)$ | 兼顾完备性、最优性和效率 |
| **A\*ε（Weighted A\*）** | $g(n) + \epsilon h(n)$，$\epsilon > 1$ | 是 | 否，但误差有界 | 更快，可达 $(1+\epsilon)$ 倍最优 | 取决于 $\epsilon$ | 加速版 A*，单位时间内找更多解 |

### A* 在自动驾驶中的应用详析

自动驾驶涉及**路径规划（Path Planning）** 和 **运动规划（Motion Planning）**，A* 是其中最核心的算法。

#### 1. 应用场景一：高层路线规划（Coarse-Grained Planning）

**问题**：给定起点和目标位置，在整个城市/地图上找到最优路线（最短距离、最少时间、最低成本）。

**状态表示**：
- 状态空间：划分为网格或使用显式的交通网络图
- 节点：网格单元或路口/道路段
- 边权：相邻网格间的距离/时间

**启发函数选择**：

$$
h(n) = \frac{\text{欧几里得距离}(n, \text{goal})}{\text{最大预期速度}}
$$

**理由**：
- 分子是地理距离的下界
- 分母归一化为时间
- 满足可采纳性：真实行驶时间 $\geq$ 直线距离/最大速度

**例子**：在 100×100 的城市网格上，汽车从 (5, 10) 要到 (95, 90)
$$
h(n) = \frac{\sqrt{(95-x)^2 + (90-y)^2}}{v_{\max}}
$$

若 $v_{\max} = 60$ km/h，欧距离 ≈ 127 km，则 $h \approx 2.1$ 小时。

#### 2. 应用场景二：细粒度运动规划（Fine-Grained Planning）

**问题**：在复杂障碍物环境中，规划平滑、可安全执行的轨迹。

**方法 A：配置空间 + A***

**配置空间（Configuration Space）**：
- 将车辆和环境映射到状态空间
- 对于平面移动的车：$\mathbf{q} = (x, y, \theta)$（位置 + 朝向）
- 对于更复杂的车（如非完整约束）：$\mathbf{q} \in \mathbb{R}^n$

**状态转移**：
$$
\mathbf{q}_{t+1} = \mathbf{q}_t + \Delta t \cdot \mathbf{v}_t
$$

其中 $\mathbf{v}_t = (v_x, v_y, \omega)$（速度 + 角速度）

**代价函数**：
$$
g(n) = \sum_{i=0}^{n-1} \text{cost}(\mathbf{q}_i \to \mathbf{q}_{i+1})
$$

常见的代价包括：
- **距离代价**：$\||\mathbf{q}_{i+1} - \mathbf{q}_i\||_2$
- **时间代价**：到达该状态的时间
- **能量代价**：$\int_0^T v^2(t) \, dt$（加速度平方和）
- **舒适度代价**：加速度、转向速率的惩罚

**启发函数**：
$$
h(\mathbf{q}) = \frac{\|\mathbf{q} - \mathbf{q}_{\text{goal}}\|_2}{v_{\max}} + k \cdot \text{angle\_diff}(\theta, \theta_{\text{goal}})
$$

其中 $k$ 是朝向差的权重系数。

#### 3. 应用场景三：混合 A* 与运动模型（Hybrid A*）

**背景**：标准 A* 假设任意转向随意改变速度，但真实车辆有运动学约束。

**车辆运动模型**（自行车模型）：

$$
\begin{align}
\dot{x} &= v \cos \theta \\
\dot{y} &= v \sin \theta \\
\dot{\theta} &= \frac{v}{L} \tan \delta
\end{align}
$$

其中：
- $v$：速度
- $\delta$：转向角
- $L$：轴距

**Hybrid A* 的改进**：

1. **离散化动作集**：而不是连续方向
   $$
   \delta \in \{-\delta_{\max}, -\delta_{\max}/2, 0, \delta_{\max}/2, \delta_{\max}\}
   $$

2. **分层搜索**：
   - **粗层**：标准 A*，忽略运动模型
   - **细层**：给定粗层的路径，用 RRT 或前向仿真验证是否满足运动约束

3. **启发函数改进**：
   $$
   h(\mathbf{q}) = \min_{\delta} \frac{\text{distance\_to\_goal}(\mathbf{q}, \mathbf{q}_g, \delta)}{v_{\max}}
   $$
   
   其中 minimum 遍历所有可能的控制输入，预计的最快到达时间。

#### 4. 应用场景四：动态环境下的重规划（Replanning in Dynamic Environments）

**问题**：环境中有移动的障碍物（其他车、行人），规划会失效，需要实时重规划。

**改进方案**：

**D* Lite（Dynamic A*）**：A* 的动态版本，只重新计算受影响的节点

**核心思想**：
- 维护两个值：$g(n)$（当前估计代价）和 $\text{rhs}(n)$（右手值）
- 当环境改变时，只更新变化区域的节点
- 避免全局重搜

$$
g(n) = \min_{n' \in \text{pred}(n)} [g(n') + c(n', n)]
$$

时间复杂度从 $O(b^d)$ 降至平均 $O(\log(b^d))$。

#### 5. 完整自动驾驶规划流程

```
输入：当前位置、目标位置、地图、传感器数据

1. 全局规划（Global Path Planning）
   ├─ 使用 A* 在高层地图上规划全局路线
   ├─ 启发函数：欧几里得距离 / 最大速度
   └─ 输出：道路点序列 [P_1, P_2, ..., P_N]

2. 局部规划（Local Path Planning）  
   ├─ 融合实时传感器数据（车道检测、障碍物）
   ├─ 在车周围的配置空间用 A* / Hybrid A*
   ├─ 启发函数：融合距离、方向、速度约束
   └─ 输出：避障轨迹

3. 动态调整（Dynamic Adjustment）
   ├─ 每 T=50 ms 重新感知环境
   ├─ 若环境变化超过阈值，触发 D* Lite 重规划
   ├─ 否则继续执行当前轨迹
   └─ 输出：实时控制指令(v, δ)

4. 控制执行（Control Execution）
   ├─ 跟踪规划轨迹
   ├─ PID/MPC 控制算法
   └─ 输出：转向、加油、制动
```

#### 6. 成本函数的现实设计

在实际自动驾驶中，$g(n)$ 不只是距离：

$$
g(n) = \alpha \cdot \text{distance} + \beta \cdot \text{time} + \gamma \cdot \text{smoothness} + \delta \cdot \text{safety}
$$

各项可展开为：

- **距离项**：路径长度 $L$
- **时间项**：预计行驶时间 $T = \sum_i L_i / v_i$
- **平滑项**：曲率积分 $\int \kappa^2(s) \, ds$，衡量乘坐舒适度
- **安全项**：与障碍物的最小距离 $d_{\min}$；若 $d_{\min} < d_{\text{threshold}}$，施加大惩罚

权重 $\alpha, \beta, \gamma, \delta$ 由模型调优或学习得出。

#### 7. A* 在自动驾驶中的实际性能

| 场景 | 求解时间 | 展开节点数 | 备注 |
|------|---------|----------|------|
| 高层路线规划（100×100 网格） | 10-50 ms | 500-2000 | 离线，可接受 |
| 局部避障（20×20 配置空间） | 5-30 ms | 100-800 | 实时要求较严格 |
| Hybrid A*（含运动模型） | 20-100 ms | 2000-5000 | 更精确但更耗时 |
| D* Lite（动态重规划） | 1-10 ms | 10-100 | 显著加速 |

**典型参数**：
- 规划频率：20-50 Hz（每 50 ms 一次规划）
- 搜索地平线：5-10 秒的行驶距离
- 启发函数更新频率：与规划同步

### A* 的局限与改进

**局限**：
1. 内存爆炸：大规模环境下 frontier 可能包含百万级节点
2. 启发函数设计困难：好的启发需要领域知识
3. 不适合连续高维空间：配置空间维数过高时搜索空间指数爆炸

**改进方向**：
1. **RRT / RRT***：随机采样，避免网格爆炸
2. **学习启发函数**：用神经网络学习更好的 $h(n)$
3. **多分辨率规划**：先用粗网格A*，再精细化
4. **GPU 并行化**：并行展开多个节点

### 启发式策略 vs 近似策略

启发式搜索与近似算法是两个**不同层面**的概念，常被混淆。现统一对比。

#### 核心区别

| 维度 | 启发式策略（Heuristic Search） | 近似策略（Approximation） |
|------|-------------------------------|-------------------------|
| **目标** | 保证最优，加速搜索过程 | 放弃最优，换取可行的近似解 |
| **最优性保证** | 有（若启发可采纳） | 无（只保证误差界） |
| **完备性保证** | 有 | 通常有 |
| **可采纳启发函数** | 必须 $h(n) \leq h^*(n)$ | 无此限制 |
| **返回解的特点** | 精确最优解 | $(1 + \epsilon)$ 倍近似解 |
| **代典代表** | A*、IDA* | Greedy、RRT、近邻启发式 |
| **何时使用** | 精确解必要，时间充足 | 精确解代价过高，需快速可行解 |

#### 启发式策略详解

**定义**：利用**启发估计函数** $h(n)$ 来指导搜索方向，但仍然保证找到最优解。

**核心条件**：启发函数必须可采纳（admissible）：
$$
0 \leq h(n) \leq h^*(n) \quad \forall n
$$

其中 $h^*(n)$ 是真实最优代价。

**典型例子**：
- **A***：$f(n) = g(n) + h(n)$，$h$ 为欧距离、曼哈顿距离
- **IDA***（Iterative Deepening A*）：深度受限的 A*，递推搜索深度

**执行流程**：
```
1. 初始化：frontier ← {start}, g(start) ← 0
2. while frontier 非空：
   3. 取 f(n) 最小的节点 n
   4. if n == goal: return 最优路径
   5. 展开 n，对每个邻接点 n'：
      ├─ 若 g(n) + cost(n,n') < g(n')：
      │  ├─ 更新 g(n') ← g(n) + cost(n,n')
      │  └─ 加入 frontier
      └─ else: 忽略
6. return 无解
```

**优点**：
- 保证最优性（若启发可采纳）
- 相比无信息搜索显著加速
- 随启发函数质量提升，加速效果越好

**缺点**：
- 仍需完整搜索，内存占用大
- 启发函数设计需领域知识
- 高维空间下仍可能爆炸

#### 近似策略详解

**定义**：主动**放弃最优性**，接受质量有保证的"足够好"的解，以换取时间或空间的显著节省。

**误差界保证**：

设算法返回的解代价为 $C$，最优解代价为 $C^*$，则要求：

$$
C \leq (1 + \epsilon) \cdot C^*
$$

其中 $\epsilon$ 是预先指定的误差率。

**典型例子**：

**1. Greedy 最近邻（Nearest Neighbor）**

用于 TSP：
```
1. 从任意城市出发
2. 每次选择距离当前最近的未访问城市
3. 返回形成的回路
```

**时间复杂度**：$O(n^2)$（相比 A* 的指数级别大幅降低）

**误差界**：返回的路径长度 $\leq 2 \times$ 最优解（在树上的情况）

**代码框架**：
```python
def nearest_neighbor_tsp(cities):
    unvisited = set(cities)
    current = cities[0]
    unvisited.remove(current)
    path = [current]
    
    while unvisited:
        nearest = min(unvisited, key=lambda c: distance(current, c))
        path.append(nearest)
        current = nearest
        unvisited.remove(nearest)
    
    return path
```

**2. Christofides 算法**

结构：
$$
\text{Christofides} = \text{MST} + \text{最小权完美匹配}
$$

**步骤**：
1. 构造图的最小生成树 MST
2. 找出 MST 中所有奇度顶点
3. 在奇度顶点间计算最小权完美匹配（Minimum-Weight Perfect Matching）
4. 合并 MST 和匹配的边，形成欧拉图
5. 在欧拉图上求欧拉回路
6. 跳过重复顶点，得到哈密顿回路近似解

**时间复杂度**：$O(n^3)$

**近似比**：$\leq 1.5 \times C^*$

**3. RRT（快速随机树）**

用于高维运动规划：

```python
def RRT(q_init, q_goal, max_iter=5000):
    tree = {q_init: None}
    
    for _ in range(max_iter):
        # 随机采样配置空间
        if random() < 0.1:
            q_rand = q_goal
        else:
            q_rand = random_config()
        
        # 找树中最近的节点
        q_nearest = nearest_node(tree, q_rand)
        
        # 朝 q_rand 方向扩展
        q_new = extend(q_nearest, q_rand, step_size=0.1)
        
        if is_collision_free(q_nearest, q_new):
            tree[q_new] = q_nearest
            
            if distance(q_new, q_goal) < threshold:
                return reconstruct_path(tree, q_init, q_new)
    
    return None
```

**特点**：
- 概率完备（无穷时间可达任意可达配置）
- 不保证最优，但返回可行解
- 处理复杂约束和高维空间的能力强

#### 启发式 vs 近似的本质区别

从**决策角度**理解：

**启发式**："我要找最优解，但我用启发函数缩小搜索范围"
- 仍在追求 $\min f(x)$
- 但通过好的 $h(n)$ 避免无谓搜索

**近似**："我放弃最优，接受 $f(x) \leq (1+\epsilon) f^*$"
- 直接使用快速算法（贪心、随机等）
- 不保证最优，但保证质量下界

#### 混合策略：Weighted A*

结合两种思想的折衷方案：

$$
f(n) = g(n) + w \cdot h(n), \quad w > 1
$$

**特点**：
- 当 $w=1$：标准 A*（保证最优）
- 当 $w$ 很大：接近贪心（加速，但可能非最优）
- 当 $1 < w < \infty$：可控的加速和误差权衡

**误差界**：返回的解代价 $\leq w \times C^*$

**时间复杂度**：$O(b^{d \cdot f_{\text{error}}})$，其中 $f_{\text{error}} = 1/w$

这样既保留了启发式的理论框架，又能根据实际需求调整速度-质量的权衡。

#### 实际应用场景选择

| 场景 | 推荐策略 | 理由 |
|------|---------|------|
| **精确路线规划**（GPS导航） | A*（启发式） | 需要最优路线，用户可接受秒级延迟 |
| **实时避障**（无人车） | RRT（近似） | 需要快速可行解，最优性非关键 |
| **大规模图搜索** | Weighted A*（混合） | 平衡最优性和速度 |
| **旅行商问题** | Christofides（近似） | 保证 1.5 倍，比穷举快指数倍 |
| **游戏AI寻路** | 分层A*（启发式） | 多分辨率，先粗后精 |
| **库存/任务调度** | 贪心启发（近似） | NP-hard，快速可行解足够 |

#### 关键总结

1. **启发式追求最优**，近似**接受次优但快速**
2. **Weighted A* 提供可调的权衡**：权重 $w$ 编码了"有多急"
3. **高维/约束多的场景倾向近似**（如 RRT 处理复杂运动学）
4. **实时系统通常混合使用**：全局用近似快速规划，局部用启发式精细优化
5. **启发函数质量决定启发式的实用性**，而**误差界决定近似算法的可用性**
