# 第 3 讲：搜索求解

> 课程：Introduction to Artificial Intelligence · 复旦大学  
> 日期：2026-03-19  
> 资料：人工智能-3-搜索求解.pdf、本讲 Markdown/实践材料

## 本讲定位

把问题表述为状态空间，在无信息搜索与启发式搜索之间权衡完备性、最优性与代价。

## 学习目标

- 掌握搜索问题五要素：状态、动作、初始状态、目标测试、路径代价。
- 能比较 BFS、DFS、UCS、贪心搜索与 A* 的适用场景。
- 理解启发函数 h(n) 如何把领域知识注入搜索。

## 知识地图

| 模块 | 内容 |
| --- | --- |
| 问题建模 | 罗马尼亚路径规划等例子说明：先定义状态图，再谈算法。 |
| 无信息搜索 | 不使用目标方向信息，依靠队列/栈/优先队列穷举。 |
| 启发式搜索 | 用 h(n) 估计到目标的剩余代价，减少搜索空间。 |

## 核心概念

- BFS 层层展开，边权相同且解较浅时效果好。
- UCS 按累计代价 g(n) 展开，适合边权不同的最短路。
- A* 使用 f(n)=g(n)+h(n)，当 h 可采纳且一致时可保证最优。

## 课堂讲解补充

搜索求解训练的是算法建模能力。很多 AI 问题看似复杂，本质上可以先写成状态空间：当前在哪里、能做什么、代价是什么、何时算成功。BFS、UCS、A* 的差别不是背诵名称，而是理解它们如何使用队列、代价和启发信息控制展开顺序。

### 复习组织方式

- **问题背景**：这节课为什么要引入这个概念。
- **方法主线**：算法或模型按什么步骤工作。
- **公式/代码**：至少抓住一个能落地计算的表达。
- **局限性**：说明它在哪些场景下会失效或需要改进。

## 公式与记忆点

- A*: f(n)=g(n)+h(n)。
- 可采纳启发：0 ≤ h(n) ≤ h*(n)，即永不高估真实剩余代价。

## 易错点

- 先套算法而不定义状态和动作，搜索问题会变成口号。
- 启发函数如果高估，A* 可能很快但不再保证最优。

## 课后巩固

- 把八数码或迷宫问题写成五要素。
- 解释 BFS、UCS、A* 在边权和启发信息上的区别。
