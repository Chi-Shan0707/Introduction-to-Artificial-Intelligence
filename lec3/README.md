# lec3 核心提炼：启发式搜索

## 1. 启发式搜索是什么

- 启发式搜索：利用领域知识引导搜索，减少盲目扩展。
- 核心思想：用启发函数估计“离目标还有多远”。

## 2. 两个核心函数

- 启发函数：$h(n)$，估计从节点 $n$ 到目标的剩余代价。
- 实际代价：$g(n)$，从起点到 $n$ 的已知代价。
- 评估函数：$f(n)=g(n)+h(n)$，用于决定扩展优先级。

常见算法对应关系：

- Dijkstra / UCS：$f(n)=g(n)$
- Greedy Best-First：$f(n)=h(n)$
- A*：$f(n)=g(n)+h(n)$

## 3. A* 一句话

每次从 Open List 中取 $f(n)$ 最小的节点展开；
若找到更小的 $g$，就更新父指针和优先级；
到达目标后按父指针回溯路径。

## 4. A* 完备性（Completeness）

- 定义：若解存在，算法最终一定能找到解。
- A* 在常见离散搜索图上的完备性条件：
	- 分支因子有限；
	- 每步代价有正下界（例如 $c(n,n') \ge \epsilon > 0$）；
	- 状态空间有限，或虽无限但不会在有限代价内出现无限扩展链。
- 结论：满足上面条件时，A* 是完备的。

可记忆为：

- 有解 + 不会“零代价无限打转” + 每层可扩展数有限 $\Rightarrow$ A* 能找到解。

## 5. 启发函数三大性质

### 5.1 可采纳性（Admissible）

$$
h(n) \le h^*(n),\ \forall n
$$

- 含义：不高估真实最优剩余代价。
- 作用：保证 A* 找到最优解。

### 5.1.1 最优性的简单证明（基于可采纳性）

设最优解总代价为 $C^*$，A* 第一次取出的目标节点为 $G$，其代价为 $g(G)$。

若 $g(G) > C^*$（反设非最优），考虑最优路径上某个尚未展开节点 $n$，有：

$$
f(n)=g(n)+h(n) \le g(n)+h^*(n)=C^*
$$

（由可采纳性 $h(n)\le h^*(n)$）。于是存在节点 $n$ 满足 $f(n)\le C^*<g(G)=f(G)$，这与 A* 总是先取最小 $f$ 的节点矛盾。

故反设不成立，A* 第一次取出的目标节点即为最优解。

### 5.2 一致性/ 单调性（Consistent / Monotone）

$$
h(n) \le c(n,n') + h(n'),\ \forall (n,n')
$$

- 含义：满足三角不等式。
- 作用：A* 通常无需重新打开已关闭节点，效率更高。

### 5.3 关系

- 一致性（= 相容性）$\Rightarrow$ 可采纳性
- 可采纳性 $\nRightarrow$ 一致性（相容性）
- 结论：一致性是更强条件，必然导出可采纳性

## 6. A* 复杂度（常见写法）

- 时间复杂度：最坏约 $O(b^d)$
- 空间复杂度：最坏约 $O(b^d)$

其中 $b$ 为分支因子，$d$ 为解深度。A* 的性能高度依赖 $h(n)$ 质量。

## 7. 常见启发函数

- 网格 4 邻域：曼哈顿距离
- 网格 8 邻域：欧几里得距离 / 切比雪夫距离
- 若 $h(n)=0$：A* 退化为 Dijkstra/UCS

## 8. 启发式策略 vs 近似策略

- 启发式策略：目标仍是最优解（典型 A*）
- 近似策略：接受次优以换速度（如 Greedy、RRT、Christofides）

可记忆为：

- 要“最优 + 可证明”：优先 A*
- 要“实时 + 快速可行”：优先近似或加权启发

## 9. 自动驾驶中的最小结论

- 全局规划：常用 A* / D* Lite（地图级路径）
- 局部规划：常用 Hybrid A* / 采样法（考虑车辆运动学）
- 工程里常做混合：全局保质量，局部保实时
