第 3 讲：搜索求解

学习目标

  • 掌握搜索问题五要素：状态、动作、初始状态、目标测试、路径代价。
  • 能比较 BFS、DFS、UCS、贪心搜索与 A* 的适用场景。
  • 理解启发函数 h(n) 如何把领域知识注入搜索。

知识地图

模块内容
问题建模罗马尼亚路径规划等例子说明：先定义状态图，再谈算法。
无信息搜索不使用目标方向信息，依靠队列/栈/优先队列穷举。
启发式搜索用 h(n) 估计到目标的剩余代价，减少搜索空间。

核心概念讲解

要点 1

BFS 层层展开，边权相同且解较浅时效果好。

要点 2

UCS 按累计代价 g(n) 展开，适合边权不同的最短路。

要点 3

A* 使用 f(n)=g(n)+h(n)，当 h 可采纳且一致时可保证最优。

课堂讲解补充

搜索求解训练的是算法建模能力。很多 AI 问题看似复杂，本质上可以先写成状态空间：当前在哪里、能做什么、代价是什么、何时算成功。BFS、UCS、A* 的差别不是背诵名称，而是理解它们如何使用队列、代价和启发信息控制展开顺序。

表达结构

复习时可以按“问题从哪里来、方法怎么做、公式或代码落在哪里、效果怎么评价、局限是什么”来整理。

与课程主线的连接

复习时顺手把它和前后几讲连起来：这一讲解决了什么问题，又给后面的模型或实验留下了哪些工具。

公式速查与解释

  • A*: f(n)=g(n)+h(n)。
  • 可采纳启发：0 ≤ h(n) ≤ h*(n)，即永不高估真实剩余代价。
阅读建议：不要孤立背公式；每个公式都要能说出变量含义、适用条件和在代码/实验中的位置。

从课堂到项目的标准流程

步骤操作要点
1. 定义任务先说清输入、输出、假设与评价指标，避免只罗列算法名。
2. 建立表示选择逻辑符号、状态空间、特征向量、张量或 MDP 状态。
3. 选择模型根据数据规模、结构先验和计算预算选择规则、搜索、线性模型、神经网络或 RL 方法。
4. 训练/推理明确损失函数、优化过程、推理步骤和失败模式。
5. 评估迭代用与任务匹配的指标验证效果，并记录可复现实验设置。

代码/伪代码示例

下面的代码只保留课堂最需要抓住的骨架；真正练习时再对照 notebooks、讲义和项目文件补全细节。

import heapq

def astar(start, goal, neighbors, heuristic):
    pq = [(heuristic(start), 0, start, [])]
    seen = {}
    while pq:
        f, g, node, path = heapq.heappop(pq)
        if node == goal:
            return path + [node], g
        if node in seen and seen[node] <= g:
            continue
        seen[node] = g
        for nxt, cost in neighbors(node):
            heapq.heappush(pq, (g + cost + heuristic(nxt), g + cost, nxt, path + [node]))

常见误区

  • 先套算法而不定义状态和动作，搜索问题会变成口号。
  • 启发函数如果高估，A* 可能很快但不再保证最优。
考试与汇报提醒：回答问题时尽量使用“任务定义 → 方法原理 → 关键公式/代码 → 指标 → 局限”的结构。
