1. Classical Graph Problems (经典图问题)

Search algorithms in AI often trace their roots to classical graph theory. Understanding path and circuit existence conditions is the foundation for efficient search. This section covers four foundational problems: Eulerian paths, Hamiltonian paths, the Chinese Postman Problem, and the Travelling Salesman Problem.

1.1 Eulerian Paths & Circuits (欧拉路径与欧拉回路)

An Eulerian circuit (欧拉回路) is a cycle that traverses every edge of a graph exactly once and returns to the starting vertex. An Eulerian path (欧拉路径) traverses every edge exactly once but does not necessarily return to the start.

Necessary and Sufficient Conditions

Graph Type Eulerian Circuit Eulerian Path
Undirected Connected + all vertices have even degree Connected + exactly 0 or 2 vertices have odd degree
Directed Strongly connected + every vertex has out-degree = in-degree Strongly connected + at most one vertex has out-degree = in-degree + 1, and at most one has in-degree = out-degree + 1
Complexity: Eulerian path/circuit existence can be verified in \(O(V + E)\) time — a relatively easy problem compared to Hamiltonian problems below.

1.2 Hamiltonian Paths & Circuits (哈密顿路径与哈密顿回路)

A Hamiltonian circuit (哈密顿回路) visits every vertex exactly once and returns to the start. A Hamiltonian path visits every vertex exactly once without necessarily returning.

Sufficient Conditions

Theorem Condition Type
Ore's Theorem For every pair of non-adjacent vertices \(u, v\): \(\deg(u) + \deg(v) \geq n\) Sufficient
Dirac's Theorem If \(n \geq 3\) and every vertex has degree \(\geq n/2\) Sufficient (special case of Ore)
Necessary Condition For any non-empty proper subset \(S\): \(k(G - S) \leq |S|\) where \(k\) = number of connected components Necessary
NP-Completeness: Determining whether a Hamiltonian circuit exists is NP-Complete. The optimization variant (finding the shortest Hamiltonian circuit, i.e., TSP) is NP-Hard. No polynomial-time algorithm is known.

2. Uninformed Search Strategies (无信息搜索)

Uninformed (blind) search strategies use no domain-specific knowledge beyond the problem definition. They systematically explore the state space according to different rules. The four classical methods are DFS, BFS, IDDFS, and Backtracking.

2.1 Depth-First Search (DFS, 深度优先搜索)

DFS explores as deep as possible along each branch before backtracking. It uses a stack (or recursion).

  • Goal: Traverse all nodes; goes deep before going wide.
  • Implementation: Recursive or explicit stack.
    • **哈密顿回路存在的充要条件**： - **必要条件**：若图存在哈密顿回路，则对于图的任意非空真子集 $S$，有 $k(G-S) \leq |S|$（其中 $k(G-S)$ 表示移除 $S$ 后连通分量个数） - **充分条件（Ore定理）**：对于任意两个不相邻的顶点 $u, v$，有 $deg(u) + deg(v) \geq n$，则图存在哈密顿回路 - **充分条件（Dirac定理）**：若 $n \geq 3$ 且每个顶点的度都 $\geq n/2$，则图存在哈密顿回路
  • Pros: Low memory footprint (\(O(m)\) for trees, \(O(V)\) for graphs).
    • 旅行商问题（Travelling Salesman Problem, TSP）
  • Cons: Can get stuck in deep, useless branches; does not guarantee shortest path.
Complexity:
Graph: \(O(V + E)\)  |  Tree: \(O(b^m)\) time, \(O(m)\) space

2.2 Breadth-First Search (BFS, 广度优先搜索)

BFS explores level by level, visiting all nodes at depth \(k\) before any node at depth \(k+1\). It uses a queue.

2.3 Iterative Deepening DFS (IDDFS, 迭代加深搜索)

IDDFS performs repeated depth-limited DFS with increasing depth limits: \(0, 1, 2, \ldots, d\). It combines the space efficiency of DFS with the completeness and optimality of BFS.

  • Goal: Find the shallowest solution with minimal memory.
    • **启发函数**： $$ h(\mathbf{q}) = \frac{\|\mathbf{q} - \mathbf{q}_{\text{goal}}\|_2}{v_{\max}} + k \cdot \text{angle\_diff}(\theta, \theta_{\text{goal}}) $$
  • Implementation: Multiple rounds of depth-limited DFS.
  • Pros: Complete, optimal (unit cost), and uses only \(O(bd)\) space.
  • Cons: Repeated work on shallow nodes — but total cost is still \(O(b^d)\) (same order as BFS).
    • **定义**：启发函数 $h$ 称为**一致的**（consistent）或**相容的**（compatible），若对所有边 $(n, n')$ 满足：
Key Insight: IDDFS is often the preferred uninformed search for large state spaces when the solution depth is unknown. The overhead of re-expanding shallow nodes is small: for a branching factor \(b \geq 2\), the overhead factor is at most \(\frac{b}{b-1} \leq 2\).

2.4 Backtracking Search (回溯搜索)

Backtracking is an algorithm design paradigm for solving constraint satisfaction and combinatorial search problems. It is essentially DFS with constraint checking and pruning on an implicit solution-space tree.

  • Goal: Find a feasible/optimal solution satisfying constraints — not just traversal.
  • Constraint check: At each step, verify if the partial solution is viable.
    • **启发函数**： $$ h(\mathbf{q}) = \frac{\|\mathbf{q} - \mathbf{q}_{\text{goal}}\|_2}{v_{\max}} + k \cdot \text{angle\_diff}(\theta, \theta_{\text{goal}}) $$
  • Pruning: Abandon a branch as soon as it is known to be infeasible.
    • **相容性（Compatibility）**就是**一致性（Consistency）**的同义词，在中文文献中有时也称为**单调性（Monotonicity）**。它们都指：
  • Applications: N-Queens, Sudoku, permutation/combination generation, CSPs, knapsack.
Complexity:
Worst case: \(O(b^d)\)  |  Actual: often dramatically less due to pruning  |  Space: \(O(d)\)

One-Line Distinctions

DFS: Go deep, then backtrack — for traversal.
BFS: Go wide first — for shortest paths.
IDDFS: BFS-like shallowest solution, DFS-like memory.
Backtracking: DFS + constraints + pruning — for finding feasible solutions.

2.5 Dynamic Programming & the Markov Property (动态规划与马尔可夫性)

Markov Property (马尔可夫性): Once the current state is determined, the future evolution depends only on the current state, not on the history of how we arrived there. All information needed for future decisions is compressed into the "state."

Role in Dynamic Programming

  • DP exploits the Markov property to decompose a complex problem into overlapping subproblems.
  • As long as the state fully captures all information needed for future decisions, we can write a recurrence relation (状态转移方程).
  • If the state design is incomplete, the Markov property is lost and correct recursion becomes impossible.
Typical Formulation:
State: \(dp[i]\) or \(dp[i][j]\)
Transition: \(dp[i]\) is computed only from smaller-scale states
Examples: shortest paths, knapsack, sequence alignment — all rely on "current state determines the future."

3. Heuristic Search Foundations (启发式搜索基础)

Heuristic search (启发式搜索) uses domain-specific knowledge to guide the search toward the goal via an evaluation function \(f(n)\). The core idea: no longer search blindly, but steer toward the goal with informed estimates.

3.1 Evaluation Function & Heuristic Function (评估函数与启发函数)

Heuristic Function \(h(n)\) — 启发函数

A heuristic function estimates the cost from node \(n\) to the nearest goal:

$$h(n) \approx \text{minimum cost from } n \text{ to the goal}$$

Key properties of \(h(n)\):

Evaluation Function \(f(n)\) — 评估函数

The evaluation function determines node expansion order in the priority queue:

```python def A_star(start, goal, graph): frontier = PriorityQueue() frontier.put((0, start)) # (f_value, node) came_from = {start: None} g = {start: 0} h = lambda n: heuristic(n, goal) # 启发函数
$$f(n) = g(n) + h(n)$$
  • \(g(n)\): actual cost from start to node \(n\) (已知代价)
  • \(h(n)\): estimated cost from \(n\) to goal (启发估计)
  • \(f(n)\): estimated total cost of the path through \(n\) (总预计代价)

3.2 Admissibility (可采纳性)

Definition 定义

A heuristic \(h(n)\) is admissible if it never overestimates the true optimal cost to the goal:

```python def A_star(start, goal, graph): frontier = PriorityQueue() frontier.put((0, start)) # (f_value, node) came_from = {start: None} g = {start: 0} h = lambda n: heuristic(n, goal) # 启发函数

\(h(n) \leq h^*(n), \quad \forall n\)

where \(h^*(n)\) is the true optimal cost from \(n\) to the goal.

```python def A_star(start, goal, graph): frontier = PriorityQueue() frontier.put((0, start)) # (f_value, node) came_from = {start: None} g = {start: 0} h = lambda n: heuristic(n, goal) # 启发函数

Significance: Admissibility is necessary and sufficient for A* to guarantee finding the optimal solution.

**可采纳性（Admissibility）**：

Intuition: An admissible heuristic is always optimistic — it underestimates or exactly estimates the true cost. It never "gives up" on a path that might turn out to be optimal.

```python def A_star(start, goal, graph): frontier = PriorityQueue() frontier.put((0, start)) # (f_value, node) came_from = {start: None} g = {start: 0} h = lambda n: heuristic(n, goal) # 启发函数

Relaxation principle: Admissible heuristics are often derived by relaxing (removing) some constraints of the original problem and solving the easier problem exactly.

Counter-example: If you estimate "Beijing to Shanghai = 2000 km" but the true shortest path is 1200 km, the heuristic overestimates and is not admissible. A* with this heuristic may miss the optimal path.

3.3 Consistency / Monotonicity (一致性 / 单调性)

Definition 定义

A heuristic \(h(n)\) is consistent (also called monotone or compatible) if for every edge \((n, n')\) with cost \(c(n, n')\):

**定义**：启发函数 $h$ 称为**一致的**（consistent）或**相容的**（compatible），若对所有边 $(n, n')$ 满足：

\(h(n) \leq c(n, n') + h(n')\)

This is the triangle inequality — the direct estimate cannot exceed going one step plus the remaining estimate.

Physical intuition (navigation): \(h(n)\) = straight-line distance from current position to destination. \(c(n, n')\) = actual road distance to the next town. \(h(n')\) = straight-line distance from there to destination. The triangle inequality guarantees the estimate is internally coherent.

3.4 Dominance (优势性)

If two heuristics \(h_1\) and \(h_2\) are both admissible, and

def compare(n1, n2): if f[n1] != f[n2]: return f[n1] < f[n2] else: return h[n1] < h[n2] # 偏好更接近目标的节点 ```
$$h_1(n) \leq h_2(n) \leq h^*(n) \quad \forall n$$

then \(h_2\) dominates \(h_1\) (支配). A* with \(h_2\) will expand fewer nodes (or equal) than with \(h_1\), because \(h_2\) provides tighter estimates.

Practical implication: Always prefer the tighter (higher-valued but still admissible) heuristic. For the 8-puzzle, Manhattan distance dominates the misplaced-tiles count, so A* with Manhattan distance explores fewer nodes.

Combining Heuristics

If \(h_1, h_2, \ldots, h_k\) are all admissible, then:

def compare(n1, n2): if f[n1] != f[n2]: return f[n1] < f[n2] else: return h[n1] < h[n2] # 偏好更接近目标的节点 ```
$$h(n) = \max(h_1(n), h_2(n), \ldots, h_k(n))$$

is also admissible and dominates every individual \(h_i\). This is a powerful technique for combining multiple heuristic estimates.

3.5 Heuristic Design Methods (启发函数设计方法)

Method 1: Problem Relaxation (松弛法)

Remove some constraints from the original problem. The optimal cost of the relaxed problem is an admissible heuristic for the original.

Example — 8-Puzzle: Original constraint: tiles can only slide to adjacent positions. Relaxation: allow any tile to move to any position in one step. The relaxed solution cost = Manhattan distance.

Method 2: Pattern Databases (模式数据库)

Pre-compute optimal costs for sub-problems and store them in a lookup table. At runtime, query the table for an admissible estimate.

Example — 15-Puzzle: Pre-compute distances for a subset of tiles (e.g., the first 10); use the table as a heuristic.

4. The A* Algorithm (A* 算法)

A* is the most widely used heuristic search algorithm. It expands nodes in order of increasing \(f(n) = g(n) + h(n)\), combining the actual cost-so-far with a heuristic estimate of the remaining cost.

```python def A_star(start, goal, graph): frontier = PriorityQueue() frontier.put((0, start)) # (f_value, node) came_from = {start: None} g = {start: 0} h = lambda n: heuristic(n, goal) # 启发函数

4.1 Pseudocode

Algorithm A-Star(start, goal, graph, h)
// h: admissible heuristic function
1. OpenList ← priority queue containing (f(start), start)
2. ClosedSet ← {}
3. g[start] ← 0
4. f[start] ← g[start] + h(start)
5. came_from[start] ← null
6. while OpenList is not empty:
7. current ← node in OpenList with minimum f value
8. if current == goal:
9. return reconstruct_path(came_from, current)
10. remove current from OpenList
11. add current to ClosedSet
12. for each neighbor of current:
13. if neighbor in ClosedSet: continue
14. tentative_g ← g[current] + cost(current, neighbor)
15. if neighbor not in OpenList:
16. g[neighbor] ← tentative_g
17. f[neighbor] ← tentative_g + h(neighbor)
18. came_from[neighbor] ← current
19. add (f[neighbor], neighbor) to OpenList
20. else if tentative_g < g[neighbor]:
21. g[neighbor] ← tentative_g
22. f[neighbor] ← tentative_g + h(neighbor)
23. came_from[neighbor] ← current
24. // re-heapify OpenList
25. return Failure // no path found

In real driving, environments are dynamic — obstacles appear and disappear. D* Lite avoids re-searching the entire map when changes occur.

8.3 Christofides Algorithm 算法 (for metric TSP)

Christofides is a 1.5-approximation algorithm for TSP when the distance function satisfies the triangle inequality.

Christofides Algorithm 算法 Steps

  1. Compute the Minimum Spanning Tree (MST) of the graph.
    2. **近似算法**： - **最近邻算法**：从某点出发，每次选择最近的未访问点 - **最小生成树算法**：基于MST的近似解，路径长度 $\leq 2 \times$ 最优解（满足三角不等式） - **Christofides算法**：基于MST + 最小权匹配，路径长度 $\leq 1.5 \times$ 最优解（需满足三角不等式）
  2. Identify all odd-degree vertices in the MST.
    3. **配置空间（Configuration Space）**： - 将车辆和环境映射到状态空间 - 对于平面移动的车：$\mathbf{q} = (x, y, \theta)$（位置 + 朝向） - 对于更复杂的车（如非完整约束）：$\mathbf{q} \in \mathbb{R}^n$
  3. Compute a minimum-weight perfect matching among odd-degree vertices.
    4. **步骤**： 1. 构造图的最小生成树 MST 2. 找出 MST 中所有奇度顶点 3. 在奇度顶点间计算最小权完美匹配（Minimum-Weight Perfect Matching） 4. 合并 MST 和匹配的边，形成欧拉图 5. 在欧拉图上求欧拉回路 6. 跳过重复顶点，得到哈密顿回路近似解
  4. Merge MST edges and matching edges — the result is an Eulerian multigraph.
  5. Find an Eulerian circuit in the merged graph.
    6. 中国邮递员问题（Chinese Postman Problem）
  6. Shortcut repeated vertices to obtain a Hamiltonian circuit.
    7. **相容性（Compatibility）**就是**一致性（Consistency）**的同义词，在中文文献中有时也称为**单调性（Monotonicity）**。它们都指：

Guarantee: \(C \leq 1.5 \times C^*\)  |  Complexity: \(O(n^3)\)

**哈密顿回路存在的充要条件**： - **必要条件**：若图存在哈密顿回路，则对于图的任意非空真子集 $S$，有 $k(G-S) \leq |S|$（其中 $k(G-S)$ 表示移除 $S$ 后连通分量个数） - **充分条件（Ore定理）**：对于任意两个不相邻的顶点 $u, v$，有 $deg(u) + deg(v) \geq n$，则图存在哈密顿回路 - **充分条件（Dirac定理）**：若 $n \geq 3$ 且每个顶点的度都 $\geq n/2$，则图存在哈密顿回路

Practical Application Guide

Scenario Recommended Strategy Rationale
GPS navigation (exact route) A* (heuristic) Optimal route needed; seconds of delay acceptable
Real-time obstacle avoidance RRT (approximation) Fast feasible path; optimality not critical
Large-scale graph search Weighted A* (hybrid) Balance optimality and speed
TSP (metric, medium n) Christofides (approximation) 1.5× guarantee; \(O(n^3)\) vs. exponential exact

9. Summary (总结)

Core Takeaways

  1. Eulerian paths exist iff even degrees (easy to check); Hamiltonian problems are NP-Complete.
  2. IDDFS is the preferred uninformed search for memory-limited settings — it combines BFS's completeness/optimality with DFS's space efficiency.
    3. - 图遍历与经典图问题：欧拉、哈密顿、中国邮递员、TSP - 无信息搜索：DFS、BFS、IDDFS、回溯 - 启发式搜索：启发函数、评估函数、A* 与性质分析 - 工程应用：自动驾驶路径规划与启发式/近似策略权衡
  3. Dynamic programming exploits the Markov property: current state encodes all information needed for future decisions.
  4. The evaluation function \(f(n) = g(n) + h(n)\) balances actual cost-so-far with estimated remaining cost.
    5. 启发式搜索的核心由两个关键函数组成：**evaluation function（评估函数）** 和 **heuristic function（启发函数）**。理解它们的区别与联系至关重要。
  5. An admissible heuristic never overestimates: \(h(n) \leq h^*(n)\). This is required for A* optimality.
    6. **核心条件**：启发函数必须可采纳（admissible）： $$ 0 \leq h(n) \leq h^*(n) \quad \forall n $$
  6. A consistent heuristic satisfies the triangle inequality: \(h(n) \leq c(n, n') + h(n')\). Consistency \(\Rightarrow\) admissibility and prevents node re-expansion.

The Big Picture

Search is the backbone of AI decision-making. From classical graph theory (Euler, Hamilton) through blind search (DFS, BFS, IDDFS) to informed search (A*, Hybrid A*), the key progression is: systematic exploration → informed guidance → domain-specific optimization. Understanding when to use exact heuristic search versus approximation is a critical engineering judgment.
