# 第6讲：无监督学习 — K-Means、PCA 与 GMM

---

## 本讲概览

本讲围绕**无监督学习**展开，核心主线可以概括为两句话：

- **化繁为简**：把高维、复杂的数据压缩成更简单、更易处理的形式
  - 聚类（Clustering）：把相似的数据放在一起
  - 降维（Dimensionality Reduction）：把冗余特征压缩掉
- **无中生有**：用生成模型去解释数据从哪里来
  - 例如高斯混合模型（GMM）

与监督学习的本质区别在于：无监督学习**没有标签**，模型只能依赖特征本身去寻找数据中的结构。

## 1. 监督学习 vs 无监督学习

| | 监督学习 | 无监督学习 |
|---|---|---|
| 数据 | $\{(x_i, y_i)\}_{i=1}^n$ | $\{x_i\}_{i=1}^n$ |
| 目标 | 学习映射 $f(x) \approx y$ | 发现数据本身的结构 |
| 关键 | 标签质量决定上限 | 特征选择与相似度定义决定上限 |

课堂强调了一个核心观点：**相似性不是天然存在的，而是由特征和距离度量共同定义的。** 例如：
- 图像分割时可以根据颜色、纹理判断像素是否同区域
- 分子分类时可以根据 atomic features 判断是否同类

因此无监督学习的第一步往往不是算法本身，而是**"特征怎么选、距离怎么算"**。

## 2. K-Means 聚类

**目标**：将数据分成 $K$ 个簇，最小化簇内平方和

$$J = \sum_{i=1}^n \|x_i - \mu_{z_i}\|^2$$

**算法流程**（交替优化）：

1. 随机初始化 $K$ 个中心点
2. **分配步**：$z_i = \arg\min_k \|x_i - \mu_k\|^2$
3. **更新步**：$\mu_k = \frac{1}{n_k}\sum_{i:z_i=k} x_i$
4. 重复 2-3 直到收敛

**优点**：简单快速、易于实现
**缺点**：
- 需预先指定 $K$
- 对初始中心敏感，只保证局部最优
- 假设簇形状近似球形
- 对异常值敏感

**DBSCAN** 作为补充：基于密度的聚类，无需指定 $K$，能发现任意形状的簇，但对参数 $\varepsilon$ 和 MinPts 敏感。

## 3. PCA：主成分分析

**目标**：找到一组正交方向，使数据投影后的方差最大（等价于重构误差最小）。

### 数学推导

1. **中心化**：$\tilde{x}_i = x_i - \bar{x}$
2. **协方差矩阵**：$S = \frac{1}{n} X_c^T X_c$
3. **特征值分解**：$Sw = \lambda w$
4. **选前 $k$ 个最大特征值对应的特征向量**组成投影矩阵 $W$

核心结论：

$$\max_{\|w\|=1} w^T S w \quad \Longrightarrow \quad Sw = \lambda w$$

**解释方差比**：$\text{EVR}(k) = \frac{\sum_{j=1}^k \lambda_j}{\sum_{j=1}^d \lambda_j}$

**应用**：人脸识别（Eigenfaces）、数据压缩、可视化、去噪。

---

## 4. GMM 与 EM 算法

**模型**：假设数据由 $K$ 个高斯分布混合生成

$$p(x) = \sum_{k=1}^K \pi_k \mathcal{N}(x \mid \mu_k, \Sigma_k)$$

**EM 算法**（交替迭代）：

| 步骤 | 做什么 | 类比 K-Means |
|---|---|---|
| E-step | 计算责任度 $\gamma_{ik} = p(z_i=k \mid x_i)$ | 分配步（但是软分配） |
| M-step | 更新 $\pi_k, \mu_k, \Sigma_k$ | 更新步（但允许协方差不同） |

**GMM vs K-Means**：

| | K-Means | GMM |
|---|---|---|
| 分配方式 | 硬分配（0/1） | 软分配（概率） |
| 簇形状 | 球形 | 椭球形 |
| 本质 | GMM 的特殊极限情况 | 更一般的概率模型 |

---

## 5. 三个算法的统一视角

| 算法 | 任务 | 关键思想 |
|---|---|---|
| K-Means | 聚类 | 最近中心 + 均值更新 |
| PCA | 降维 | 最大方差 / 最小重构误差 |
| GMM | 生成建模 + 软聚类 | 概率生成 + EM |

**一句话总结本讲**：

> 在没有标签的情况下，先用特征定义相似性，再用聚类、降维和生成模型去发现数据结构。

## 6. 深度思考：维度与无监督学习 — 如何理解"无监督学习是降维"

### 6.1 什么是维度

维度（Dimension）是描述一个数据点所需的**独立特征数量**。

- 一张 $100 \times 100$ 的灰度图，维度是 10000
- 一个学生的数据如果有身高、体重、年龄、成绩等 10 个特征，维度就是 10

维度高不等于信息多。高维数据中往往存在大量**冗余**和**噪声**——某些特征之间高度相关，某些特征几乎不携带有用信息。

### 6.2 为什么说"无监督学习是降维"

这句话的本质是：**无监督学习的核心目标就是从高维、冗余的原始数据中，提取出更紧凑、更本质的低维表示。**

可以从三个层次理解：

#### 第一层：PCA 是字面意义上的降维

PCA 直接把 $d$ 维数据压缩到 $k$ 维（$k < d$）。它做的事情就是：
- 找到数据中**方差最大**的方向（最重要的结构）
- 丢弃方差小的方向（噪声和冗余）
- 用更少的维度"说清楚"数据的分布

这是最直观的"降维"。

#### 第二层：聚类也是一种降维

原始数据有 $d$ 维特征，聚类后每个样本只得到一个**离散的簇标签** $z_i \in \{1, 2, \dots, K\}$。

从信息的角度看：
- 输入：$d$ 维连续向量
- 输出：1 维离散值（簇编号）

这就是一种降维——把连续的高维信息压缩成了离散的低维标签。K-Means 的优化目标 $\min \sum \|x_i - \mu_{z_i}\|^2$ 本质上就是在衡量"用 $K$ 个中心点代表所有数据点"时的**信息损失**。

#### 第三层：GMM 的隐变量也是降维

GMM 引入隐变量 $z_i$（样本来自哪个高斯分量），然后通过 EM 算法去学习。最终每个样本被赋予一个 $K$ 维的概率向量（责任度）：

$$\gamma_i = [\gamma_{i1}, \gamma_{i2}, \dots, \gamma_{iK}], \quad \sum_k \gamma_{ik} = 1$$

这也是降维：原始数据从 $d$ 维空间被映射到了一个 $K$ 维的**概率单纯形**（simplex）上。

### 6.3 统一框架

```
原始数据（高维、冗余、有噪声）
        │
        ▼
   无监督学习算法
        │
   ┌────┼────┐
   ▼    ▼    ▼
  PCA  聚类  GMM
   │    │    │
   ▼    ▼    ▼
低维    簇    概率
连续    标签  表示
表示    (离散) (K维)
(k维)
```

三种方法虽然形式不同，但本质都是：

> **从高维输入中提取更紧凑的表示，丢弃冗余和噪声，保留最核心的结构信息。**

### 6.4 更深一层：表示学习

从现代机器学习的视角看，无监督学习和降维的关系可以进一步推广为**表示学习（Representation Learning）**：

- 降维学习的是**低维表示**（如 PCA 的投影）
- 聚类学习的是**离散表示**（如 K-Means 的簇标签）
- GMM 学习的是**概率表示**（如责任度向量）
- 自编码器学习的是**非线性表示**（如 bottleneck 层的编码）

它们都在做同一件事：**找到一种比原始特征更精炼、更有意义的表示方式。**

所以，"无监督学习是降维"不是说无监督学习只有一个算法叫降维，而是说：**无监督学习的本质目标就是发现数据中真正重要的低维结构，去除冗余，提取本质。**

## 7. 表示学习（Representation Learning）的哲学

### 7.1 一个根本问题：什么是"好的表示"

表示学习问的不是"怎么预测标签"，而是一个更根本的问题：

> **数据背后有没有一个更简洁、更真实的形式？如果有，我们能不能找到它？**

这个问题在哲学上至少可以追溯到两个传统：

- **柏拉图的理型论**：我们看到的具体事物（马、椅子、美）只是永恒"理型"（Form/Idea）的不完美投影。表示学习做的事情惊人地相似——从纷繁的表象中提取出稳定的抽象结构。
- **奥卡姆剃刀**：如无必要，勿增实体。一个好的表示应该是**用最少的参数抓住最多的本质**。PCA 选择前 $k$ 个主成分、聚类用 $K$ 个中心代表全部数据，都是这条原则的数学实现。

### 7.2 表示的本质：编码与解码的对称性

一个好的表示必须满足一个条件：**能压缩，也能还原。**

```
原始数据  ──编码器 f──▶  表示  ──解码器 g──▶  重构数据
   x                    z                      x̂
```

- **PCA**：编码是投影 $z = W^T x$，解码是重构 $\hat{x} = W z$
- **自编码器**：编码和解码都是神经网络，可以学非线性表示
- **GMM**：编码是推断隐变量 $p(z|x)$，解码是从隐变量生成 $p(x|z)$

这里蕴含着一个深刻的哲学立场：

> **如果你不能从一个表示中重建出原始世界的主要特征，那么这个表示就遗漏了重要的东西。**

这不是"有用"或"高效"的工程考量，而是关于**什么才算"理解"**的认识论问题——理解一个事物，意味着你能抓住它的本质结构，以至于能从本质出发重新生成它。

### 7.3 人类认知本身就是表示学习

我们的大脑每时每刻都在做无监督的表示学习：

- 视觉系统：视网膜接收上亿像素的光信号，但大脑最终提取的是"边缘、纹理、形状、物体"——这就是从高维到低维的编码
- 语言：听到一串声波，大脑提取出"音素 → 单词 → 句子 → 含义"，每一层都是更低维、更抽象的表示
- 记忆：我们记不住一个陌生人的所有像素级细节，但能记住"高个子、戴眼镜、声音低沉"——这是大脑自动进行的降维压缩

有趣的是，研究表明**视觉皮层对自然图像的反应模式与 PCA 的主成分高度吻合**（Olshausen & Field, 1996）。这意味着进化本身就在做表示学习。

所以，表示学习不是"机器学人"，而是**"机器在做人脑一直在做的事"**。

### 7.4 监督与无监督的根本分歧：理解 vs 预测

| | 监督学习 | 无监督 / 表示学习 |
|---|---|---|
| 核心问题 | 给定 $x$，预测 $y$ 是什么？ | 给定 $x$，$x$ 本身的结构是什么？ |
| 哲学姿态 | 实用主义：只要预测准就行 | 本质主义：想搞清楚数据"是什么" |
| 依赖 | 人类给的标签 | 数据自身的统计规律 |
| 类比 | 考试找标准答案 | 自己琢磨世界是怎么回事 |

监督学习像一个学生：老师给答案，学生学规律。能力很强，但**完全依赖老师给的任务定义**。

无监督学习像一个婴儿：没有标准答案，只有感官输入，却从中自发形成了关于世界的概念——物体、颜色、因果、语言。这种能力也许才是"智能"更深层的部分。

这也是为什么 Yann LeCun 说：

> **"如果智能是一个蛋糕，无监督学习是蛋糕的胚子，监督学习是蛋糕上的糖霜。"**

### 7.5 从"特征工程"到"特征学习"的历史转向

| 阶段 | 做法 | 谁来定义"好的表示" |
|---|---|---|
| 传统机器学习 | 人工设计特征（SIFT、HOG、TF-IDF） | 人类领域专家 |
| PCA / 降维 | 从数据中自动提取线性表示 | 数据的统计结构 |
| 深度学习 | 用神经网络学习非线性表示 | 数据 + 网络架构 |
| 自监督学习（BERT、GPT 等） | 设计前置任务让模型自己学表示 | 数据本身 + 代理任务 |

这个演进过程的哲学含义是：**"好的表示"的定义权正在从人逐步交给数据本身。** 我们越来越不告诉机器"应该看什么"，而是让机器自己去发现"什么是值得看的"。

### 7.6 一个开放的问题

如果表示学习真的是智能的核心，那么一个终极问题是：

> **世界本身存在一个"最优表示"吗？还是说，"好的表示"总是相对于任务而存在的？**

- 如果存在最优表示，那么无监督学习可以独立于任何下游任务去追求它——这是纯粹的、自足的智能
- 如果表示总是相对于任务的，那么不存在脱离语境的"理解"，任何表示的好坏都取决于你要用它做什么

这个问题至今没有定论。但无论答案是什么，表示学习都迫使我们思考一个古老的认识论问题：**我们所"理解"的世界，到底是世界的本来面目，还是我们大脑为了高效运作而构造的压缩编码？**
