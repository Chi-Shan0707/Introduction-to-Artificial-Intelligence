1. Overview and Main Themes 1. 概述与主线

This lecture addresses the fundamental challenge of unsupervised learning: discovering structure in data without labeled examples. The core themes can be summarized by two principles:

Core Principles

  • Simplification (化繁为简): Compress high-dimensional, complex data into simpler, more manageable forms.
    • Clustering: Group similar data points together.
    • Dimensionality Reduction: Remove redundant features while preserving essential structure.
      • 2.2 降维（Dimensionality Reduction）
  • Generation (无中生有): Use generative models to explain where data comes from.
    • Gaussian Mixture Models (GMM): Model data as generated from multiple distributions; once we have the distribution, we can sample new data points.

The essential difference from supervised learning is the absence of labels. The model must rely solely on the features themselves to discover patterns and structure. This means the critical questions become:

  1. What features represent the data?
    2. - 图像分割时，可以根据颜色、纹理、亮度等特征判断像素是否属于同一区域 - 分子分类时，可以根据 atomic features 判断原子或分子是否属于同类
  2. What similarity metric captures the relationship between data points?
  3. What model organizes these relationships into useful structure?
Key Insight: Similarity is not intrinsic to data — it is defined by the choice of features and distance metrics. For example, image segmentation may group pixels by color, texture, or brightness; molecular classification may use atomic features. The first step in unsupervised learning is therefore not the algorithm itself, but "what features to use and how to measure distance."

2. Supervised vs. Unsupervised Learning 2. 监督学习与无监督学习

In supervised learning, the model learns from paired examples \((x, y)\). Each training sample comes with a label that tells the model the "correct answer." In unsupervised learning, only inputs \(\{x_1, x_2, \ldots, x_n\}\) are provided — no labels — and the model must find structure without guidance.

Aspect Supervised Learning Unsupervised Learning
Data \(\{(x_i, y_i)\}_{i=1}^n\) \(\{x_i\}_{i=1}^n\)
Goal Learn mapping \(f(x) \approx y\) Discover structure in data itself
Key Factor Label quality determines upper bound Feature selection and similarity definition determine upper bound
Example Tasks Classification, regression Clustering, dimensionality reduction, density estimation
Evaluation Compare predictions to ground truth labels Intrinsic metrics (silhouette, reconstruction error) or downstream task performance
Core Observation: In supervised learning, labels define "what is correct." In unsupervised learning, the choice of features and distance defines "what is similar." Two different feature representations of the same data can lead to completely different cluster structures.

3. Clustering 3. 聚类

Clustering is the task of grouping a set of objects such that objects in the same group (cluster) are more similar to each other than to those in other groups. It is the most direct form of "simplification" in unsupervised learning.

3.1 K-Means Algorithm 算法

K-Means is one of the most widely used clustering algorithms. Its objective is to partition \(n\) observations into \(K\) clusters, minimizing the within-cluster sum of squares (WCSS). The core idea is intuitive:

  1. Choose \(K\) initial center points (centroids).
  2. Assign each sample to the nearest centroid.
  3. Update each centroid to the mean of its assigned points.
  4. Repeat until convergence.

3.1.1 Objective Function

Given a dataset:

第6讲：无监督学习、K-Means、PCA 与 GMM
$$X = \{x_1, x_2, \ldots, x_n\}, \quad x_i \in \mathbb{R}^d$$

Let \(z_i \in \{1, 2, \ldots, K\}\) denote the cluster assignment of sample \(x_i\), and \(\mu_k\) denote the centroid of cluster \(k\). The K-Means objective is to minimize the within-cluster sum of squares:

- $z_i \in \{1,2,\dots,K\}$ 表示样本 $x_i$ 的簇编号 - $\mu_k$ 表示第 $k$ 个簇的中心
$$J = \sum_{i=1}^n \|x_i - \mu_{z_i}\|^2 = \sum_{k=1}^K \sum_{i: z_i=k} \|x_i - \mu_k\|^2$$

This objective says: make every sample as close as possible to the center of its own cluster.

3.1.2 Alternating Optimization (Coordinate Descent)

Since the objective is non-convex in both assignments and centroids simultaneously, K-Means uses an alternating optimization strategy — analogous to the E-step/M-step pattern in EM:

| 算法 | 任务 | 输出 | 关键思想 | 优点 | 局限 | |---|---|---|---|---|---| | K-Means | 聚类 | 簇标签、中心 | 最近中心 + 均值更新 | 简单快速 | 只能处理近似球形簇 | | PCA | 降维 | 低维表示、投影矩阵 | 最大方差 / 最小重构误差 | 数学清晰、可解释性强 | 只能学线性结构 | | GMM | 生成建模 / 软聚类 | 混合参数、责任度 | 概率生成 + EM | 表达能力强 | 计算更重、易陷局部最优 |

Assignment Step (E-step analog): Given fixed centroids, assign each point to its nearest centroid. For each sample \(x_i\):

这一步叫 **E-step**，即 **Expectation step**。它做的事情不是直接改参数，而是计算“当前参数下，隐变量最可能是什么”。
$$z_i = \arg\min_{k} \|x_i - \mu_k\|^2$$

This is the "nearest center" rule. When centroids are fixed, the objective decomposes over individual points, and each point independently chooses its closest center.

Update Step (M-step analog): Given fixed assignments, update each centroid as the mean of its assigned points. For cluster \(k\) with \(n_k\) points:

$$\mu_k = \frac{1}{n_k} \sum_{i: z_i=k} x_i$$

3.1.3 Derivation of the Centroid Update

The centroid update is not arbitrary — it is the exact minimizer of the per-cluster objective. For cluster \(k\), define:

$$J_k(\mu_k) = \sum_{i: z_i=k} \|x_i - \mu_k\|^2$$

Take the gradient with respect to \(\mu_k\) and set it to zero:

$$\frac{\partial J_k}{\partial \mu_k} = \sum_{i: z_i=k} 2(\mu_k - x_i) = 2n_k\mu_k - 2\sum_{i: z_i=k} x_i = 0$$

Solving yields:

$$\mu_k = \frac{1}{n_k}\sum_{i: z_i=k} x_i$$

This confirms that the optimal centroid is the mean of its cluster members. Since each step (assignment or update) decreases or maintains the objective \(J\), and \(J \geq 0\), the algorithm is guaranteed to converge — though only to a local optimum.

3.1.4 K-Means Pseudocode

Algorithm K-Means(X, K, max_iter, tol)
// Input: Data matrix X (n x d), number of clusters K
// Output: Cluster labels z and centroids mu

1. mu = InitializeCentroids(X, K) // Random or K-Means++
2. for iter = 1 to max_iter do
3. mu_old = copy(mu)
4. // Assignment Step
5. for each x_i in X do
6. z_i = argmin_k ||x_i - mu_k||^2
7. end for
8. // Update Step
9. for k = 1 to K do
10. C_k = {x_i : z_i = k}
11. if |C_k| > 0 then
12. mu_k = (1/|C_k|) * sum(C_k)
13. else
14. mu_k = Reinitialize(X) // Handle empty cluster
15. end if
16. end for
17. if ||mu - mu_old|| < tol then
18. break
19. end if
20. end for
21. return z, mu

3.2 K-Means++ and the Elbow Method

3.2.1 K-Means++ Initialization

Random initialization can lead to poor convergence. K-Means++ (Arthur & Vassilvitskii, 2007) chooses initial centroids that are spread apart:

Algorithm K-Means++(X, K)
1. Choose first centroid mu_1 uniformly at random from X
2. for k = 2 to K do
3. For each x in X, compute D(x) = min_j ||x - mu_j||^2
4. Choose next centroid mu_k with probability proportional to D(x)^2
5. end for
6. return {mu_1, ..., mu_K}

This ensures initial centroids are far from each other, giving K-Means a much better starting point. K-Means++ guarantees an expected approximation ratio of \(O(\log K)\) on the objective.

3.2.2 The Elbow Method

To choose \(K\), run K-Means for a range of \(K\) values and plot the objective \(J(K)\). The "elbow" — where the rate of decrease sharply changes — suggests a good \(K\). Additional criteria include:

  • Silhouette Score: Measures how similar a point is to its own cluster vs. other clusters. Range \([-1, 1]\); higher is better.
  • Business/domain knowledge: Sometimes \(K\) is determined by the application (e.g., number of customer segments desired).
Practical Note: When feature scales differ significantly, standardization (zero mean, unit variance) is essential before running K-Means. Otherwise, features with larger scales dominate the Euclidean distance. K-Means++ initialization is strongly recommended.

3.3 DBSCAN: Density-Based Clustering 聚类

DBSCAN (Density-Based Spatial Clustering of Applications with Noise) is a density-based algorithm that discovers clusters of arbitrary shape and identifies outliers. Unlike K-Means, it does not require specifying \(K\) in advance.

3.3.1 Core Concepts

  • \(\varepsilon\)-neighborhood: The set of points within distance \(\varepsilon\) of a given point: \(N_\varepsilon(p) = \{q \mid d(p, q) \leq \varepsilon\}\).
  • Core point: A point \(p\) with at least \(\text{MinPts}\) neighbors within its \(\varepsilon\)-neighborhood: \(|N_\varepsilon(p)| \geq \text{MinPts}\).
    • EM 是 **Expectation-Maximization** 的缩写，中文一般叫**期望最大化算法**。
  • Directly density-reachable: Point \(q\) is directly density-reachable from core point \(p\) if \(q \in N_\varepsilon(p)\).
  • Density-reachable: A chain of directly density-reachable points forms a density-reachable path.
  • Density-connected: Two points \(p\) and \(q\) are density-connected if there exists a point \(o\) such that both \(p\) and \(q\) are density-reachable from \(o\).

3.3.2 DBSCAN Pseudocode

Algorithm DBSCAN(DB, eps, MinPts)
// Input: Database DB, radius eps, minimum points MinPts
// Output: Cluster labels

1. C = 0 // Cluster counter
2. for each unvisited point P in DB do
3. mark P as visited
4. Neighbors = RegionQuery(P, eps)
5. if |Neighbors| < MinPts then
6. mark P as NOISE
7. else
8. C = C + 1
9. ExpandCluster(P, Neighbors, C, eps, MinPts)
10. end if
11. end for

Function ExpandCluster(P, Neighbors, C, eps, MinPts)
1. assign P to cluster C
2. for each point Q in Neighbors do
3. if Q is not visited then
4. mark Q as visited
5. Neighbors_Q = RegionQuery(Q, eps)
6. if |Neighbors_Q| ≥ MinPts then
7. Neighbors = Neighbors UNION Neighbors_Q
8. end if
9. end if
10. if Q is not yet assigned to any cluster then
11. assign Q to cluster C
12. end if
13. end for

3.4 Modern Clustering 聚类 Methods

3.4.1 HDBSCAN*

HDBSCAN* (Hierarchical DBSCAN, Campello, Moulavi & Sander 2013/2015) extends DBSCAN to address its key limitations:

  • Eliminates the notion of border points — only core points form clusters.
  • Produces a hierarchical clustering (dendrogram) instead of a flat partition.
  • Extracts the most prominent clusters from the hierarchy using cluster stability.
  • More robust to parameter choices: only needs \(\text{MinPts}\); \(\varepsilon\) is derived automatically.
  • Handles varying-density clusters naturally.

3.4.2 Spectral Clustering 聚类

Spectral clustering uses the spectrum (eigenvalues) of a similarity matrix to perform dimensionality reduction before clustering. It can discover non-convex clusters that K-Means cannot:

  1. Construct a similarity graph (adjacency matrix \(A\)) from data, e.g., using an RBF kernel: \(A_{ij} = \exp(-\|x_i - x_j\|^2 / 2\sigma^2)\).
    2. - $\pi_k \ge 0$，且 $\sum_{k=1}^K \pi_k = 1$ - $\mu_k$ 是第 $k$ 个高斯的均值 - $\Sigma_k$ 是协方差矩阵
  2. Compute the degree matrix \(D = \text{diag}(d_1, \ldots, d_n)\) where \(d_i = \sum_j A_{ij}\).

  3. Compute the graph Laplacian: \(L = D - A\).

  4. Compute the first \(k\) eigenvectors of \(L\) (corresponding to the \(k\) smallest eigenvalues).
  5. Embed data into the \(k\)-dimensional eigenspace and apply K-Means in this reduced space.

The normalized Laplacian is \(L^{\text{sym}} = I - D^{-1/2}AD^{-1/2}\). Spectral clustering is closely related to normalized cuts (Shi & Malik, 2000). Computational cost: \(O(n^3)\) for dense graphs, but \(O(n)\) for sparse graphs with \(k \ll n\).

4. Principal Component Analysis 4. 主成分分析 (PCA) (PCA)

PCA is the most important mathematical tool in this lecture. It is a fundamental linear dimensionality reduction technique that finds a new orthogonal coordinate system such that the greatest variance lies on the first coordinate (first principal component), the second greatest variance on the second coordinate, and so on.

Two Equivalent Views of PCA:
  1. Maximize projected variance: Find directions along which data varies the most.
  2. Minimize reconstruction error: Find the subspace that loses the least information when projecting and reconstructing.
These two objectives are mathematically equivalent.

4.1 Intuition and Motivation

Consider data in \(\mathbb{R}^d\) that actually lies near a lower-dimensional subspace. For example, points in 3D that lie close to a plane. PCA finds this plane by identifying the directions of maximum spread. The intuition is: the directions of maximum variance carry the most information about the data's structure.

4.2 Mathematical Derivation

4.2.1 Centering

Given data matrix \(X \in \mathbb{R}^{n \times d}\) (each row is a sample), first center the data:

2.3 生成建模（Generative Modeling）
$$\bar{x} = \frac{1}{n}\sum_{i=1}^n x_i, \quad \tilde{x}_i = x_i - \bar{x}$$

Let \(X_c\) be the centered data matrix. Centering removes the mean offset so PCA captures variation around the mean, not the absolute position.

4.2.2 Maximizing Projected Variance

Seek a unit vector \(w\) (\(\|w\| = 1\)) that maximizes the variance of projections \(y_i = w^T\tilde{x}_i\):

2.2 降维（Dimensionality Reduction）
$$\text{Var}(y) = \frac{1}{n}\sum_{i=1}^n (w^T\tilde{x}_i)^2 = w^T\left(\frac{1}{n}\sum_{i=1}^n \tilde{x}_i\tilde{x}_i^T\right)w = w^T S w$$

where \(S = \frac{1}{n}X_c^T X_c\) is the sample covariance matrix. The optimization problem is:

这一步叫 **E-step**，即 **Expectation step**。它做的事情不是直接改参数，而是计算“当前参数下，隐变量最可能是什么”。
$$\max_{\|w\|=1} w^T S w$$

This is a Rayleigh quotient maximization problem.

这就是一个标准的 Rayleigh quotient 最大化问题。

4.2.3 Lagrangian and Eigenvalue Problem

Form the Lagrangian with multiplier \(\lambda\) for the unit-norm constraint:

$$\mathcal{L}(w, \lambda) = w^T S w - \lambda(w^T w - 1)$$

Taking the derivative and setting to zero:

$$\frac{\partial \mathcal{L}}{\partial w} = 2Sw - 2\lambda w = 0 \implies Sw = \lambda w$$

This is an eigenvalue equation! The optimal \(w\) must be an eigenvector of \(S\), and the projected variance equals the eigenvalue: \(w^T S w = \lambda\). To maximize variance, choose the eigenvector corresponding to the largest eigenvalue.

Fundamental Result: The principal components are the eigenvectors of the covariance matrix \(S\), ordered by decreasing eigenvalues. The first principal component \(w_1\) corresponds to the largest eigenvalue \(\lambda_1\), and \(\lambda_1\) equals the variance along that direction.

4.2.4 Extension to \(k\) Dimensions

To reduce to \(k\) dimensions, form matrix \(W = [w_1, w_2, \ldots, w_k] \in \mathbb{R}^{d \times k}\) with orthonormal columns (\(W^T W = I_k\)). The projected data is:

2.3 生成建模（Generative Modeling）
$$Z = X_c W \in \mathbb{R}^{n \times k}$$

The total variance retained is:

高斯分布（Gaussian distribution）是最常见的连续概率分布之一。单变量形式为：
$$\text{tr}(W^T S W) = \sum_{j=1}^k w_j^T S w_j = \sum_{j=1}^k \lambda_j$$

The optimization becomes:

$$\max_{W^T W = I} \text{tr}(W^T S W)$$

The optimal solution is: take the top \(k\) eigenvectors of \(S\), i.e., those corresponding to the \(k\) largest eigenvalues.

- 图像分割时，可以根据颜色、纹理、亮度等特征判断像素是否属于同一区域 - 分子分类时，可以根据 atomic features 判断原子或分子是否属于同类

4.3 Equivalence to Minimizing Reconstruction Error

PCA is also equivalent to minimizing the reconstruction error. After projecting to \(k\) dimensions, reconstruct in the original space:

$$\hat{x}_i = W W^T \tilde{x}_i$$

The reconstruction error in Frobenius norm:

$$\|X_c - X_c W W^T\|_F^2 = \sum_{i=1}^n \|\tilde{x}_i - \hat{x}_i\|^2$$

Since \(W\) has orthonormal columns, the projection and residual are orthogonal and complementary:

$$\|\tilde{x}_i\|^2 = \|W W^T \tilde{x}_i\|^2 + \|(I - W W^T)\tilde{x}_i\|^2$$

Therefore, maximizing retained variance is mathematically equivalent to minimizing lost reconstruction error. This duality provides both geometric intuition and computational flexibility.

4.4 SVD and Eigenvalue Decomposition

4.4.1 Eigenvalue Decomposition of the Covariance Matrix

$$S = U \Lambda U^T$$

where \(\Lambda = \text{diag}(\lambda_1, \lambda_2, \ldots, \lambda_d)\) with \(\lambda_1 \geq \lambda_2 \geq \cdots \geq \lambda_d \geq 0\), and \(U\) contains orthonormal eigenvectors. The top \(k\) principal components are the first \(k\) columns of \(U\).

4.4.2 Singular Value Decomposition (SVD)

In practice, it is often better to directly decompose the centered data matrix via SVD rather than forming the covariance matrix explicitly (which can lose numerical precision):

$$X_c = U \Sigma V^T$$

where \(\Sigma\) contains singular values \(\sigma_1 \geq \sigma_2 \geq \cdots \geq 0\). The principal components are the columns of \(V\), and:

$$\lambda_j = \frac{\sigma_j^2}{n}$$

SVD is numerically more stable than explicit covariance computation and is the preferred method in practice (e.g., numpy.linalg.svd, sklearn.decomposition.PCA).

4.4.3 Explained Variance Ratio

The proportion of variance explained by the first \(k\) components:

$$\text{EVR}(k) = \frac{\sum_{j=1}^k \lambda_j}{\sum_{j=1}^d \lambda_j} = \frac{\sum_{j=1}^k \sigma_j^2}{\sum_{j=1}^d \sigma_j^2}$$

This ratio helps determine how many dimensions to retain. Common heuristics:

  • Choose \(k\) such that \(\text{EVR} \geq 95\%\).
  • Plot the eigenvalues (scree plot) and look for an "elbow."
  • Use cross-validation on a downstream task.

4.5 PCA Algorithm 算法 and Pseudocode

Algorithm PCA(X, k)
// Input: Data matrix X (n x d), target dimension k
// Output: Projected data Z, projection matrix W, reconstruction X_hat

1. x_mean = mean(X, axis=0)
2. Xc = X - x_mean // Center the data
3. // Method 1: Covariance + Eigendecomposition
4. S = (Xc^T @ Xc) / n // Covariance matrix (d x d)
5. eigvals, eigvecs = eig(S) // Eigen decomposition
6. idx = argsort(eigvals, descending=True)
7. W = eigvecs[:, idx[0:k]] // Top k eigenvectors
8. // Method 2: SVD (preferred for numerical stability)
9. // U, Sigma, Vt = svd(Xc); W = Vt.T[:, 0:k]
10. Z = Xc @ W // Project to k dimensions
11. X_hat = Z @ W.T + x_mean // Optional reconstruction
12. return Z, W, X_hat

4.6 Practical Note 注意s on PCA

  • Standardization: If features have different units or scales, standardize (zero mean, unit variance) before PCA. Otherwise, features with large variance dominate.
  • Linearity: PCA is a linear method. It can only capture linear relationships. For nonlinear structure, consider kernel PCA, autoencoders, or manifold learning (t-SNE, UMAP).
  • Interpretability: Each principal component is a linear combination of original features. The loadings (elements of \(w_j\)) indicate feature importance.
  • Whitening: Dividing projected data by \(\sqrt{\lambda_j}\) decorrelates and normalizes the features: \(Z_{\text{white}} = Z \Lambda^{-1/2}\).
  • Computational cost: \(O(nd^2 + d^3)\) for eigendecomposition, or \(O(ndk)\) for truncated SVD when \(k \ll d\).

5. Gaussian Mixture Models 5. 高斯混合模型 (GMM) and the EM Algorithm

While K-Means performs "hard" assignment (each point belongs to exactly one cluster), Gaussian Mixture Models (GMM) provide a probabilistic framework for "soft" clustering, where each point has a probability of belonging to each cluster. GMM also serves as a generative model — once fitted, we can sample new data points from the learned distribution.

5.1 Gaussian Distribution Review

The Gaussian (normal) distribution is the most widely used continuous probability distribution. Its univariate form is:

高斯分布（Gaussian distribution）是最常见的连续概率分布之一。单变量形式为：
$$\mathcal{N}(x \mid \mu, \sigma^2) = \frac{1}{\sqrt{2\pi}\sigma} \exp\left(-\frac{(x-\mu)^2}{2\sigma^2}\right)$$

The multivariate form for \(x \in \mathbb{R}^d\) is:

2.3 生成建模（Generative Modeling）
$$\mathcal{N}(x \mid \mu, \Sigma) = \frac{1}{(2\pi)^{d/2}|\Sigma|^{1/2}} \exp\left(-\frac{1}{2}(x-\mu)^T\Sigma^{-1}(x-\mu)\right)$$

where:

  • \(\mu \in \mathbb{R}^d\) controls the center (mean) of the distribution.
  • \(\Sigma \in \mathbb{R}^{d \times d}\) is the covariance matrix, controlling the shape and orientation of the probability "ellipsoid."
  • \(|\Sigma|\) is the determinant of \(\Sigma\), and \(\Sigma^{-1}\) is its inverse.

If we fit a single Gaussian to a dataset, we are essentially fitting an "ellipsoidal" probability model to the data cloud. The maximum likelihood estimates are simply the sample mean and sample covariance:

$$\hat{\mu} = \frac{1}{n}\sum_{i=1}^n x_i, \quad \hat{\Sigma} = \frac{1}{n}\sum_{i=1}^n (x_i - \hat{\mu})(x_i - \hat{\mu})^T$$

5.2 GMM Model Definition 定义

A GMM assumes data is generated from a mixture of \(K\) Gaussian components:

第6讲：无监督学习、K-Means、PCA 与 GMM
$$p(x) = \sum_{k=1}^K \pi_k \mathcal{N}(x \mid \mu_k, \Sigma_k)$$

where:

  • \(\pi_k \geq 0\) and \(\sum_{k=1}^K \pi_k = 1\) are the mixing coefficients (prior probabilities of each component).
  • \(\mu_k \in \mathbb{R}^d\) is the mean of component \(k\).
  • \(\Sigma_k \in \mathbb{R}^{d \times d}\) is the covariance matrix of component \(k\).

Introduce a latent (hidden) variable \(z_i \in \{1, \ldots, K\}\) indicating which component generated \(x_i\):

- $z_i \in \{1,2,\dots,K\}$ 表示样本 $x_i$ 的簇编号 - $\mu_k$ 表示第 $k$ 个簇的中心
$$p(z_i = k) = \pi_k, \quad p(x_i \mid z_i = k) = \mathcal{N}(x_i \mid \mu_k, \Sigma_k)$$

The generative story is: first sample a component \(z_i\) from the categorical distribution \(\text{Cat}(\pi_1, \ldots, \pi_K)\), then sample \(x_i\) from the corresponding Gaussian.

5.3 The Expectation-Maximization Algorithm 算法

EM is an iterative algorithm for maximum likelihood estimation in models with latent variables. The log-likelihood of the observed data is:

$$\log p(X \mid \theta) = \sum_{i=1}^n \log \left(\sum_{k=1}^K \pi_k \mathcal{N}(x_i \mid \mu_k, \Sigma_k)\right)$$

The sum inside the log makes direct optimization difficult. EM resolves this by iteratively optimizing a lower bound (the ELBO).

Why EM? Directly maximizing \(\log p(X \mid \theta)\) with latent variables \(Z\) is intractable due to the sum inside the log. EM constructs a lower bound (Evidence Lower Bound, ELBO) and maximizes it iteratively. Each EM iteration is guaranteed not to decrease the likelihood — it monotonically converges to a local optimum.

5.3.1 E-Step: Compute Responsibilities

The responsibility \(\gamma_{ik}\) is the posterior probability that sample \(i\) belongs to component \(k\), given current parameter estimates:

$$\gamma_{ik} = p(z_i = k \mid x_i, \theta^{\text{old}}) = \frac{\pi_k \mathcal{N}(x_i \mid \mu_k, \Sigma_k)}{\sum_{j=1}^K \pi_j \mathcal{N}(x_i \mid \mu_j, \Sigma_j)}$$

Intuitively: if \(x_i\) is closer to component \(k\) (in Mahalanobis distance), \(\gamma_{ik}\) is larger. All responsibilities for a given point sum to 1: \(\sum_{k=1}^K \gamma_{ik} = 1\). This step does not change parameters — it estimates the "hidden information" (which component each point likely came from).

5.3.2 M-Step: Update Parameters

Using the responsibilities from the E-step, define the effective number of points assigned to component \(k\):

$$N_k = \sum_{i=1}^n \gamma_{ik}$$

Update mixing coefficients:

$$\pi_k^{\text{new}} = \frac{N_k}{n}$$

Update means (weighted average using responsibilities):

$$\mu_k^{\text{new}} = \frac{1}{N_k}\sum_{i=1}^n \gamma_{ik} x_i$$

Update covariances (weighted scatter):

$$\Sigma_k^{\text{new}} = \frac{1}{N_k}\sum_{i=1}^n \gamma_{ik} (x_i - \mu_k^{\text{new}})(x_i - \mu_k^{\text{new}})^T$$

The M-step formulas are exactly the weighted versions of the standard maximum likelihood estimates for a Gaussian. If responsibilities were hard (0 or 1), these reduce to the usual sample mean and covariance.

5.3.3 EM as "Assign then Optimize"

The EM procedure can be summarized as:

  • E-step: Estimate the hidden information — "which component likely generated each point?"
  • M-step: Given this estimate, optimize the parameters — "what are the best means, covariances, and mixing weights?"
  • Repeat until convergence.

This is analogous to K-Means' alternating optimization, but with soft assignments instead of hard ones. K-Means is the limiting case of GMM-EM when all covariances are \(\sigma^2 I\) and \(\sigma^2 \to 0\).

5.4 GMM-EM Pseudocode

Algorithm GMM-EM(X, K, max_iter, tol)
// Input: Data X, number of components K
// Output: Parameters pi, mu, Sigma, responsibilities gamma

1. pi, mu, Sigma = InitializeGMM(X, K) // e.g., from K-Means
2. for iter = 1 to max_iter do
3. // E-Step: Compute responsibilities
4. for i = 1 to n do
5. for k = 1 to K do
6. gamma[i,k] = pi[k] * N(x_i | mu[k], Sigma[k])
7. end for
8. gamma[i,:] = gamma[i,:] / sum(gamma[i,:])
9. end for
10. // M-Step: Update parameters
11. for k = 1 to K do
12. N_k = sum(gamma[:, k])
13. pi[k] = N_k / n
14. mu[k] = (1/N_k) * sum_i(gamma[i,k] * x_i)
15. Sigma[k] = (1/N_k) * sum_i(gamma[i,k] * (x_i - mu[k]) * (x_i - mu[k])^T)
16. end for
17. if log_likelihood_converged(tol) then
18. break
19. end if
20. end for
21. return pi, mu, Sigma, gamma

6. Nonlinear Dimensionality Reduction 6. 非线性降维 for Visualization

PCA is a powerful linear method, but real-world data often lies on nonlinear manifolds (e.g., a Swiss roll in 3D). Two modern methods are essential for visualizing high-dimensional data in 2D or 3D:

6.1 t-SNE (t-Distributed Stochastic Neighbor Embedding)

t-SNE, developed by van der Maaten and Hinton (2008), is a nonlinear technique for visualizing high-dimensional data in 2D or 3D. It preserves local neighborhood structure — nearby points in high-D stay nearby in low-D.

6.1.1 Core Idea

t-SNE constructs probability distributions over pairs of points in both high-D and low-D spaces, then minimizes their divergence:

Step 1 — High-dimensional similarities: Define conditional probabilities using Gaussian kernels:

$$p_{j|i} = \frac{\exp(-\|x_i - x_j\|^2 / 2\sigma_i^2)}{\sum_{k \neq i} \exp(-\|x_i - x_k\|^2 / 2\sigma_i^2)}$$

The bandwidth \(\sigma_i\) is adapted to the local density around point \(i\) via the perplexity parameter (typically 5–50). Symmetrize to get joint probabilities:

$$p_{ij} = \frac{p_{j|i} + p_{i|j}}{2n}$$

Step 2 — Low-dimensional similarities: Define \(q_{ij}\) using a Student t-distribution with one degree of freedom (heavy tails):

$$q_{ij} = \frac{(1 + \|y_i - y_j\|^2)^{-1}}{\sum_{k \neq l}(1 + \|y_k - y_l\|^2)^{-1}}$$

The heavy-tailed t-distribution in the low-dimensional space alleviates the "crowding problem" — the tendency of points to collapse into a single cluster when mapping from high to low dimensions.

Step 3 — Optimization: Minimize the Kullback-Leibler (KL) divergence:

$$\text{KL}(P \| Q) = \sum_{i \neq j} p_{ij} \log\frac{p_{ij}}{q_{ij}}$$

KL divergence is asymmetric: it heavily penalizes placing distant points together (\(p_{ij}\) large, \(q_{ij}\) small), but is lenient about placing dissimilar points close. This is why t-SNE preserves local neighborhoods well but may distort global structure.

6.1.2 Perplexity

Perplexity can be interpreted as a smooth measure of the effective number of neighbors. Low perplexity (e.g., 5) focuses on very local structure; high perplexity (e.g., 50) considers broader neighborhoods. There is no single "correct" value — it depends on the dataset.

6.1.3 Caveats

  • Cluster sizes and inter-cluster distances in the visualization may not reflect true structure.
  • Results depend strongly on perplexity and are non-deterministic (different runs yield different layouts).
  • Cannot transform new data points — the entire dataset must be re-embedded.
  • Computational cost: \(O(n^2)\) time and space; Barnes-Hut approximation reduces to \(O(n \log n)\).

7. Representation Learning 7. 表示学习

Representation learning asks: Does data have a simpler, more essential form beneath its surface? All unsupervised methods in this lecture can be viewed as finding useful representations. Autoencoders provide a neural-network framework for learning such representations, generalizing PCA to nonlinear settings.

7.1 Autoencoders as Nonlinear Representation Learning 表示学习

An autoencoder is a neural network trained to copy its input to its output through an intermediate compressed representation. It consists of:

  • Encoder: \(z = E_\phi(x)\) maps input to a lower-dimensional latent code.
    • EM 是 **Expectation-Maximization** 的缩写，中文一般叫**期望最大化算法**。
  • Decoder: \(\hat{x} = D_\theta(z)\) reconstructs the input from the code.
    • - $z_i \in \{1,2,\dots,K\}$ 表示样本 $x_i$ 的簇编号 - $\mu_k$ 表示第 $k$ 个簇的中心

The objective is to minimize reconstruction error:

$$\min_{\phi, \theta} \frac{1}{n}\sum_{i=1}^n \|x_i - D_\theta(E_\phi(x_i))\|^2$$

When the code dimension \(k < d\), the autoencoder performs nonlinear dimensionality reduction, learning features that capture the most salient structure. Unlike PCA, the encoder and decoder can be arbitrary nonlinear functions (deep neural networks).

7.1.1 Key Variants

Variant Key Idea Application
Denoising Autoencoder (DAE) Reconstruct clean input from a corrupted version (add noise to input, train to recover original) Robust feature learning, denoising
Sparse Autoencoder Enforce sparsity constraint on latent code (most activations are zero) Interpretable, overcomplete representations
Variational Autoencoder (VAE) Latent space is a probability distribution \(q(z|x) = \mathcal{N}(\mu(x), \sigma^2(x))\); trained with ELBO objective Generative modeling, smooth latent space
Contractive Autoencoder Penalize the Frobenius norm of the encoder Jacobian Locally invariant features

7.2 Autoencoders and PCA

A deep connection exists between autoencoders and PCA:

第6讲：无监督学习、K-Means、PCA 与 GMM
Equivalence Theorem: A linear autoencoder (encoder and decoder are both single linear layers) with squared error loss and tied weights (\(D = E^T\)) learns the same subspace as PCA. The optimal encoder weight matrix spans the top \(k\) principal components. Deep autoencoders with nonlinear activations generalize this to nonlinear manifolds.

This means PCA can be seen as the simplest possible autoencoder — one with no hidden nonlinearities. Each additional layer of nonlinearity allows the autoencoder to capture more complex structure that PCA cannot.

7.3 Philosophical Perspectives on Representation Learning 表示学习

Representation learning connects to fundamental questions in epistemology:

  • Plato's Theory of Forms: Observations are imperfect projections of stable abstract structures. The latent representation \(z\) is the "Form" — the essential truth behind the noisy data \(x\).
  • Occam's Razor: A good representation captures maximal essence with minimal parameters. Compressing \(d\) dimensions to \(k\) forces the model to find what truly matters.
  • Encoding-Decoding Symmetry: A valid representation must be both compressible (encoder) and reconstructible (decoder). If you cannot reconstruct the original from a representation, that representation has lost something essential.
Human Cognition: The human brain performs unsupervised learning constantly — transforming pixels into objects, waveforms into meaning. This reflects a historical shift in AI: from human-defined features (hand-crafted rules) to data-defined features (learned representations).
"If intelligence is a cake, unsupervised learning is the cake itself, supervised learning is the icing, and reinforcement learning is the cherry on top." — Yann LeCun

The Open Question: Does an "optimal representation" of the world exist independently of any task, or is every representation task-relative? This touches the core of epistemology: Is our understanding the world as it truly is, or merely an efficient compression our brains construct?

9. Summary 9. 本讲总结

This lecture explored unsupervised learning as a journey from raw data to essential structure. By stripping away labels, we force our models to discover the intrinsic geometry and statistical distributions that define "similarity."

Core Takeaways

  1. K-Means: Geometric clustering via hard centroid assignment. Minimizes WCSS by alternating between assignment and mean update. Simple, fast, but assumes spherical clusters.
  2. PCA: Linear dimensionality reduction via variance maximization (equivalently, reconstruction error minimization). Principal components are eigenvectors of the covariance matrix. SVD is the preferred computation method.
  3. GMM + EM: Probabilistic soft clustering via latent variable modeling. EM alternates between estimating responsibilities (E-step) and updating parameters (M-step). Subsumes K-Means as a limiting case.

  4. DBSCAN: Density-based discovery of arbitrary-shaped clusters with explicit noise handling. No need to specify \(K\).
  5. t-SNE / UMAP: Manifold learning for high-dimensional visualization. t-SNE uses KL divergence on heavy-tailed distributions; UMAP uses cross-entropy on fuzzy topological structures.
  6. Autoencoders: Nonlinear generalization of PCA through neural network encode-decode architectures. Linear autoencoders are equivalent to PCA; deep autoencoders capture nonlinear manifolds.

The One Equation to Remember

$$\max_{\|w\|=1} w^T S w \quad \Longrightarrow \quad Sw = \lambda w$$

This is PCA's core: principal components are eigenvectors of the covariance matrix, ordered by eigenvalue magnitude.

The Single Most Important 重要 Conclusion

The core of unsupervised learning is not "having no labels" — it is "how to define structure." K-Means solves "how to group," PCA solves "how to compress information," GMM solves "how data is generated and how to assign softly." The choice of features and similarity metrics determines everything.
